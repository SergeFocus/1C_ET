Briefly how it works:
Users work with the accounting system - open the window, enter data into the table.
You can set the time between such operations, which are considered acceptable pause (at least 2 minutes, I recommend - 15).

While the program detects the user experience, it is counted during this period as working time.
If the user is idle longer valid pause, it is believed that he left the workplace.
You can activate a question for some time before the expiration of the user to confirm that he was on the job for 30 seconds.

If the user returns for the workplace - it is over the break and starts a new unit of work.
