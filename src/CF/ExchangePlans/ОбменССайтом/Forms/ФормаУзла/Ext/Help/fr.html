Le noeud de communication avec le site Web

Permet de configurer l'échange de données entre le système d'information comptable et de boutique en ligne qui prend en charge Protocole entre le système "1C: Enterprise" et le site Web.

Principales caractéristiques:

  • Décharge du site le catalogue de produits, y compris toutes les propriétés des produits, les prix et les restes des marchandises dans les entrepôts, ainsi que la maintenance automatique des données à jour du programme.
  • Télécharger le programme de l'ordonnance rendue dans la boutique, ainsi que la mise à jour des données sur le site Web de l'état de paiement et expédition des commandes.

paramètres de base
  • Mode de communication - Il détermine les ensembles de données participeront à l'échange (déchargement des marchandises et / ou d'échange d'informations sur demande).
  • Procédé d'échange de données - La communication directe avec le site ou par l'intermédiaire d'un répertoire sur le disque, dans lequel les fichiers seront placés échange.
  • L'échange automatique - La capacité de tourner sur l'échange automatique et régler la fréquence de son exécution.
  • Configuration avancée - Réglez la taille de la portion téléchargé les données du site et le nombre de tentatives de change en cas d'erreur.

Méthode de communication Définition:
  • Dans un échange direct avec le site, vous devez indiquer l'adresse du site, le nom d'utilisateur et mot de passe pour accéder au site.
  • Lorsque vous utilisez le partage de répertoire est nécessaire de préciser le téléchargement et le fichier répertoire, qui contient des informations sur les commandes passées sur le site.

Drapeau Les ordres d'échange sera disponible uniquement si l'option est activée, le fonctionnel Commandes des clients Dans le chapitre Administration - CRM et de la vente.

Réglage de déchargement des marchandises
  • L'organisation - le propriétaire du catalogue de produits - L'organisation au nom de laquelle le catalogue publié sur le site.
  • Les données chargées - Catalogue des produits et \ ou des prix et des résidus. Il indique la nécessité pour le déchargement des images et autres fichiers entrepôts disponibles pour la sélection par l'acheteur sur le site. Il est également possible d'installer une nomenclature de classification variante - par type d'élément. La version par défaut de la classification - par des groupes de la nomenclature.
  • Données mode de téléchargement - Décharge pleine, ou uniquement les données modifiées depuis le dernier envoi.
  • Catalogues paramètres produits - Vous devez définir un ou plusieurs dossiers sur le site et mettre en place la sélection libérables de biens et services.

Téléchargez seulement les accords sur les prix et les restes d'entrepôts sans le catalogue de produits de sens que si le catalogue avait déjà été téléchargé sur le site.

Les ordres d'échange

Paramètres supplémentaires ordres de change
  • Enregistrez documente la date actuelle - Si elle est cochée, les commandes passées sur le site enregistre la date de réception de l'information dans la base de données, sinon, la date de la commande sur le site.
  • Maintenir l'ordre de numérotation selon le site - Si le drapeau est réglé, la numérotation des commandes passées sur le site sera maintenu, sinon, la commande sera attribué le numéro suivant, selon la numérotation de la base de l'information.
  • L'état des commandes "Annulé" sur le site - Si le site a reçu des informations que la demande est attribué le statut, toutes les lignes dans l'ordre créé précédemment seront annulées.
  • Raison de l'annulation - En cas d'annulation, à partir des lignes annulées sera installé cette raison de l'annulation.
  • Conformité avec les informations d'état de l'ordre dans la base de données et sur le site Web - Définir le statut approprié de commandes en ligne, et la base de l'information
