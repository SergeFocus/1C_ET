Der Kommunikationsknoten mit der Website

Hier können Sie den Datenaustausch zwischen den Rechnungslegungsinformationssystem und Online-Shop, unterstützt die Konfiguration Protokoll zwischen dem System "1C: Enterprise" und Website.

Hauptmerkmale:

  • Entlassung aus der Programm-Website Katalog von Produkten, einschließlich aller Eigenschaften der Ware, die Preise und den Überresten der Waren in Lagerhallen, sowie die automatische Pflege der Daten auf dem neuesten Stand.
  • Laden Sie das Programm der im Laden ausgestellt Ordnung, sowie die Aktualisierung der Daten auf der Website der Status der Zahlung und Versand von Bestellungen.

Grundeinstellungen
  • Kommunikationsmodus - Es legt fest, welche Datenmengen wird in der Vermittlungsstelle (Entladen von Gütern und / oder den Austausch von Informationen auf Anfrage) zu beteiligen.
  • Das Verfahren für den Datenaustausch - Direkte Kommunikation mit der Website oder über ein Verzeichnis auf der Festplatte, in dem die Dateien Austausch gelegt werden.
  • Automatischer Austausch - Die Fähigkeit, auf den automatischen Austausch wenden und stellen Sie die Frequenz der Ausführung.
  • Advanced Setup - Stellen Sie die Portionsgröße auf die Website hochgeladen Daten und die Anzahl der Wiederholungen der Austausch im Fehlerfall.

Einstellen Methode der Kommunikation:
  • Im direkten Austausch mit der Website müssen Sie die Adresse der Website der Benutzername und das Passwort, um Zugriff auf die Website angeben.
  • Bei Verwendung der Verzeichnisfreigabe erforderlich ist, um das Verzeichnis-Upload und-Datei, die Informationen über die Bestellungen auf der Website platziert enthält angeben.

Flagge Umtauschanträge ist nur verfügbar, wenn die Option aktiviert ist, wird die Funktions Kundenaufträge Im Kapitel Administration - CRM und Vertrieb.

Einstellung Entladen von Gütern
  • Die Organisation - der Besitzer des Produktkatalogs - Die Organisation, in deren Auftrag der Katalog auf der Website veröffentlicht.
  • Hochgeladene Daten - Katalog von Waren und \ oder die Preise und Rückstände. Es zeigt die Notwendigkeit für das Entladen von Bildern und anderen Dateien, Lagerhallen zur Auswahl durch den Käufer auf der Website. Es ist auch möglich, eine Variante Klassifikationsschema zu installieren - vom Typ des Elements. Die Standard-Version des Klassifikations - durch Gruppen von Nomenklatur.
  • Modus Upload-Daten - Vollentladung, oder nur die geänderten Daten seit dem letzten Upload.
  • Kataloge Einstellungen Waren - Sie müssen einen oder mehrere Ordner auf der Website definieren und die entladbare Auswahl an Waren und Dienstleistungen.

Laden Sie nur die Preisvereinbarungen und die Überreste von Lagerhallen, ohne den Produktkatalog ist sinnvoll, wenn der Katalog bereits auf die Website hochgeladen.

Weitere Einstellungen Umtauschaufträge
  • Nehmen Sie dokumentiert das aktuelle Datum - Wenn aktiviert, werden die Aufträge auf der Website platziert das Datum des Eingangs der Informationen in der Datenbank aufnehmen, wenn nicht, das Datum der Bestellung auf der Website.
  • Pflegen Sie die Nummerierung Um nach der Website - Wenn das Flag gesetzt ist, wird die Nummerierung der Bestellungen auf der Website platziert aufrecht erhalten werden, wenn nicht, wird der Auftrag die nächste Nummer zugewiesen werden, entsprechend der Nummerierung der Informationsbasis.
  • Auftragsstatus "Cancelled" auf der Website - Wenn die Website Informationen, die der Antrag den Status zugewiesen erhalten, werden alle Zeilen in der zuvor angelegten Bestellung storniert.
  • Grund für die Stornierung - Im Falle der Annullierung, der abgebrochen Linien dieses Kündigungsgrund installiert werden.
  • Die Einhaltung der Auftragsstatusinformationen in der Datenbank und auf der Website - Legen Sie den entsprechenden Status von Bestellungen online, und die Informationsbasis
