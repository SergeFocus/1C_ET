Console Anforderungen für verwaltete Anwendungen

Tool "Console-Anforderungen" wird für das Debuggen und das Anzeigen von Abfrageergebnissen in der 1C bestimmt sind: Enterprise-Modus.Diese Behandlung ist vor allem für Entwickler von Konfigurationen und Implementierungsspezialisten.

Wenn Abfragen im Konfigurator, in der Regel zu entwickeln, eine Abfrage auf realen Daten zu debuggen notwendig ist.Dieses Tool ermöglicht es, die Entwicklung (oder Abfrage-Paket) parallel zu den Scan-Ergebnissen zu beantragen.Wenn es mit dem Werkzeug in einem Thick-Client arbeiten, können Sie die Abfrage-Designer verwenden, als würden Sie im Konfigurator.Eigenschaften: Ergebnis-Analyse sind:

  • ableitung von Datentabellen,
  • messzeit der Abfrage und der Anzahl der Zeilen,
  • diese Zellen als Folge der Anforderung Hervorhebungen,
  • interaktiven Vergleich der beiden Ergebnisse der Abfrage (im Thick-Client-only)
  • sowie die Ausgabe Ergebnis der Abfrage in einem neuen Fenster.
  • abgeschlossen Abfrageausführungsplan und SQL-Abfragetext in der Datenbank generiert.Für Microsoft SQL Server DBMS Ausführungsplan wird als ein Baum, und für den Rest der Datenbank angezeigt - im Textformat-Technologie-Magazin.Um die Analyse von Anforderungen vereinfachen auch zwei Textanzeigemodus zur Verfügung gestellt wird, Abfragen mit den Namen der Datenbank-Tabellen und Spalten, oder mit den Namen der Metadatenobjekte und Konfigurationsdetails.

Nach Abschluss des Debugging, kann die Abfrage Text auf den Code übertragen werden (durch Eingabe von Textform Anfrage Konfigurator) oder in den Konfigurationsberichte.Für Service-Funktionen umfassen mehrere Anfragen (Request-Paket) Handhabung, die Erhaltung des Textes und die Abfrage-Parameter in der Datei, das automatische Speichern, Export Abfrageergebnisse in eine Tabellendokument und vieles mehr.

Wenn die Abfrage bei der Ableitung der Ergebnisse erforderlich ist, ein Semikolon zu verwenden, sollte es Backslash entkommen - "
Beispiel:WÄHLEN Nomenklatura.Naimenovanie + "\;" + Nomenklatura.Naimenovanie VON Spravochnik.Nomenklatura

Schnelle Datenübertragungsanforderung des Textes und seine Parameter in der Abfrage von der Debugger-Konsole.

Die Standard Library Subsystemen Version 2.3.2 und höher gibt es eine Funktion "ObschegoNaznacheniya.ZaprosVStrokuXML ", die Sie schnell auf Anfrage (alle den Text und die Parameter) aus dem Debugger übertragen die Konsole abfragen.Um dies zu tun:

  • Der Konfigurator auf einen Haltepunkt unmittelbar vor der Methode "Ausführen" den Aufruf der Abfrage und Informationsbasis im Debug-Modus laufen zu lassen.
  • Wenn die Codeausführung an dem Haltepunkt stoppt, drücken Sie die Umschalttaste + F9 (oder im Menü "Test", drücken Sie "Berechnen Sie den Ausdruck").Im Fenster "Ausdruck":
    • Im Feld Ausdruck eingeben "ObschegoNaznacheniya.ZaprosVStrokuXML (Request) ", wobei die" Anforderung "ist der Name einer Variable vom Typ Request.
    • Die Tabelle, die Ergebnisse Markierungsbalken wird mit dem Ergebnis, erscheinen die Expression und drücken Sie F2, der Auswertung (oder drücken Sie die "Show-Werte in einem separaten Fenster" aus dem Kontextmenü).
    • Im Fenster "Ansicht des Ausdrucks" den gesamten Text zu verteilen und kopieren Sie sie in die Zwischenablage.
  • Klicken Sie auf "OK", um das Fenster "Ausdruck" zu schließen, um einen Haltepunkt zu deaktivieren, weiterhin Ausführung (F5) und geben Sie die Abfrage in der Enterprise Console-Modus.
  • Die Konsole Abfragen den kopierten Text in der Abfragetext einfügen und das Menü "Mehr" Presse "die XML-Ausfüllen".
