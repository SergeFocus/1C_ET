Richieste di console per un'applicazione gestita

Strumenti "richieste per Console" è progettato per stabilire e rivedere i risultati della query in 1C: Enterprise.Questo trattamento è destinato principalmente agli sviluppatori configurazioni ed esperti di implementazione.

Nello sviluppo query nel configuratore di solito è necessario condurre richiesta adeguamento sui dati reali.Questo strumento permette di progettare la query (o query) di pacchetti risultati di visualizzazione parallele.Quando si utilizza lo strumento nel client di spessore in grado di utilizzare il Generatore di query, come si farebbe nel configuratore.Caratteristiche della domanda risultato dell'analisi includono:

  • trovare le tabelle di dati temporanei
  • tempo di misura della richiesta e il numero di linee
  • evidenziando queste cellule come risultato della richiesta,
  • interattivo confronto tra due risultati delle query (solo client di spessore)
  • e trovare il risultato della query in una nuova finestra.
  • interrogazione piano di esecuzione e SQL-query testo concluso generato nel database.Per Microsoft SQL Server DBMS piano di esecuzione viene visualizzato come un albero, e per il resto del database - in rivista di tecnologia formato testo.Per semplificare la richiesta di analisi prevede anche due interrogazioni modalità di visualizzazione di parole: i nomi di tabelle del database e colonne o nomi degli oggetti e dettagli di configurazione dei metadati.

Dopo la regolazione, la query può essere trasferito in codice (usando il testo richiesta per la formazione configuratore) o la segnalazione di configurazione.Capacità di servizio per includere il lavoro con diverse query (interrogazioni pacchetto) conservano parametri di testo e di query nel file di salvataggio automatico, i risultati delle query Esporta in un foglio di calcolo e documenti di più.

Se i risultati della query di derivazione devono utilizzare un punto e virgola, deve essere sfuggito backslash - "
Esempio:SCEGLIERE Nomenklatura.Naimenovanie + "\"; + Con Nomenklatura.Naimenovanie Spravochnik.Nomenklatura

testo rapido avvolgendo la richiesta ed i suoi parametri delle richieste console debugger.

I sottosistemi Biblioteca versioni standard 2.3.2 e, soprattutto, c'è l'opzione "ObschehoNaznacheniya.ZaprosVStrokuXML ", che permette di trasferire rapidamente richiesta (tutto il testo e le impostazioni) da una console debugger richieste.Per fare questo:

  • Nella Configurazione e impostare un punto di interruzione immediatamente prima di chiamare il metodo "Esegui" ed eseguire le informazioni della query base in modalità di debug.
  • Quando l'esecuzione di codice si ferma alla fermata punto premete Shift + F9 (o in "Impostazioni" cliccare su "Calcola l'espressione").Nella "Expression":
    • Nel campo Espressione immettere "ObschehoNaznacheniya.ZaprosVStrokuXML (Request) ", dove" Request "è il nome di una variabile di tipo Request.
    • Nella tabella dei risultati è apparso evidenziare la linea con il risultato dell'espressione di calcolo e premere F2 (o il menu di scelta rapida, fare clic su "Mostra valore in una finestra separata").
    • Nella "Visualizzazione l'espressione" seleziona tutto e copiare il testo negli appunti.
  • Fai clic su "OK" per chiudere la finestra "espressione", disabilitare punto di interruzione, continuare l'esecuzione (F5) e andare alla query console in Enterprise.
  • Richieste Console incollare il testo copiato in un campo di testo e query nella "More" click "Riempire con XML".
