설정 메모리 누수

메모리 누수의 제어를위한 두 가지 설정을 충족합니다 :

  • 모니터 서버 메모리 사용량. 이것만으로도 하나, 이벤트 등 모든 이벤트의 프로세스 로그에 결론을 포함해야한다 (MEM) (메모리 누수 서버).
  • 코드 구성의 트랙 메모리 누수. 이 경우, 당신은 당신이 메모리 누수의 구성 여부를 제어 할 필요가있는 지역을 지정해야합니다. 이것은 플래그를 사용하여 수행 할 수있다 체크 포인트 스타트 / 클라이언트에서 코드 실행을 중지.체크 포인트 시작 / 서버에서 코드 실행을 중지그리고 제어 모듈에 대한 정보를 입력 나열합니다. 이것만으로도 하나, 이벤트 등 모든 이벤트의 프로세스 로그에 결론을 포함해야한다 누수 (메모리 누수).

과학 기술 기본 로그의 설정

기본의 기술 저널 - ( ": 기업 1C"시스템의 관점에서) 중요한 상황에서 발생하는 이벤트의 기록 처리 로그의 특별한 종류이다. 이를 위해 잡지 플랫폼을 형성하는 변경할 수없는 고정 된 이벤트 필터를 만들었다.

그러나, 구성 할 수있다 위치 시스템 기술 잡지 과 이력, 클럭의 저장 기간. 기본적으로 로그를 저장하는 과정을 설정하려면 기본 하이퍼 링크 시스템 구성 프로세스의 기본 로그 설정.

테이블을 사용하여 시스템 이벤트 당신은 기록 된 이벤트의 세부 사항을 지정할 수 있습니다.
