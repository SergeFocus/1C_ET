Механизм "реестр продаж" предназначен для регистрации операций по реализации товара или услуг клиентам по-одному в виде таблицы в режиме "незакрываемого окна" с крупными элементами интерфейса.


Механизм может быть интерес только для случаев, когда бизнес процесс для данного торгового места выглядит как реализация одного товара или услуги для одного клиента.

В конфигурации предусмотрено 5 режимов регистрации продаж:
  • Обычный (создание отдельных документов реализации)
  • За счет инвентаризации
  • Простые продажи (окно регистрации реализации не закрывается)
  • Реестр продаж (продажа по-одной позиции для покупателей в виде общей таблицы)
  • АРМ кассира (на апрель 2016 не реализовано)
В интерфейсе "реестр продаж" функционал регистрации реализации ограничен.
Для учета сложных элементов придётся открывать обыкновенный (обычный) документ реализации.

Так, например, за счет механизма регистрации "простые продажи" 
НЕЛЬЗЯ регистрировать:
  • продавать более одного товара одной одному клиенту операцией
  • учитывать договора клиентов
  • продавать товары и услуги со скидкой
  • принимать деньги от клиента при оплате одновременно в нескольких различных валютах
  • принимать деньги от клиента при оплате одновременно в нескольких формах оплаты
  • принимать деньги от клиента при оплате одновременно в различные хранилища денег
  • списывать товар
  • продавать серийный товар
  • продавать товар в дополнительных единицах измерения
  • возвращать товары поставщику
  • регистрировать один документ реализации, но по разным заказам клиента
Механизм включается в настройках интерфейса и внешнего вида:


Работа в режиме "реестр продаж" выглядит следующим образом:
Перед пользователем всё время открыто окно таблицы. Где каждая строка - самостоятельный документ реализации.

Пользователь вносит условия реализации - покупателя, товар или услугу, сумму оплаты и программа формирует документ реализации.

Дополнительно на форме пользователю доступно:
1 - кнопки перехода к обычному списку документов реализации, документы движения денег и другие.
2 - выбор периода для таблицы. Когда пользователь меняет период - программа считывает в таблицу операции реализации. ВАЖНО: в таблицу попадут только операции реализации с одной строкой таблицы товары. Прочие, более сложные документы реализации в таблицу не считываются.
3 - возможность задать общие условия для документов реализации: вид цен, склад, собственная организация, хранилища денег.
 
Это окно не самостоятельный интерфейс. Пользователь по-прежнему может работать с прочими участками базы данных и, при необходимости, открывать реализацию в привычном виде.

Форму "реестра продаж" можно назначить пользователю в качестве формы "на старте".

Форма "реестр продаж" доступна для выбора в качестве элемента начальной страницы пользователя.
