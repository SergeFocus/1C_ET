고객에게 상품이나 서비스의 판매 기록 거래에 대해 "영업의 등록"의 메커니즘하나씩 주요 인터페이스 요소와 "비 - 재 밀봉 박스"의 테이블의 형태 일 수있다.


소매 공간에 대한 비즈니스 프로세스가 하나의 클라이언트에 대한 상품 또는 서비스 중 하나의 실현과 같은 곳 메커니즘은 경우에 관심이 될 수 있습니다.

구성은 5 등록 모드 판매를 제공 :
  • 정상 (개별 구현 문서의 작성)
  • ...을 통하여목록
  • 단순 판매 (등록 윈도우의 구현은 폐쇄되지 않음)
  • 등록 판매 (일반 테이블의 형태로 구매자에게 판매-항목)
  • ARM의 점원 (4 월 2016이 구현되지 않음)
인터페이스 "판매 등록은"등록 기능 구현이 제한된다.
복잡한 과금 요소의 구현을위한 통상의 (공통의) 문서를 열거 할 것이다.

따라서, 예를 들어,기구 "단순 판매"로그인하여
등록하지 :
  • 이상의 제품 하나의 클라이언트의 동작에 판매
  • 계정 고객 계약 고려
  • 할인 상품과 서비스를 판매
  • 여러 다른 통화로 지불하는 경우 클라이언트에서 돈을 촬영합니다
  • 여러 가지로 지불하는 경우 클라이언트에서 돈을 촬영합니다 결제 방법
  • 동시에 지불 할 때 클라이언트에서 돈을 촬영합니다 다양한 돈을 금고에
  • 직불 항목
  • 시리얼 제품을 판매
  • 추가 단위로 상품을 판매하는
  • 공급 업체에 제품을 반환
  • 한 문서의 구현,하지만 서로 다른 고객의 주문을 등록
메커니즘은 인터페이스의 설정 및 외관에 포함된다 :


다음과 같이 "판매 등록"에서 작업 모드는 다음과 같습니다
테이블 창 항상 사용 전에 개방된다.여기서 각 라인 - 별도의 문서의 구현입니다.

사용자는 이행 조건하게 - 구매자, 제품 또는 서비스의 결제 금액을 상기 프로그램 실행 문서를 생성한다.

사용자가 이용할 수있는 형태 이외에 :
1 - 문서의 일반적인 목록 주요 문서 및 기타 현금 흐름의 구현으로 이동합니다.
2 - 테이블의 기간을 선택합니다.사용자가 시간을 변경하는 경우 - 프로그램은 작업 테이블의 구현에서 읽는다.중요 : 테이블은 테이블 제품의 단일 라인 작업의 구현을 포함 할 것이다.문서의 다른 더 복잡한 구현 테이블에서 판독 할 수 없다.
- 3 문서의 구현을위한 일반 조건 문의 : 가격, 저장, 민간 조직, 돈의 저장의 유형입니다.
이 창은 독립적 인 인터페이스가 아닙니다.사용자는 필요에 오픈 형태의 일반적인 구현, 데이터베이스의 다른 부분과 협력을 계속할 수 있습니다.
