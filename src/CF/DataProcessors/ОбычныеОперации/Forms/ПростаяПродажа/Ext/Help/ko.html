"단순 판매"의 메커니즘 주요 인터페이스 요소와 함께 "비 - 재 밀폐 박스"에서 상품과 서비스의 판매 트랜잭션을 기록.


구성은 5 등록 모드 판매를 제공 :
  • 정상 (개별 구현 문서의 작성)
  • ...을 통하여 목록
  • 단순 판매 (등록 윈도우의 구현은 폐쇄되지 않음)
  • 판매의 등록 (일반 테이블의 형태로 고객에게 판매 한 위치)
  • ARM의 점원 (4 월 2016이 구현되지 않음)
인터페이스 "단순 판매"등록 기능 구현이 제한된다.
복잡한 과금 요소의 구현을위한 통상의 (공통의) 문서를 열거 할 것이다.

따라서, 예를 들어,기구 "단순 판매"로그인하여
등록하지 :
  • 할인 상품과 서비스를 판매
  • 여러 다른 통화로 지불하는 경우 클라이언트에서 돈을 촬영합니다
  • 여러 가지로 지불하는 경우 클라이언트에서 돈을 촬영합니다 결제 방법
  • 동시에 지불 할 때 클라이언트에서 돈을 촬영합니다 다양한 돈을 금고에
  • 직불 항목
  • 시리얼 제품을 판매
  • 추가 단위로 상품을 판매하는
  • 공급 업체에 제품을 반환
  • 한 문서의 구현,하지만 서로 다른 고객의 주문을 등록
메커니즘은 인터페이스의 설정 및 외관에 포함된다 :


다음과 같이 "단순 판매"모드에서 작동 :
사용 전에 모든 시간 윈도우는 문서 열기 구현이다.
고객의 선택 후, 판매, 등록 명명법 돈 영수증의 조건은, 사용자가 "판매"버튼을 누른다.
고객의 선택
판매 목록에있는 상품과 서비스의 항목
결제 정보를 입력합니다 (설정에서 활성화 된 경우)
이때, 구현이 기록되고, 윈도우는 다음의 판매를 위해 클리어된다.
이 문서는 "이야기에."보내마지막 문서 위에 표시됩니다.

간단히 볼 수있는 다른 사용자의 능력에 대한 :
  1. 본체의 버튼 :
    1. 이 윈도우의 구현 기존 문서를 여는
    2. 구현 장비의 목록
    3. 여기 개구는 마지막이 창에 다시 문서 구현을 등록
  2. 구현을위한 조건을 설정 (창고, 가격의 민간 조직 형태, 돈의 저장, 고객의 주문)
  3. 인쇄 된 문서 형태의 구현의 형성에 거주
  4. 여기에 입력 된 문서의 실현의 일반 모드에서 열기 데이터.이 창에서, 등록 자금이 충분하지 않은 것으로 나타났습니다 경우 (예를 들어,이 메커니즘의 제한 목록 위에 참조).
  5. 그 모니터에 따라 "자체"모양을 사용자 정의합니다.
이 창은 독립적 인 인터페이스가 아닙니다.사용자는 일반적인 형태로 데이터베이스 및 오픈 구현의 다른 영역, 필요한 경우 사용할 수 있습니다.

"단순 판매"의 형태는 형태로 사용자에게 할당 할 수있는 "시작시."

모양 "단순 판매는"사용자의 페이지의 구성원으로 선택할 수 있습니다.
