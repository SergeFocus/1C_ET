Traitement "déchargement et le chargement XML"

Modes de fonctionnement

Traitement implémente deux modes de fonctionnement: débarquements (créer un données spécifiées par l'utilisateur Téléchargement de fichier) et Téléchargements (lecture d'un téléchargement de fichier, le même nom créé par le régime, et de l'écrire dans les données disponibles). Cadre est effectuée dans le mode de sélection.

Avant de commencer un mode particulier (appuyez sur le bouton Exécuter), vous devez spécifier le nom de l'upload de fichiers ou en entrant manuellement dans le "Nom de fichier" ou sélectionnez-le en utilisant le champ de bouton et de dialogue de fichier standard.

Dans la chaussure il est possible d'utiliser les résultats des registres d'enregistrement d'édition qui peuvent affecter la vitesse de téléchargement.
Boutons "Désactiver les résultats" et "Inclure les résultats" sont disponibles lorsque le drapeau est réglé "Activer la possibilité de modifier les résultats de l'utilisation pendant le chargement des données» et sont utilisés pour le manuel mode d'utilisation des résultats de démarrage de l'opération données.

Conditions d'application du traitement

Traitement ne peut être utilisé dans les cas où la base de l'information, dans laquelle les données de décharge et celui dans lequel les données sont chargées, sont homogènes (données identiques de configuration peuvent varier), ou tous les objets déchargés sont presque complètement identique en composition et le type de détails et des parties de table, propriétés "hôte" métadonnées d'objet, etc. Il convient de noter que, dans le cadre de ces contraintes, le traitement est principalement destiné aux échanges entre IB homogène.

Format de fichier est différent du format de fichier de déchargement créé pendant le déchargement échange plan chapeau. Pour télécharger des données (répertoires éléments, registres et dossiers d'ensembles, etc.) traitement utilise le même mécanisme XML-sérialisation que les plans d'évacuation pour l'échange, dans cette partie des formats de fichiers sont identiques.

Détermination de la décharge

Traitement permet à la fois le déchargement complet ou partiel de l'information de la base de données dans un fichier. Réglage de la composition des données téléchargées est fait dans un dialogue en définissant des indicateurs dans la colonne d'un arbre qui affiche les objets de métadonnées pour lesquels les données peuvent être téléchargées. Drapeaux de colonne supplémentaire, «si nécessaire», établit la nécessité de décharger des objets de ce «lien» de type. Autrement dit, si la case est cochée uniquement dans le «si nécessaire», les données de cet objet ne seront pas déchargées dans son intégralité, et seulement dans la mesure qui est nécessaire pour maintenir l'intégrité référentielle dans la base de données, qui sera télécharger le fichier télécharger.

Lorsque vous ouvrez un traitement de formulaire définit le lien drapeau pour télécharger tous les objets qui garantit l'intégrité référentielle déchargée fragment de base de données.

En cliquant sur les "Définir les données déchargées sur le lien" analyse le traitement et dont les références peuvent être contenues dans les objets qui ont le signe d'une décharge complète et remplit automatiquement dans la colonne des indicateurs qui indiquent la nécessité de décharger ici. Si l'objet a déjà une boîte ensemble complet décharge, puis vérifier le lien de déchargement est pas établie.

Les applications possibles

L'utilisation de ce traitement peut être, par exemple, pour créer un ensemble de données complet ou partiel de la sauvegarde échange de données entre les bases de données d'information, ainsi que d'un outil auxiliaire dans la réduction du problème de bases de données.
