Traitement "déchargement et le chargement des données XML"

Les modes de fonctionnement

Traitement met en œuvre deux modes de fonctionnement: Débarquements (la création de ces données par l'utilisateur de télécharger un fichier) et la charge (décharger la lecture du fichier créé le mode éponyme, et enregistrer les données disponibles à l'intérieur). Réglage du mode en choisissant le champ Mode.

Avant de commencer ce ou tout autre mode (appuyez sur le bouton Exécuter), vous devez spécifier le nom de l'upload de fichiers ou en entrant manuellement dans le "Nom de fichier", ou en utilisant le bouton sélectionner ce domaine et de dialogue de sélection de fichier standard.

En mode de démarrage, il est possible de modifier les résultats de l'utilisation de registres à l'entrée, ce qui peut affecter la vitesse de téléchargement.
Boutons "Désactiver les résultats" et "Inclure les résultats" sont disponibles lorsque le drapeau est réglé sur "Activer la possibilité de modifier les résultats de l'utilisation lors du téléchargement de données" et sont utilisés pour le contrôle manuel des résultats de l'utilisation lors du chargement des données.

Les conditions de traitement pour l'applicabilité

Traitement ne peut être utilisé dans les cas où la base de l'information, dans laquelle des données est déchargé, et celui dans lequel les données sont chargées, sont homogènes (données identiques de configuration peuvent varier).Soit tous les objets déchargés à peu près identiques dans la composition et le type de détails et de pièces de table, les propriétés "leader" des métadonnées d'objet, etc.Il convient de noter que, dans le cadre de ces contraintes, le traitement est principalement destinée à l'échange entre IB homogène.

Télécharger un format de fichier différent du fichier de format créé lors du déchargement sur le plan de l'échange, la partie d'en-tête.Pour télécharger les données (guides éléments, ensembles de registres et dossiers, etc.) traitement en utilisant le même mécanisme XML-sérialisation que la décharge pour l'échange de plans dans cette partie des formats de fichiers sont identiques.

Détermination de la composition des rejets

Traitement permet à la fois le déchargement complet ou partiel de l'information de la base de données dans un fichier.Réglage des données de composition est déchargé dans le dialogue en définissant des indicateurs dans la colonne de bois qui affiche les objets de métadonnées pour lesquels les données peuvent être téléchargées. drapeaux des colonnes supplémentaires "si nécessaire", établit la nécessité pour le déchargement des objets de "lien" ce type.Autrement dit, si la case est cochée uniquement dans le «si nécessaire», les données de cet objet ne seront pas déchargées dans son intégralité, et uniquement dans la mesure où cela est nécessaire pour maintenir l'intégrité référentielle dans la base de données, qui sera charger le fichier télécharger.

Lorsque vous ouvrez un traitement de forme définit l'indicateur de télécharger le lien à tous les objets qui assure l'intégrité référentielle déchargé fragment de base de données.

En cliquant sur le bouton "Définir les données, de télécharger le lien" analyse le traitement, les références à laquelle les données peuvent être retenues dans des locaux qui ont installé un signe de décharge complète, et remplit automatiquement dans la colonne des indicateurs qui indiquent la nécessité de télécharger le lien.Si l'objet a déjà un ensemble de boîte de décharge complète, dans ce cas, la décharge vérifier le lien ne semble pas établie.

Les applications possibles

L'utilisation de ce traitement peut être, par exemple, pour créer un ensemble de données de sauvegarde complète ou partielle de l'échange de données entre les bases de données, ainsi que d'un outil auxiliaire dans la réduction du problème de bases de données.
