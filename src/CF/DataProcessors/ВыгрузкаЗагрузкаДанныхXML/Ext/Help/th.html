การประมวลผล "อัพโหลดและดาวน์โหลด XML"

โหมด

ประมวลผลการขาย 2 โหมดของการดำเนินงาน: (ข้อมูลผู้ใช้ขนถ่ายการสร้างแฟ้มกำหนด) และโหลด Discharge (ขนอ่านไฟล์ที่สร้างขึ้นโดยระบอบการปกครองเดียวกันและบันทึกข้อมูลที่มีอยู่) เลือกโหมดการติดตั้งจะทำในโหมดที่เหมาะสม

ก่อนที่จะเริ่มโหมดเฉพาะ (คลิก Run) ต้องระบุชื่อของการอัปโหลดไฟล์หรือด้วยตนเองป้อนใน "ชื่อแฟ้ม" หรือจะใช้ปุ่มเพื่อเลือกสาขานี้และโต้ตอบเลือกไฟล์มาตรฐาน

ในโหมดการบูตมันจะเป็นไปได้ในการปรับเปลี่ยนการใช้ผลของการลงทะเบียนในรายการซึ่งสามารถส่งผลกระทบต่อความเร็วในการดาวน์โหลด
ปุ่ม "ปิดใช้ผล" และ "เปิดขึ้น" สามารถใช้ได้เมื่อธง "เปิดใช้งานความสามารถในการแก้ไขโดยใช้ผลเมื่อโหลดข้อมูล" และมีการใช้โหมดการควบคุมด้วยตนเองโดยใช้ผลเมื่อโหลดข้อมูล

เงื่อนไขของการประมวลผลการบังคับใช้

มันควรจะตั้งข้อสังเกตว่าในการเชื่อมต่อกับข้อ จำกัด เหล่านี้จัดการวัตถุประสงค์หลักสำหรับการแลกเปลี่ยนระหว่างที่แตกต่างกัน IB

รูปแบบไฟล์ที่แตกต่างกันจากการปลดรูปแบบไฟล์ที่สร้างขึ้นในระหว่างการขนถ่ายแลกเปลี่ยนแผนส่วนหนึ่งชื่อยกเลิกการโหลดข้อมูล (องค์ประกอบอ้างอิงชุดลงทะเบียนบันทึก ฯลฯ ) การประมวลผลโดยใช้กลไก XML ที่เป็นอันดับเดียวกับที่ขนถ่ายและการแบ่งปันแผนในส่วนของรูปแบบไฟล์นี้เหมือนกัน

ความหมายของการปล่อย

การประมวลผลช่วยให้ข้อมูลทั้งปล่อยเต็มและบางส่วนในไฟล์ฐานข้อมูลข้อมูลการตั้งค่าของข้อมูลที่อัปโหลดจะจัดขึ้นในการเจรจาโดยการตั้งธงในต้นไม้คอลัมน์ที่แสดงวัตถุเมข้อมูลซึ่งสามารถอัปโหลด ธงคอลัมน์เพิ่มเติม "ถ้าจำเป็น" กำหนดความจำเป็นในการขนวัตถุของ "ลิงก์" ประเภทนี้นั่นคือถ้าธงมีการตั้งค่าเฉพาะใน 'ถ้าจำเป็น' ข้อมูลใน vyvantazhuvatymutsya เรื่องนี้เต็ม แต่เพียงเท่าที่จำเป็นเพื่อรักษาความสมบูรณ์อ้างอิงในฐานข้อมูลที่จะดาวน์โหลดไฟล์ที่อัปโหลด

เมื่อคุณเปิดคุณสมบัติการประมวลผลแบบกำหนดยกเลิกการเชื่อมโยงวัตถุทั้งหมดที่รับประกันความสมบูรณ์ของการเชื่อมโยงฐานข้อมูลถอดชิ้นส่วน

คลิกที่ "ระบุเชื่อมโยงข้อมูลที่อัปโหลด" การประมวลผลการวิเคราะห์การเชื่อมโยงข้อมูลที่อาจจะอยู่ในสถานที่ซึ่งมีชุดเต็มของการปล่อยและอัตโนมัติเติมธงคอลัมน์ yndytsyruyut จำเป็นที่จะต้องปล่อยลิงค์ถ้าวัตถุที่มีอยู่แล้วมีกล่องกาปล่อยเต็มซึ่งในกรณีกล่องขนถ่ายเชื่อมโยงจะไม่เป็นที่ยอมรับ

การใช้งานที่เป็นไปได้

การใช้การรักษาแบบนี้เป็นไปได้เช่นการสร้างการสำรองข้อมูลทั้งหมดหรือบางส่วนของข้อมูลการแลกเปลี่ยนข้อมูลระหว่างฐานข้อมูลเช่นเดียวกับการเป็นเครื่องมือที่ช่วยในการกู้คืนฐานข้อมูลมีปัญหา
