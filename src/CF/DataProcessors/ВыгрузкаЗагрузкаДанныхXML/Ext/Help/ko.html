처리 "하역 및 적재 XML 데이터"

동작 모드

착륙 (사용자가 이러한 데이터의 생성 파일을 업로드) 및로드 (그것에서 사용할 수있는 데이터를 파일 생성 시조 모드를 읽기, 기록 언로드) : 처리는 두 가지 동작 모드를 구현합니다. 모드 필드를 선택하여 모드 설정.

이 또는 다른 모드 (실행 버튼을 누릅니다) 시작하기 전에, 당신은 파일 업로드의 이름을 지정해야 또는하여 "파일 이름"에 수동으로 입력하거나 버튼이 분야 표준 파일 선택 대화 상자를 선택 사용.

부팅 모드에서, 다운로드 속도에 영향을 미칠 수있는 엔트리의 레지스터의 사용의 결과를 수정하는 것이 가능하다.
플래그가 "데이터 다운로드 중 사용의 결과를 편집 할 수있는 기능을 Enable"로 설정하고 데이터를로드 사용의 결과 수동 제어에 사용하는 경우 버튼은 "결과를 사용 안 함"과 "결과를 포함"사용할 수 있습니다.

적용하는 처리 조건

처리는 데이터가 언로드되는 정보베이스, 및 데이터가로드 된 하나의 균질 한 (동일한 구성 데이터가 다를 수 있음)되는 경우에 사용될 수있다.어느 모든 개체는 구성과 유형 세부 사항 및 테이블 부품, 속성 "선도"개체 메타 데이터 등의 거의 동일한 배출이러한 제약들과 관련하여, 처리는 주로 균질 IB 사이의 교환 대상으로, 주목해야한다.

과거 계획 헤더 부에 언로드 때 생성 형식 파일로부터 다른 파일 포맷을 업로드.파일 형식이 부분에서 계획의 교환을위한 방전이 동일하다 같은 XML 직렬화 메커니즘을 사용하여 치료를 (등 요소 가이드, 레지스터 및 기록의 집합) 데이터를 다운로드합니다.

배출 조성의 결정

프로세싱은 파일에 데이터베이스 정보 전체 및 부분 언로드 가능하다.조성물 설정 데이터는 데이터가 다운로드 될 수있는 메타 데이터 오브젝트를 표시 나무 열에 플래그를 설정하여 대화 언로드된다. 추가 열 플래그는 "필요한 경우"이러한 유형의 "링크"의 객체를 언로드에 대한 필요성을 설정합니다.박스 만 체크하는 경우 즉,이 개체에 대한 데이터는 "필요한 경우"전체 배출 및되지 않습니다 만 파일을로드 할 정보베이스에서 참조 무​​결성을 유지하기 위해 필요한 범위에 업로드 할 수 있습니다.

당신이 열 때 양식 처리 정보 기반의 조각을 배출 참조 무​​결성을 보장합니다 모든 개체에 대한 링크를 업로드 할 플래그를 설정합니다.

버튼을 클릭하면, 처리 분석 데이터가 완전 방전의 표시를 설치 한 시설에 보관하실 수 있습니다 참조를 "링크를 업로드, 데이터 정의", 자동으로 업로드 할 필요성을 나타내는 플래그 열을 채 웁니다 링크.개체가 이미 완전 방전 박스 세트를 가지고 있다면,이 경우에도, 방전은 링크가 설정되어 있지 확인한다.

가능한 응용 프로그램

이 치료의 사용은, 예를 들어, 데이터베이스 사이의 데이터 교환의 전체 또는 부분 백업 데이터뿐 아니라 정보 데이터베이스 문제 감소 보조 도구를 만들 수있다.
