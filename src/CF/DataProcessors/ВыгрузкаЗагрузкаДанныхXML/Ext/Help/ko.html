처리 "하역 및 적재 XML"

작동 모드

착륙 (파일 업로드 사용자가 지정한 데이터를 생성) 및 다운로드 (파일 업로드, 정권에 의해 생성 된 같은 이름을 읽기, 사용 가능한 데이터에 쓰기) : 처리는 두 가지 동작 모드를 구현합니다. 설정 선택 모드에서 수행된다.

특정 모드 (실행 버튼을 누릅니다) 시작하기 전에, 당신은 파일 업로드의 이름을 지정해야 또는 "파일 이름"에 수동으로 입력하거나 선택하여 버튼 필드와 표준 파일 대화 상자를 사용하여.

부트에서 다운로드 속도에 영향을 미칠 수있는 편집 기록 레지스터의 결과의 이용 가능성이있다.
버튼 "결과를 비활성화"과 플래그 "데이터 로딩시 사용 결과를 편집 할 수있는 기능을 활성화"로 설정되어있는 부트의 결과의 이용 수동 운전 모드에 사용하는 경우 사용할 수있는 "결과를 포함" 데이터.

이용 약관 및 치료의 적용 조건

프로세싱은 토출 데이터 및 데이터가로드되어있는 하나 (구성 동일한 데이터가 다를 수 있음)를 균질되는 정보베이스, 또는 모든 방전 개체 조성물 및 유형 세부 거의 동일한 경우에 이용 될 수있다 표 부품, 속성 "호스트"오브젝트 메타 데이터 등과 이들의 제약들과 관련하여, 처리는 주로 균질 한 IB 사이의 교환에 이용되고 있음을 주목해야한다.

파일 형식은 계획 교환 chapeau 하역 과정에서 생성 하역 파일 형식과 다릅니다. 치료는 파일 형식의이 부분에서 교환을위한 방전 계획, 동일 동일한 XML 직렬화 메커니즘을 사용 (등 요소 디렉토리, 레지스터 세트의 기록) 데이터를 다운로드합니다.

방전의 결정

처리는 파일의 데이타베이스 정보 전체 및 부분 하역 할 수있다. 업로드 데이터의 조성을 조정은 데이터가 다운로드 될 수있는 메타 데이터 오브젝트를 표시 트리 열에 플래그를 설정하여 대화를 수행한다. 추가 열 플래그는 "필요한 경우"이러한 유형의 "링크"의 객체를 언로드 할 필요가 설정합니다. 박스 만 체크되어있는 경우 즉,이 개체에 대한 데이터는 "필요한 경우"전체 배출 될 수 없습니다 만 파일을 다운로드 할 정보 기반의 참조 무​​결성을 유지하기 위해 필요한 범위에 업로드 할 수 있습니다.

당신이 열 때 폼 처리는 정보 기반의 조각을 배출 참조 무​​결성을 보장하는 모든 개체를 업로드 할 수있는 링크 플래그를 설정합니다.

"링크 언로드 정의 데이터"를 클릭하여 처리 참조는 전체 방전의 부호를 가지며 여기에 자동으로 언로드 할 필요성을 나타내는 플래그 열을 채우고 개체에 포함될 수도있는, 분석한다. 개체가 이미 상자 전체 세트 방전이있는 경우, 하역 링크가 설정되지 않은 확인합니다.

가능한 응용 프로그램

이러한 처리를 이용하면, 예를 들면, 상기 정보 데이터베이스 간의 데이터 교환의 전체 또는 부분 백업 데이터뿐 아니라 정보 데이터베이스의 문제의 감소를 보조 도구를 만들 수있다.
