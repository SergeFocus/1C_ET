แถบเหตุการณ์เริ่มต้นเทคโนโลยีนิตยสารและระบบการ คุณกำหนดค่าพารามิเตอร์ของกระบวนการของการสร้างนิตยสารโดยเริ่มต้นที่

พิจารณาวิธีการทำงานร่วมกับบรรณาธิการของพารามิเตอร์เทคโนโลยีนิตยสาร

การตั้งค่าบรรณาธิการของนิตยสารเทคโนโลยี

กว่าบรรณาธิการ settles การตั้งค่ากลุ่มการตั้งค่าในกลุ่มนี้จะได้รับความจำเป็นในการที่จะดำเนินการนิตยสารเทคโนโลยีสถานที่และระยะเวลาของการจัดเก็บข้อมูลเช่นเดียวกับตัวเลือกที่สามารถทำให้กระบวนการแก้ไขได้ง่ายขึ้นกลุ่มของการล่มสลายหลังจากการตั้งค่าทั้งหมดจะทำเพื่อที่จะใช้พื้นที่น้อยบนหน้าจอ

ธงนิตยสารเทคโนโลยีการขึ้นรูป การควบคุมจะต้องสร้างระบบการลงทะเบียน "1C: เอ็นเตอร์ไพรส์"

สถานที่ตั้งนิตยสารเทคโนโลยีและระยะเวลาการเก็บรักษาของประวัติศาสตร์, นาฬิกา กำหนดตำแหน่งและระยะยาวว่ามันจะถูกเก็บไว้ที่ลงทะเบียน

อยู่ทางด้านขวาของการตั้งค่าการควบคุมเสริมที่ช่วยให้คุณที่จะทำให้กระบวนการแก้ไขได้ง่ายขึ้นสวิทช์กรุ๊ป (คุณสมบัติทั้งหมดคุณสมบัติที่เลือกเท่านั้นและคุณสมบัติทั้งหมดกิจกรรมที่เลือก) ตรวจสอบเหตุการณ์ที่จะปรากฏในการแก้ไข:

  • คุณสมบัติทั้งหมดกิจกรรมที่เลือก - โหมดแบบไดนามิกนี้จะแสดงคุณสมบัติเหล่านั้นเท่านั้นที่มีอยู่สำหรับกิจกรรมที่เลือก (ทำเครื่องหมายคอลัมน์แรกมาก)เพิ่มหรือลบกิจกรรม - องค์ประกอบมากมายที่แสดงคุณสมบัติสะท้อนละเว้นความจริงที่ว่าลำโพง nevidznachenymy อาจจะถามคุณสมบัติบางอย่าง
  • คุณสมบัติที่เลือกเท่านั้น - ในกรณีนี้จะแสดงคุณสมบัติเฉพาะผู้ที่มีการทำเครื่องหมายอย่างน้อยหนึ่งลงทะเบียนเหตุการณ์โหมดไม่ได้เป็นแบบไดนามิก
  • คุณสมบัติทั้งหมด - ในกรณีนี้ก็จะแสดงคุณสมบัติทั้งหมดของทุกเหตุการณ์ที่ลงทะเบียนโหมดไม่ได้เป็นแบบไดนามิก

หากคุณต้องการที่จะแสดงเหตุการณ์เหล่านั้นเท่านั้นที่อยู่ในทางใดทางหนึ่งได้รับการแต่งตั้ง (ทำเครื่องหมายคอลัมน์กรณีใด ๆ โดยเฉพาะอย่างยิ่ง) จากนั้นทำมันด้วยธงเฉพาะเหตุการณ์การทำเครื่องหมาย

สวิทช์แสดงรายละเอียดและชื่อที่แสดง คุณสามารถควบคุมคุณสมบัติวิธีการแสดงและกิจกรรมต่างๆในการแก้ไข

นอกจากนี้ที่ด้านล่างของรายละเอียดรูปแบบของ otobrazhaemtsya เหตุการณ์ปัจจุบัน (Groupเหตุการณ์ปัจจุบัน) และลักษณะปัจจุบัน (กลุ่มสถานที่ให้บริการในปัจจุบันID ของเหตุการณ์ปัจจุบันหรือคุณสมบัติที่แสดงในวงเล็บหลังข้อความ sootvestvuyuschehoตัวอย่างเช่นสำหรับเหตุการณ์ "SQL, Microsoft SQL Server" มุ่งหน้าไปทางกลุ่มจะมีลักษณะเช่นเหตุการณ์ปัจจุบัน (dbmssql)หากข้อมูลเสริมนี้ไม่จำเป็นต้องก็สามารถลบออกได้กลุ่มหันข้อมูลเพิ่มเติม

เซลล์ที่มีพื้นหลังเป็นสีขาวและกล่องวงกลมดำเนินการใด ๆ เกี่ยวกับพวกเขาเมื่อคุณคลิกเมาส์คลิกเมาส์บนมือถือทำให้เกิดเครื่องหมาย - สัญลักษณ์เกิด "วี"ซ้ำคลิกเอาเครื่องหมาย

ถ้าเซลล์คำอธิบายประกอบในคอลัมน์กิจกรรมทั้งหมด หรือใด ๆ ของคอลัมน์ไปทางขวาของคอลัมน์นี้ก็เป็นไปพร้อม ๆ กันนอกจากนี้ยังเป็นเครื่องหมายของเซลล์แรกในเหตุการณ์ปัจจุบัน (รวมอยู่ในการลงทะเบียนของเหตุการณ์)การกำจัดของเครื่องหมายไม่ได้มีคุณสมบัติดังกล่าว

ทำเครื่องหมายคุณสมบัติสำหรับเหตุการณ์ทั้งหมด (ซึ่งคุณสมบัตินี้ถูกกำหนด) จะดำเนินการโดยใช้เมนูบริบทรวมถึงเหตุการณ์ที่เกิดขึ้นทั้งหมดในการลบเครื่องหมายยังต้องพิมพ์เมนูทางลัดปิดสำหรับเหตุการณ์ที่เกิดขึ้นทั้งหมดยืนอยู่ในสายของลำโพงใด ๆ กับบริการที่เลือกไว้

หากคุณต้องการที่จะกำหนดเงื่อนไขที่จะมีเหตุการณ์ที่มันควรจะทำโดยใช้เมนูบริบทเงื่อนไขสำหรับการแข่งขัน ...ในขณะที่ในคอลัมน์ของเหตุการณ์ที่เลือกใด ๆหากเงื่อนไขที่มีการตั้งค่ามุมมอง pryobrete มาร์ค ""(มาร์คตัวหนา)ในการลบเงื่อนไขสำหรับเหตุการณ์ที่เกิดขึ้นควรลบกิจกรรมทั้งหมดในแง่ของการแก้ไขหรือใช้คำสั่งเมนูบริบทเหตุการณ์ล้างตัวกรองเครื่องหมายนี้จะยังคงอยู่ แต่จะสูญเสียการออกแบบ poluzhyrnoe

การตั้งค่าหน่วยความจำแผนแบบสอบถาม

สำหรับการเก็บรักษาของแผนแบบสอบถามในนิตยสารเทคโนโลยีตอบกล่องแผนติดตามคำขอ (& lt; planSQL & gt;) และทรัพย์สิน งบจัดกิจกรรม SQL ดำเนินการฐานข้อมูลต่างๆการติดตั้งธงเป็นครั้งแรกรวมถึงความเป็นไปได้ในการสร้างแผนแบบสอบถามและการรวมหรือยกเว้นของคุณสมบัติที่มีการจัดการโดยความพร้อมของข้อมูลในการสอบถามข้อมูลแผนในการจัดกิจกรรมที่เกี่ยวข้องหากแผนประหยัดรวมถึงการร้องขอก็จะแนะนำยังรวมถึงคุณสมบัติการบันทึก สำหรับกิจกรรมที่ดำเนินการคำสั่ง SQL ฐานข้อมูลต่างๆ
















ในตัวอย่างนี้ผลการดำเนินงานของผู้ประกอบการสำหรับเซิร์ฟเวอร์ฐานข้อมูล MSSQL เหตุการณ์ (เหตุการณ์) รวมถึงการแก้ไข SQL ภาษาแบบสอบถาม (ทรัพย์สิน) และแผนการดำเนินการแบบสอบถาม (ทรัพย์สินนอกจากนี้ยังรวมอยู่ในคอลเลกชันของแผนปฏิบัติการแบบสอบถาม (รายการ

ได้รับแผนแบบสอบถามช้าสืบค้นฐานข้อมูลสำหรับบางคนการชะลอตัวของฐานข้อมูลอาจจะมีนัยสำคัญ

การตั้งค่าหน่วยความจำรั่ว

สำหรับการควบคุมการรั่วไหลของหน่วยความจำที่ตรงกับการตั้งค่าสอง

  • ตรวจสอบการใช้หน่วยความจำเซิร์ฟเวอร์นี้ควรจะมีข้อสรุปหรือเข้าสู่ระบบของการพัฒนาเทคโนโลยีทั้งหมดหรือแยกกันรวมถึงเหตุการณ์การรั่วไหลของเซิร์ฟเวอร์
  • การรั่วไหลของการติดตามหน่วยความจำในการกำหนดค่ารหัสในกรณีนี้มันเป็นสิ่งที่จำเป็นเพื่อระบุพื้นที่ที่จำเป็นในการควบคุมการรั่วไหลของหน่วยความจำการกำหนดค่าซึ่งสามารถทำได้ด้วยธงโค้ดจากเริ่มต้น / สิ้นสุดเกณฑ์มาตรฐานบนไคลเอนต์โค้ดจากเริ่มต้น / สิ้นสุดเกณฑ์มาตรฐานบนเซิร์ฟเวอร์เช่นเดียวกับรายการที่จะป้อนข้อมูลเกี่ยวกับโมดูล kontrolliruemyhนี้ควรจะมีข้อสรุปหรือเข้าสู่ระบบของการพัฒนาเทคโนโลยีทั้งหมดหรือแยกกันรวมถึงเหตุการณ์หน่วยความจำรั่ว

การตั้งค่าการรักษาลงทะเบียนเริ่มต้น

เทคโนโลยีเริ่มต้นนิตยสาร - ชนิดพิเศษของนิตยสารเทคโนโลยีที่ออกแบบมาเพื่อบัน​​ทึกเหตุการณ์ที่เกิดขึ้นในสถานการณ์ที่สำคัญ (ในแง่ของระบบ "1C: บริษัท ")การทำเช่นนี้สร้างการแก้ไขเหตุการณ์เข้าสู่ระบบกรองซึ่งรูปแบบแพลตฟอร์มและที่ไม่สามารถเปลี่ยนแปลงได้

แต่ก็เป็นไปได้ที่จะตั้งขึ้นลงทะเบียนระบบสถานที่ตั้งและระยะเวลาการเก็บรักษาของประวัติศาสตร์, นาฬิกาตั้งค่าตัวเลือกการจัดเก็บข้อมูลการลงทะเบียนค่าเริ่มต้นเชื่อมโยงหลายมิติเริ่มต้นการตั้งค่าเริ่มต้นของการกำหนดค่าระบบลงทะเบียน
