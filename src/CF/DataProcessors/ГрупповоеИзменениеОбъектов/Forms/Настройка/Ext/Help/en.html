Additionallyon the first page of the Assistant You can specify custom settings with the commandCustomize settings changesThe system can handle objects in the transaction.If in the process of recording with the flagChange in the transactioncan not write to one of the objects (for example, because the object being edited by another user), the changes will be canceled in all objects.By default, this box is unchecked.

If the change objects are not in a transaction, it is recommended to run it with a check markInterrupt on errorIn this case, if the object record for any reason is not possible, the system will issue a warning and offer to continue to change the remaining objects or interrupt to determine the cause of the problem.If the box is not checked, and the object can not be recorded, the processing will skip such object and will continue to change the remaining objects.

TickingTreat the child elements (Default is not checked) lets you change the child elements of the selected groups and parent objects without the need to install flags for all elements of the groups and child objects.If unchecked, the system processes the group itself, and not its child elements.
