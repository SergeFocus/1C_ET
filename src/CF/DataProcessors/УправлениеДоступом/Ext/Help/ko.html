사용자 액세스 제어 시스템

시스템은 대화 형 고려 사업 환경을 고려하여,이 설정은 미묘한 수행 객체에 자신의 사용자 액세스 레벨을 정의하는 매니저를 제공하도록 설계된다.

기본적으로 사용자는 모든 권한을 갖습니다.

액세스의 다음 단계 :

  • 전체 액세스 (전체 무조건 액세스)
  • 조건없이 - 읽기 (전체 액세스,하지만 조건에 때 실패 조건 -. 데이터보기)
  • 액세스 제공 (전체 액세스 권한을하지만, 조건의 경우 조건 불이행 -. 실패)
  • 읽기 전용 (전용 뷰 데이터)
  • 주제를 읽기 (데이터 만과를 볼 경우에만 경우 조건 불이행 -. 실패)
  • 권한 없음 없습니다 (거부)

언어 1c에 별도의 필드에 채워 조건. 예를 들면: Obekt.Naimenovanie는 = "예"

또 다른 예 : Obekt.Otvetstvenny = ParametrySeansa.TekuschiyPolzovatel

다양한 객체의 사용자 인터페이스의 존재는 사용자의 "역"에 의해 결정된다. 그러나 이들 원소의 이용은 시스템에 의해 장착 될 수있다.
