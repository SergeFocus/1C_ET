User Access Control System

The system is designed to give managers to interactively define their user access levels to objects, perform this setting more subtle, taking into account the business environment.

By default, the user has all the rights.

The following levels of access:

  • Full Access (Full unconditional access)
  • Without the condition - reading (full access, but on the condition. When failure conditions - viewing data)
  • Access provided (full access, but on the condition. If non-compliance with conditions - Failure)
  • Read-only (view data only)
  • Reading the subject (only view the data and only if. If non-compliance with conditions - Failure)
  • No Access (refusal)

Conditions filled in a separate field in language 1C. For example: Obekt.Naimenovanie = "example"

Another example: Obekt.Otvetstvenny = ParametrySeansa.TekuschiyPolzovatel

The presence in the user interface of various objects is determined by the "role" of the user. But the availability of these elements can be mounted by the system.
