Обработка зеленый день

Данная обработка предназначена для накопления и сверки статистики работы пользователей (регистрации данных) в системе на базе платформы 1С 8.

Основная задача ее использования - мониторинг (в периоде) состояний некоторых "Показателей", которые описывают число свершившихся событий.

Удобно для случаев, когда многогранный процесс регистрации данных нужно отслеживать за период "одним взглядом" на предмет невведенных или неразнесенных оперативных данных.

Приведу пример работы с обработкой:

  1.  Открываем обработку и указываем период в шапке окна - период для сбора статистики. Например, интервал размером с день - прошлый понедельник. Это будет как бы "эталонный период". Такой, с которым будут сравниваться данные других периодов. 
  2.  Запускаем на закладке "Авто-Сбор и настройки показателей" кнопку "Проанализировать базу данных за указанный период для автоматического составления нового списка показателей".
  3.  Перед нами будет некоторый перечень показателей, полученных по Вашей базе данных автоматически. Далее откроем снова меняем период в шапке окна - теперь, как интервал для анализа, - на понедельник текущей недели и соглашаемся на принятие его в таблицу показателей.
  4.  Запустим основную процедуру обработки по кнопке "Выполнить анализ показателей".
  5.  На закладке "Результаты анализа" мы видим результаты сверки числа документов, записей регистров сведений, движений регистров накоплений в новом интервале времени. Если отклонения превысили норму, они становятся желтыми или красными). Процент установки для желтого и зеленого участка отклонений задается на закладке "Авто-Сбор и настройки показателей".
  6. Другой сособ изучения подходит для суточных показателей - выбрать несколько показателей в таблице и запустить "Ежедневный анализ каждого дня периода" для них. Затем на закладке "Результаты анализа" сформировать диаграмму (Alt-F3).

 

Конечно, будет эфективнее и интереснее создавать более сложные "показатели" вручную. Для этого нужно указать в колонке "Тип показателя" с помощью кнопки "открыть", из доступных в конфигурации типов документов, справочников, регистров сведений и накопления.

В колонке "Реквизит объекта" с помощью кнопки "открыть" можно указать реквизит данного типа. Например для документа - реквизит "Ответственный". Тогда в колонке "Значение реквизита объекта" можно указать значение из справочника "Пользователи" - "Иванов А.С.". В таком случае данный показатель будет использовать этот отбор.

Как Вы уже поняли, после ввода показателя можно проанализировать его в "зеленом" эталонном периоде и установить это значение в поле "100% Число объектов". Можно вывести отображение колонок в таблице показателей для задания собственного периода анализа.

Следует отметить, что отбор показателя в колонках "Начало периода изучения" и "Конец периода изучения" для РегистровСведений сработает только для периодических. А для показателей справочников, анализ в периоде выполнится только один раз и будет позиционирован для периода окончания анализа.

Не забывайте сохранять настройки обработки должным образом, когда подготовите свой перечень показателей!

Также предоставлена возможность сохранять и обмениваться списками показателей через файлы DBF. Правда при этом нет возможности передать значение реквизитов показателей.
