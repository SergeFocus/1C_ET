업로드하고 사용자 정보 데이터베이스의 다운로드

업로드 처리는 정보 기반의 사용자들의리스트를 다운로드하도록 구성된다.

눌러야 사용자를 언로드하려면 사용자를 업로드 .... 필요한 데이터의 형성 후, 시스템이 실행 언 로딩 할 파일을 지정하라는 메시지를 표시합니다. 파일의 기본 이름 : IBUsers.xml.

클릭해야합니다 사용자를 다운로드하려면 사용자가 다운로드 .... 당신은 (이전에 정보 기반에서 다운로드) 사용자의 목록이 들어있는 파일을 선택하라는 메시지가 다음 사용자 목록을 다운로드합니다. 파일의 기본 이름 : IBUsers.xml.

중요! 사용자 매핑 이름으로 이루어집니다.

파일의 데이터를 데이터베이스에서 사용자 정보를 업데이트 할 수있는 기회가 존재하는 경우, 파일과 같은 데이터베이스의 사용자 이름. 이렇게하려면 확인란을 동일한 사용자 이름, 사용자 업데이트의 호텔. 그렇지 않으면, 사용자 정보 데이터베이스에있는 동일한 이름으로 변경되지 않습니다.

부트 프로토콜을 얻고 싶은 경우에, 당신은 체크를 지정해야합니다 부트 프로토콜을 형성하도록 다운로드하기 전에. 이 경우, 각 사용자의 파일의로드 정보를 포함하는 텍스트 파일을 형성 할 것이다 :

  • 사용자 : 사용자 이름. 변경되지 않음. 파일의 사용자 정보 데이터베이스에서 발견. 그의 데이터가 변경되지 않습니다.
  • 사용자 : 사용자 이름. 파일에서 업데이트 된 데이터. 파일의 사용자 정보 데이터베이스에서 발견. 그 데이터는 파일의 데이터로 업데이트됩니다.
  • 사용자 : 사용자 이름. 새. 지정된 이름을 가지는 새로운 사용자 정보 데이터베이스.
