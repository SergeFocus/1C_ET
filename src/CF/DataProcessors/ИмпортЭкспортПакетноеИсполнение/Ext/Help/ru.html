Обработка импортно-экспортных операций

Пакетная обработка предназначена для импорта и экспорта одновременно карточек справочников/шапок документов и их табличных частей.

Обмен с ПО DBF - загрузка списком:

Описание параметров и функций и их его влияние на ведение учета на предприятии.

Так, с помощью флажка «Обновлять также существующие заказы» можно установить возможность обновления заказов в конфигурации, при внесении в них изменений в программе ПОDBF, например, при изменении требований заказчика к его заказу.

Функция «Регистрировать предоплаты приходными кассовыми ордерами» позволит производить регистрацию внесения предоплаты заказчиками формированием приходного кассового ордера.

«Пытаться провести документы». При установленном флажке появляется возможность автоматического проведения документов программой. При этом, проведение документов будет осуществляться только в случае заполнения всех необходимых данных в документе.
Рассмотрим функции в разделе «Данные заказчика».

Функция «Создавать заказчика при новом наименовании» позволяет программе автоматически создавать карточку контрагента. При этом, необходимо в поле «Группа новых контрагентов» указать папку, в которую будут заноситься все новые контрагенты. Следует также произвести настройку параметров карточки предопределенного «Невыясненного контрагента». Для этого, необходимо заполнить поля «Расчеты с покупателями», «Расчеты с поставщиком», «Авансы покупателя» и «Авансы поставщику» в разделе «Счета учета расчетов» карточки невыясненного покупателя. Данные настройки будут по умолчанию внесены программой в карточку нового контрагента.

Функция «Создавать договоры с заказчиком автоматически» позволяет программе при создании нового контрагента автоматически создавать с ним договор.

В разделе «Данные изделий» необходимо в поле «Группа новой номенклатуры изделий» указать папку, в которую будут заноситься все элементы справочника номенклатуры. При этом, следует также произвести настройку параметров карточки предопределенного «Неизвестного товара». По аналогии с заполнением программой настроек карточки нового контрагента из настроек внесенных в карточку предопределенного «Невыясненного контрагента», программа будет заполнять настройки карточки новой номенклатуры изделий согласно  настроек, введенных по умолчанию в предопределенную карточку «Неизвестный товар».

Функция данного раздела «Формировать наименование изделий с номером заказа впереди» позволяет программе указывать номер заказа в наименовании изделия, что достаточно актуально в позаказном производстве, при своеобразной «уникальности» каждого произведенного изделия.

Рассмотрим настройки, вносимые в разделе «Данные материалов»:

Установка флажка «Загрузить в номенклатуру материалы, участвующие в спецификации» позволит программе автоматически загружать в справочник номенклатуры материалы, внесенные при заполнении спецификаций изделий. В поле «Группа новой номенклатуры материалов» необходимо указать папку, в которую будут заноситься все новые элементы спецификаций изделий.
В данном разделе необходимо также заполнить параметры карточки предопределенного «Неизвестного материала», которые программа будет использовать при автоматическом заполнении полей новой номенклатуры материалов.

Установка флажка «Использовать артикулы контрагентов при синхронизации номенклатуры» позволит программе автоматически синхронизировать справочник номенклатуры, используя при этом артикулы контрагентов.

После внесения настроек можно приступать к загрузке данных. Для этого необходимо нажать кнопку «Загрузка всех данных списком». Следует учитывать, что загрузка производится только для элементов, которые упоминаются в новых заказах.

При установке флажка «Режим тестирования» данные загрузки не будут записаны в программу. Это позволяет практически проверить внесенные настройки и, при необходимости, внести дополнительные корректировки настроек.
