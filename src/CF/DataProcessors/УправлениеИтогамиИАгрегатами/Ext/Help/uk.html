Управління підсумками включає в себе процедури, пов'язані з підсумками регістрів накопичення і бухгалтерії, а також агрегатами оборотних регістрів накопичення.Управління підсумками розділяється на два режими:

  • Приватно використовувані можливості - Дозволяють "одним натисканням" виконати найбільш часті дії, що виникають при обслуговуванні системи.
  • Повні можливості - Дозволяє виконувати всі операції як з одним регістром, так і з довільним списком регістрів.

Для перемикання режимів служить гіперпосилання в правому нижньому куті форми.Форма запам'ятовує режим відкриття.

Часто використовувані можливості

Операції з підсумками

  • Встановити період розрахованих підсумків - Для прискорення роботи системи виконує установку періоду розрахованих підсумків для всіх регістрів накопичення і бухгалтерії з включеними підсумками.Для регістрів накопичення - на кінець попереднього місяця, для регістрів бухгалтерії - на кінець поточного місяця.
  • Включити використання підсумків - Включає використання підсумків для всіх регістрів, у яких цей режим вимкнений (наприклад в результаті аварійного завершення будь-якої процедури, що відключає використання підсумків).

Операції з агрегатами

  • Перебудувати і заповнити - Для всіх оборотних регістрів накопичення, для яких включений режим агрегатів і встановлено їх використання, виконує операцію перестроювання використання агрегатів і виконує заповнення агрегатів.Операція може виконуватися тривалий час.
  • Отримати оптимальні агрегати - Виконує отримання списку оптимальних агрегатів для всіх оборотних регістрів накопичення, для яких в Конфігураторі задані агрегати.Буде запитаний каталог, куди будуть збережені xml-файли зі списками оптимальних агрегатів.Кожен файл буде названий ідентифікатором відповідного регістра.Так, для регістраПродажі файл буде називатисяПродажі.xml

Повні можливості

У режимі повних можливостей доступний режим множинного виділення.Будь-яка дія застосовується до всіх виділених об'єктах, або до того, на якому встановлений курсор.

Підсумки

На даній закладці можна виконувати наступні операції:

  • вмикання і вимикання використання підсумків,
  • вмикання і вимикання використання поточних підсумків,
  • вмикання і вимикання поділу підсумків,
  • установка обраного періоду розрахованих підсумків,
  • виконати перерахунок підсумків, поточних підсумків або виконати перерахунок підсумків за вибраний період.

У список потрапляють тільки ті регістри, у яких для поточного користувача встановлено правоУправління підсумками

Агрегати

На даній закладці можна виконувати наступні операції:

  • установка режиму використання регістра: підсумки або агрегати,
  • вмикання і вимикання використання агрегатів,
  • виконати перестроювання використання агрегатів,
  • оновити операційну агрегатів,
  • виконати очистку агрегатів,
  • виконати розрахунок списку оптимальних агрегатів.Буде запитаний каталог, куди будуть збережені xml-файли зі списками оптимальних агрегатів.Кожен файл буде названий ідентифікатором відповідного регістра.Так, для регістраПродажі файл буде називатисяПродажі.xmlПісля виконання операції, в списку регістрів жирним шрифтом будуть виділені ті регістри, для яких необхідно оновити список агрегатів на розраховані оптимальні агрегати.Оновлення необхідно виконувати в Конфігураторі.

У нижній частині форми відображається список агрегатів регістра, на якому встановлений курсор у верхньому списку.

У список потрапляють тільки ті регістри оборотні регістри накопичення, у яких для поточного користувача встановлено правоУправління підсумками і в Конфігураторі задані агрегати.
