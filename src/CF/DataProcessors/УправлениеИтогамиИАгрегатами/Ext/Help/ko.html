자주 사용하는 기능

결과 운영

  • 의 계산 된 결과 사이에 - 시스템의 속도를하려면 포함 된 결과에 축적 및 회계 레지스터의 결과에 대해 계산 기간을 설치합니다. 스토리지를 등록 - 회계 레지스터 이전 달의 끝에 - 월말.
  • 결과를 사용할 수있게 - 모드가 꺼져있는 모든 레지스터의 결과의 이용 참여합니다 (모든 절차의 충돌의 결과로서 예를 들어, 결과를 사용할 수 없도록).

작업 단위

  • 다시 채운다 - 사용을 가능 유닛을 발견하고 순환 레지스터의 모든 축적, 기기를 이용하여 다시 수행하고, 상기 충전 장치를 행한다. 동작이 장기간 수행 될 수있다.
  • 최고의 단위를 얻기 - 구성 관리자 단위로 지정 순환 레지스터의 축적에 가장 적합한 단위의 목록을 검색합니다. 당신은 최고의 단위의 목록과 XML-파일이 저장 될 디렉토리를 묻는 메시지가 표시됩니다. 각 파일은 레지스터를 해당 식별자 지정됩니다. 그래서, 등록 판매 파일 이름이 지정됩니다 Prodazhi.xml.
