업로드와 데이터를 다운로드합니다. 유사한 프로그램을 공유

이 처리에서, 유사한 구성의 데이터베이스와 데이터를 교환하는 방법 :

기간에 대한 데이터를 교환하는 네 가지 방법이 있습니다 :

1. 업로드 - 구현에 데이터를 다운로드. 이 이동 데이터에 데이터 구현 생성물 별도의에 유효성 또는 자기와 관련하여 별도의 콘센트에. 그들은 단지 기간 동안 지출, 복구 및 상각 제품에 대한 데이터를 포함한다. 돈 상태웨어 하우스와 타협이 선점되지 않습니다. 선택적으로 숨기고 매출원가 수 있습니다. 과거의 이러한 종류의 독립적 인 뷰 (이익)과 함께 구현에서 데이터를 검색 할 필요가있다. 이러한 데이터는 구현 그것에 전송 된 상품의 잔당을 추적하기 위해 필요하다. 당신은 패키지 형성 문서 유형을로드 할 때 비용 구현하고 제품의 오프를 작성, 영수증 상품의 반품의 경우. 부팅되면 레코드를 추적 할 필요가없는 경우에는, 최대 하나의 문서의 기간 동안 모든 트래픽을 축소 할 계약. 다른 부팅 구성 "새 문서의 매개 변수", "설정 명칭", "새로운 고객의 매개 변수가 있습니다."

2. 업로드 - 머리에 데이터를 다운로드. 이 데이터베이스. 부하 모든 데이터와 실제 실적과 수익성을 계산하기 위해 (예 : 영수증의 가격이) 회계 데이터를 완료 - 이러한 교환은 머리가 판매 시점의 전망을보고 경우에 필요하다. 로드 할 때 이러한 패키지는 기간 동안 모든 문서 그들에 나와있는 디렉토리의 요소의 사본에 나타납니다. 이 번호와 문서가 이미있는 경우 - 그들은 무시됩니다. 프로그램 시작 prveryaet 문서 디렉토리 항목의 번호 / 날짜와 이름. 명명법 뿐만 아니라 메인 의해 바코드 및 제품. 상대방 INN / 옥포에 의해. 그것은 프로그램 관리자의 디렉토리 엔트리 카드의 데이터를 변경하고 만 보충하지 않는 것이 좋습니다.

3. 언로드 - 제품, 가격, 돈 균형, 타협의 지표의 부하 균형-이 가능성이 없거나 이전의 모든 레코드를 전송해야하지만,이 때, 프로그램의 다른 복사본에 데이터를 한 번 전송에 필요한 특정 날짜에 대한 데이터 만이 아니라. 로드 할 때 이러한 패키지는 문서를 형성한다 보정.

4. 업로드 - 프로그램 설정을 다운로드- 모두 교환 할 설정 프로그램의 다른 사본.

선택 데이터 창고.명명법 그룹.제조사 과 지방 필터 하역 및 적재 데이터 동안 적용.

이러한 치료 외에도, 거기 수출입 서브 시스템 다른 데이터베이스와 매개 변수 교환을위한 파일을 통해 데이터,.
