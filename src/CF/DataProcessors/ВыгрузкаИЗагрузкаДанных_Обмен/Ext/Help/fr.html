Charger et télécharger des données. Partage d'un programme similaire

Dans ce traitement, des procédés pour échanger des données avec une base de données de configuration similaire:

Il ya quatre façons d'échanger des données pour la période:

1. L'ajout - Téléchargement des données sur la mise en œuvre. Les données sur la mise en œuvre de ces données de mouvement produit sur un séparé Disponibilité ou dans une sortie séparée avec l'auto-sujet. Ils ne comprennent que des données sur les dépenses, la récupération et les produits de radiation pour la période. L'argent de l'entrepôt et des établissements Etat ne sont pas préemptées. En option, vous pouvez masquer et le coût des marchandises vendues. Ce type d'échange est nécessaire pour récupérer les données de l'exécutant avec une vue indépendante de (sur le revenu). Ces données sont nécessaires pour suivre les restes des produits transférés aux fins de la mise en œuvre. Lorsque vous chargez un tel emballage formé type de document frais à mettre en œuvre et la radiation des biens, recettes cas pour le retour de la marchandise. Lorsque vous démarrez, vous pouvez réduire l'ensemble du trafic pendant une période maximale d'un document, si il n'y a pas besoin de garder une trace des dossiers entrepreneurs. Il ya une autre configuration de démarrage "Paramètres de nouveaux documents», «Nomenclature Paramètres", "Paramètres de la nouvelle clientèle".

2. L'ajout - Téléchargement de données vers la tête. Cette base de données. Un tel échange est nécessaire dans les cas où la tête regarde vue de point de vente - des charges de toutes les données et complète ses données comptables (tels que les prix de la réception) pour calculer les revenus réels et de la rentabilité. Lorsque vous chargez un tel paquet apparaîtra dans les copies de tous les documents pour la période et les éléments des répertoires qui sont mentionnés dans eux. Si des documents avec ce nombre ont déjà - ils seront ignorés. Prveryaet programme de démarrage Documentation par numéro / date et le nom de l'entrée de répertoire. La nomenclature ainsi par le principal code à barre et l'article. Contreparties par DCI / Okpo. Il est recommandé de ne pas modifier les données dans les cartes d'entrée de répertoire dans le gestionnaire de programme, et seul supplément.

Décharger - soldes de chargement de marchandises, les prix, les encaisses monétaires, les indicateurs de colonies- Il est nécessaire pour le transfert ponctuel de données à une autre copie du programme, quand il n'y a pas de possibilité ou le besoin de transférer tous les records précédents, mais seulement les données sur une certaine date. Lorsque vous chargez un tel paquet est formé de documents correction.
