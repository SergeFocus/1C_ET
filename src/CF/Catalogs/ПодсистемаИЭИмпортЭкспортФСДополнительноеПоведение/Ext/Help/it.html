Directory "regole supplementari di condotta per i modelli di importazione ed esportazione"

Utilizzare le regole di condotta - Consente di organizzare le regole dure-Aware per riempire un'importazione a seconda dei dati, o eseguire qualsiasi codice fornito.

Regole di comportamento indicato nella tabella in ordine e memorizzato in una directory separata. Essi possono essere utilizzati in qualsiasi numero di modelli.

La scheda è necessario specificare le regole di condotta Props Nome (Per le importazioni) o il nome del campo / colonna (per l'esportazione) Condizioni comportamento (importazioni) e valore.

Nelle tabelle di questo - il comportamento effettivo della lista di importare o esportare condizioni.

Ecco un esempio - suppongo che è necessario che quando si importano i dati nel caso in cui il valore di oggetti di scena "Commento" è la parola "altro" set di oggetti di scena "Articolo" a "altre spese" della directory articolo flusso di cassa.

Poi props hat "comportamento" Nome - "Commento", la condizione - "Contiene", che significa - "" altro "" e nelle tabelle delle dimensioni: Questo puntelli tabella - "no" nome - "articolo", Condition - è compilato, Valore - "Altri costi" del articolo directory flusso di cassa.

Quando si esporta una condizione è soddisfatta nella direzione opposta - a condizione che tutte le condizioni della sezione tabella, il campo (dal tappo) prende il valore (del tappo).
