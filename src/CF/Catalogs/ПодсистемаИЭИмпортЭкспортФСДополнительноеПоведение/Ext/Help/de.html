Directory "Zusätzliche Verhaltensregeln für den Import und Export-Vorlagen"

Verwenden Sie die Verhaltensregeln - Hier können Sie die Fest-Aware Rules zu organisieren, um einen Import abhängig von den Daten zu füllen, oder beliebigen Code vorgesehen durchzuführen.

Verhaltensregeln in der Tabelle, um angezeigt und in einem separaten Verzeichnis gespeichert. Sie können in jeder Anzahl von Mustern verwendet werden.

Die Karte ist notwendig, um die Verhaltensregeln festlegen Name Requisiten (Bei der Einfuhr) oder den Namen des Feldes / Spalte (für den Export) Bedingung Verhalten (Importe) und Wert.

In den Tabellen dieser - das tatsächliche Verhalten der Liste importieren oder exportieren Bedingungen.

Hier ist ein Beispiel - nehme an, es ist notwendig, dass, wenn Sie Daten für den Fall zu importieren, dass der Wert von Requisiten "Kommentar" ist das Wort "andere" gesetzt Requisiten "Artikel" auf "Sonstige Aufwendungen" der Artikel Verzeichnisse Cashflow.

Dann hat "Verhalten" Vorname Requisiten - "Kommentar", der Zustand - "Enthält", also - "" Andere "" und in den Maßtabellen: Diese Tabelle Requisiten - "nein" Namen - "Artikel", Zustand - nicht gefüllt, Wert - "Sonstige Aufwendungen" der Artikel Verzeichnisse Cashflow.

Beim Export eine Bedingung in die entgegengesetzte Richtung zufrieden - sofern alle Bedingungen der Tabellenbereich, der Bereich (von der Kappe) den Wert (der Kappe).
