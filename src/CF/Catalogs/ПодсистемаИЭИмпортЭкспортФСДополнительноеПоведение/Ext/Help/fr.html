Répertoire "des règles supplémentaires de conduite pour les modèles d'importation et d'exportation"

Utilisez les règles de conduite - Vous permet d'organiser les règles dures-Aware pour remplir une importation en fonction des données, ou effectuez une code fourni.

Règles de conduite indiqué dans le tableau dans l'ordre et stocké dans un répertoire distinct. Ils peuvent être utilisés dans ne importe quel nombre de motifs.

La carte est nécessaire de préciser les règles de conduite Nom accessoires (Pour les importations) ou le nom du champ / colonne (à l'exportation) conditionnent le comportement (importations) et valeur.

Dans les tableaux du présent - le comportement réel de la liste pour importer ou d'exporter conditions.

Voici un exemple - suppose qu'il est nécessaire que lorsque vous importez des données dans le cas où la valeur des accessoires "Commentaire" est le mot "autre" mettre des accessoires "article" à "Autres charges" du flux de trésorerie des répertoires de l'article.

Alors chapeau "comportement" Nom accessoires - "Commentaire", la condition - «Contient», ce qui signifie - "" autre "" et dans les tables de dimension: Cette accessoires de table - «non» le nom - "article", Condition - pas rempli, valeur - «Autres charges» du flux de trésorerie des répertoires de l'article.

Lorsque vous exportez une condition est satisfaite dans la direction opposée - à condition que toutes les conditions de la section de tableau, le champ (de la PAC) prend la valeur (de la PAC).
