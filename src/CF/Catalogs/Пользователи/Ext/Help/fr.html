Répertoire des membres

Référence ne sont pas disponibles pour l'édition.Membres et leur rôle doivent être définies dans le 1C de mode Designer.

Si la base de données sur le lieu de travail bénéficiera de quelques employés, il est pas superflu de déterminer leur rôle.Pour travailler avec cette configuration, les utilisateurs, leurs mots de passe et leurs rôles sont définis en mode configurateur 1C

Soyez sûr d'être un utilisateur avec le rôle "Pleins droits"Pour régler le programme.Installez-vous que le mot de passe pour cet utilisateur.Tous les autres réglages de base du programme sont faites en son nom.

Le rôle de "Exécution"Ce rôle avec des droits limités pour l'enregistrement des ventes.
Le rôle de "La mise en œuvre et l'acceptation des produits, la fixation des prix et des taux"Il a un des droits plus étendus.Un tel utilisateur établit l'achat, réglemente les prix.
Le rôle de "Trésorier"Pour les transactions avec de l'argent et des salaires.
Le rôle de "Opérateur d'équipement"- Pour les opérateurs à la configuration du site (d'assemblage).
Le rôle de "Bureaux de l'opérateur"- Commande la caisse.

Le rôle de "Enterprise client"- Pour les utilisateurs de représentants finissent clients de l'entreprise.
Le rôle de "Enterprise Fournisseur"- Pour les utilisateurs de représentants des fournisseurs de biens et services pour l'entreprise.

Le rôle de "Administrateur du système"- Pour l'administrateur système (l'administrateur de base de données pour l'entreprise d'effectuer des travaux de maintenance avec la base de données.
Le rôle de "Opérateur de fonction"- Pour les utilisateurs ayant une fonction spécifique, par exemple avec une interface spécifiée par un traitement de connexion supplémentaire.
