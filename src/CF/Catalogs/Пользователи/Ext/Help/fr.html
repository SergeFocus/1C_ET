Répertoire des membres

Référence ne sont pas disponibles pour l'édition. Membres et leur rôle devrait être défini dans le 1C de mode Designer.

Si la base de données sur le lieu de travail bénéficiera certains employés, il est pas superflu de déterminer leur rôle. Pour travailler avec cette configuration d'utilisateurs, leurs mots de passe et leurs rôles sont définis dans le mode Configurateur 1C:

Doit être un utilisateur avec le rôle "Pleins droits"Pour ajuster le programme. Installez mot de passe requis pour cet utilisateur. Tous les autres paramètres de base du programme sont faites en son nom.

Le rôle de "La concrétisation"Il est un rôle avec des droits limités pour les ventes d'enregistrements.
Le rôle de "La mise en œuvre et l'acceptation des produits, la fixation des prix et les taux"A un plus étendue des droits. Un tel utilisateur dessine l'acquisition, réglemente les prix.
Le rôle de "Trésorier"Les opérations avec l'argent et les salaires.
Le rôle de "L'assemblage de l'opérateur"- Pour les sites sur l'ensemble (ensemble).
Le rôle de "Les bureaux de l'opérateur"- Pour l'opérateur des caisses.

Le rôle de "Société client"- Pour les utilisateurs des représentants de l'entreprise réelle client final.
Le rôle de "Entreprises fournisseurs"- Pour les utilisateurs des représentants des fournisseurs de biens et services pour l'entreprise.

Le rôle de "Administrateur du système"- Pour l'administrateur système (entreprise DBA pour effectuer la maintenance de routine sur la base de données.
Le rôle de "Fonction de l'opérateur"- Pour les utilisateurs ayant une fonction spécifique, comme avec une interface spécifiée par un traitement de connexion supplémentaire.

Dans la configuration prévue et le rôle de "View Only».
(Uniquement disponible en Raleigh la version complète *).

NE DEMANDEZ PAS rôle d'utilisateur supplémentaire!
Les rôles "Administrateur système, les bureaux de l'opérateur de fonction de l'opérateur, les clients de l'entreprise, les sociétés de fournisseur»- Peut significativement déterminer l'interface utilisateur.
Typiquement, ces rôles sont définis dans l'une et non cumulable avec d'autres rôles.
