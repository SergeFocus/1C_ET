Справочник дополнительных правил поведения для шаблонов импорта-экспорта

 

Использовать правила поведения - позволяет организовать работу сложно-зависимых правил, чтобы наполнить объект импорта в зависимости от данных или выполнить какой либо программный код при условии.

 

Правила поведения указываются в таблице по порядку и хранятся в отдельном справочнике. Они могут быть использованы в любом количестве шаблонов.

В карточке правила поведения нужно указать Имя реквизита (для импорта) или имя поля/колонки (для экспорта), условие поведение (импорт) и значение.

В табличной части - собственно перечислить поведение для импорта или условия для экспорта.

Приведу такой пример, - предположим, необходимо чтобы при импорте данных в случае если в значении реквизита "Комментарий" встречается слово "прочий" установить значение реквизита "Статья" в значение "Прочие расходы" из справочника Статьи движения денежных средств.

Тогда в шапке "поведения" Имя реквизита - "Комментарий", условие - "Содержит", значение - ""прочий"", а в табличной части: Это реквизит таблицы - "нет", Имя - "Статья", Условие - не заполнено, Значение - "Прочие расходы" из справочника Статьи движения денежных средств.

При экспорте условие выполняется в обратную сторону - если соблюдены все условия из табличной части, то поле (из шапки) принимает значение (из шапки).
