Directory "regole supplementari di condotta per i modelli di importazione e di esportazione"

Utilizzare le regole di condotta - Consente di organizzare le regole dure-Aware per riempire l'importazione dell'oggetto, a seconda dei dati, o eseguire qualsiasi codice fornito.

Regole di comportamento sono elencati nella tabella in ordine e memorizzato in una directory separata. Essi possono essere utilizzati in qualsiasi numero di modelli.

La carta è necessario precisare le regole di condotta Puntelli Nome (per le importazioni) o il nome del campo / colonna (per l'esportazione) condizione di comportamento (importazioni) e valore.

La parte tabellare - il comportamento effettivo della lista per importare o esportare condizioni.

Ecco un esempio - supponiamo è necessario quando si importano i dati se il valore dei puntelli nel "Commento" la parola "altro" set di oggetti di scena "Articolo" alla voce "Altri oneri" della directory articolo flusso di cassa.

Poi cappello "comportamento" dettagli Nome - "commento", la condizione - "Contiene", che significa - "" altro "" e nelle tabelle di questo: props il tavolo - "no" nome - "Articolo", condizione - non pieno, il valore - "Altri costi" di articolo directory dei flussi di cassa.

Quando si esporta la condizione corre nella direzione opposta - a condizione che tutte le condizioni della sezione tabellare, il campo (dal tappo) ha il valore (del tappo).
