Manuale per ulteriori regole di comportamento modelli di import-export

Utilizzare Regole di comportamento - Consente di organizzare le regole dure-dipendenti per riempire servizio di importazione a seconda dei dati o eseguire alcun codice software fornito.

Regole di condotta indicato nella tabella girano tutti e memorizzato in una directory separata.Essi possono essere utilizzati in qualsiasi numero di modelli.

La scheda deve specificare le regole di comportamentoNome oggetti di scena (Per le importazioni) o campi del nome / colonne (per l'esportazione)comportamento condizioni(Importazione) evalore

Nella parte tabellare - il comportamento effettivo della lista di importare o esportare le condizioni.

Ecco un esempio - diciamo che è necessario che l'importazione dei dati nel caso in cui gli oggetti di scena significato "Commento" la parola "un altro" set di oggetti di scena "Articolo" nel senso di "Altri costi" dal flusso di cassa di directory articoli .

Poi il cappello "comportamento" Nome oggetti di scena - "Commento" condizione - la "città", che significa - "" altro "" e la parte tabellare: E 'puntelli tavolo - "no" nome - "articolo" Condizioni - bianco, il che significa - "Altri costi" del flusso di cassa di directory articoli.

Quando si esporta la condizione corre nella direzione opposta - se tutte le condizioni della parte tabellare, il diritto (con tappo) assume il valore (con tappi).
