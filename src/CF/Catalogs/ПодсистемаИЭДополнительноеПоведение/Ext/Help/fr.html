Répertoire "des règles de conduite supplémentaires pour les modèles d'importation et d'exportation"

Utilisez les règles de conduite - Il vous permet d'organiser les règles dures-Aware pour remplir l'importation de l'objet, selon les données, ou effectuez une code fourni.

Règles de comportement sont énumérés dans le tableau dans l'ordre et stocké dans un répertoire distinct. Ils peuvent être utilisés dans n'importe quel nombre de motifs.

La carte est nécessaire de préciser les règles de conduite Nom accessoires (pour les importations) ou le nom du champ / colonne (pour l'exportation) conditionnent le comportement (importations) et valeur.

La partie de tableau - le comportement réel de la liste d'importer ou de conditions exportation.

Voici un exemple - supposons que vous devez lors de l'importation des données si la valeur des accessoires dans le «Commentaire», le mot «autre» mis d'accessoires «article» à «Autres charges» de l'article répertoires flux de trésorerie.

Alors chapeau "comportement" Nom de détails - "Commentaire", la condition - "contient", ce qui signifie - "" autre "" et dans les tableaux de cette: Il props la table - "non" nom - "article", Condition - pas rempli, la valeur - «Autres charges» de l'article répertoires flux de trésorerie.

Lors de l'exportation de l'état va dans la direction opposée - à condition que toutes les conditions de la section de tableau, le champ (de la PAC) a la valeur (de la PAC).
