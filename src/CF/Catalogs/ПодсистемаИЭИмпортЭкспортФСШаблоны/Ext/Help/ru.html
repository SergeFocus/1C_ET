Справочник "Шаблоны файлов"

 

Шаблоны файлов импортно-экспортных операций доступны также в разделе "Настройка и администрирования" в меню слева. В папке шаблонов "Предопределенные" хранятся шаблоны, установленные программистом.

Изменения в эти шаблоны могут вноситься пользователями только с разрешения программиста системы, так как поведение программы при импорте и экспорте для них нестандартное.

 

Из окна списка шаблонов доступны уникальные функции системы:

 

Список команд И-Э (Импорт-экспорт) после включения системы появляется в списках и карточках всех стандартных справочников и документов системы. Данный перечень команд позволяет запустить выгрузку или загрузку для списка элементов или одного элемента базы данных. Если подходящий шаблон только один, он будет выбран системой автоматически.

 

Но в списке справочника "Шаблоны файлов импорта-экспорта" в списке команд И-Э доступны дополнительные:

Предопределенные операции импорта-экспорта - для запуска обмена по уникальным алгоритмам, заложенным разработчиками системы.

Работа с такими задачами требует персонального обучения и не описана в рамках данной справки.

 

Пакетный обмен для набора файлов - когда необходимо загрузить или выгрузить объекты базы данных, используя одновременно два и более файлов.

Например, один для шапки документа, а два других - для табличных частей документа.

В данной обработке необходимо указать Тип объекта обмена ИЛИ собственно объект обмена, Указать шаблон файла импорта-экспорта для шапки объекта, адрес файла для него. Можно указать период (если речь идет о документах).

Затем заполнить таблицу, указав ИмяТаблицы объекта, каким оно задано в конфигураторе, шаблон файла описания данной таблицы, соответствующий адрес файла.

 

Список задач обмена - для запуска импортно-экспортных операций списком в определенной последовательности.

В данной обработке можно заполнить таблицу списка набором из шаблонов файлов и их адресов, указать направление "Импорт - Да, Экспорт - Нет".

Указать Тип объекта обмена и его ИмяТаблицы, если необходимо. А потом запустить обмен списком операций. Для операций над документами можно указать период отбора.

 

Обработка Обмен документами денежных средств (так же из раздела "Денежные средства") предоставляет возможность выполнить список операций по импорту (Ctrl+I) и экспорту (Ctrl+E) для перечня задач. При открытии обработка подбирает в список подходящие шаблоны (для обмена документами денежных средств).

 

Другие команды списка шаблонов файлов:

 

Журнал импортно-экспортных операций. В этот журнал заносится информация о произведенных в системе импортных и экспортных операциях, если для них в шаблоне файла указано "Вести журнал".

Например, для всех импортных операций с флагом "Загружать все файлы из папки" ведется журнал, чтобы исключить повторную загрузку файлов.

 

Загрузить шаблоны/Сохранить шаблоны. Данные операции позволяют обменяться шаблонами файлов импорта-экспорта с другими пользователями системы или разработчиками системы.

Файлы описания сохраняются в специальном формате "ieUNFSH" и при импорте синхронизируются по наименованию шаблона. Не все опции шаблона можно перенести за счет загрузки шаблонов. Те параметры, которые индивидуальны для базы данных, необходимо будет указать вручную.

 

Карточка шаблона файла импорта-экспорта.

Это самый сложный объект системы. Для удобства его следует заполнять сверху-вниз справа налево.

Например, галочки в шапке "только для экспорта" или "только для импорта" помогут спрятать параметры шаблона, которые не будут использованы в данном случае.

 

Вид шаблона определяет основную задачу данного шаблона, кроме параметра "Любой вариант". Советуем выбрать вид шаблона, максимально отвечающий Вашей задаче.

В связи с выбором того или иного вида шаблона станут доступны те или иные параметры шаблона файла. Каждый параметр описан ниже.

 

А виды шаблона могут быть следующими:

    • Для карточки справочника или документа - шаблон описывает сам объект или какую либо таблицу для элемента базы данных справочника или документа.

Такой шаблон позволит соответственно загрузить или выгрузить объект или его таблицу. Одна строчка такого файла импорта-экспорта - один реквизит шапки или одна строка таблицы.

    • Для списка элементов справочника или документов - такой шаблон описывает поля из расчета одна строчка файла шаблона - один элемент справочника или один документ.

На закладке "Настройки шаблона" появится блок параметров "Правила для работы со списками".

    • Для таблицы Товары - для табличных частей "Товары" документов. При импорте данных такого шаблона, программа попытается дозаполнить данные строк для товаров.

На закладке "Настройки шаблона" появятся специальные настройки. Например, если будет установлен флаг "создавать новые объекты, то будет виден блок параметров значений по умолчанию для новой номенклатуры "правила для работы с запасами".

    • Для обмена документами денежных средств - специальный вид шаблона для реализации поведения (заполнения при импорте) для документов денежных средств в т.ч. для обмена с системами Банк-Клиент.

Блок "Условия операций с документами денежных средств" позволяет описать условия обмена подробнее.

    • Результат произвольного запроса - вид шаблона, позволяющий организовать экспорт результатов произвольного запроса на языке 1С 8.

Для данного вида запроса на закладке "Настройки шаблона" можно указать текст произвольного запроса, написанного на языке 1С 8, а так же задать значения параметров запроса в таблице ниже.

При этом, на закладке "Последовательность полей" в "Имя поля" следует указывать имена полей - результатов запроса.

Кнопки "Загрузить параметры из текста запроса", "Создать поля файла по результатам запроса", "Проверить запрос" позволяют наполнить таблицы полей и параметров автоматически, проверить работоспособность запроса. Кнопка "Выполнить выгрузку" производит выгрузку данных прямо из карточки шаблона.

 

Имя таблицы - параметр для указания таблицы для справочника или документа (такой, как она названа в конфигураторе), которая и будет объектом импортно-экспортной операции.

 

Далее описаны параметры шаблона, которые встречаются на закладках его карточки в зависимости от его вида:

Закладка "Параметры файла" служит для указания параметров файла(ов) импорта или экспорта.

Загружать все файлы из папки - указывает на то, что это импорт для массива из файлов. Ниже тогда следует указать каталог, а маска файла определит отбор подходящих файлов.

Указывайте маску *.* для загрузки всех файлов. Если в журнале импортно-экспортных операций уже есть запись о такой файле - он будет пропущен.

 

Искать в подкаталогах - для загрузки будут выбраны файлы по маске в каталоге и его подкаталогах.

 

Строгий адрес файла - можно указать файл для импорта/экспорта сразу. Иначе, перед операцией импорта или экспорта будет вызываться диалог выбора файла.

 

Файл в архиве - упаковывать данный файл после экспорта / обрабатывать все файлы данного архива при импорте;

 

Маска файла - можно организовать загрузку или выгрузку в файл, имя которого сформировано по специальной маске с учетом даты.

Так, например текст #гггг# в маске будет заменен на 4 символа текущего года, а ##-##-#### превратится в 31-03-2011 (если завтра первое апреля).

 

Вести журнал импортно-экспортных операций - будет ли записываться (в случае успеха) информация о каждой операции с этим шаблоном в реестр.

 

Удалять файл после удачного импорта.

 

Формат файла. На данный момент система поддерживает обмен для форматов файла: текстовый, файл MS Excel, DBase.

Для использования формата MS Excel в системе (где расположен сервер 1С, если работа  ведется через него) должен быть установлен программный пакет MS Excel.

 

Конкретная кодировка для текстового файла.

 

Кодировка DOS (866). Иначе - 1251.

 

Разделитель полей - основной разделитель в файле. Например, точка с запятой или "|" или любой другой в т.ч. не односимвольный разделитель.

 

Дополнительный разделитель полей - если используются "заголовки блоков данных", которые так же описывают необходимые данные, то в них может применяться иной разделитель полей.

 

Разделители встречаются между знаками. Бавает, что простые разделители (например точка с запятой) встречаются собственно в данных файла.

Для таких случаев принято ограничивать поля содержащие разделитеь специальными символами. Например символом (").

 

Каждое поле - новая строка. Для форматов файла, когда каждое поле описано собственной строкой, а разделителем по сути является конец строки.

 

Знак между именем поля и значением. В случае формата файла "Каждое поле - новая строка" может быть принят принцип заполнения строки Переменная=Значение.

В таком случае нужно установить знак между именем поля и значением "=".

 

Пустые строки между записями указывает на то, что если для разделителя полей применяется новая строка, то для разделителя собственно записей - пустая строка.

 

Формат полей даты - форматная строка, позволяющая описать формат преобразования из данных файла (импорт) или для данных файла (экспорт) всех полей типа дата.

 

Пропустить число символов с начала строки. Это символы которые не будут считаться не разделителями и не данными.

 

Пропустить число символов в конце строки.

 

Номер поля, которое всегда заполнено - колонка в файле Excel, по которой будет определяться заполнено ли значение - достигнут ли конец файла.

 

Допустимое число пустых строк при поиске конца файла - при импорте программа пропустит, но не остановится при наличии определенного числа пустых строк (сделанных, например, в целях разрыва для печати).

 

Номер листа в книге - в книге excel есть понятие "Лист" (WorkSheet). Данные для импорта или при экспорте могут находится на любом из них (по номеру).

 

Число строк с начала файла, которые следует пропустить.

 

Разделитель даты. В файле может быть принят иной разделитель даты, чем системный.  Например запятая вместо точки или вовсе знак дефис.

 

Разделитель дробной части чисел. В файле может быть принят иной разделитель дробной части чисел, чем системный.

 

Первая строка заголовок сформирована из наименований полей.

 

Использовать строку-заголовок может содержать любой текст для импорта.

 

Не учитывать последнюю строку файла (одну) или Число строк с конца файла.

 

Фильтровать неподходящие символы. При данной установке все символы с кодом вне (> 31 и < 127)  и (> 1039 и < 1104) будут отфильтрованы.

 

Добавить текст в шапку/подвал файла - произвольный текст для экспорта файлов с определенной шапой и(или) подвалом.

 

На закладке "Последовательность полей" задается перечень полей файла в строгой последовательности.

Специальные команды "Оценить" помогут заполнить таблицу перечня полей по конкретному файлу или объекту конфигурации или его таблицы.

После их использования нужно внимательно отредактировать таблицу перечня полей, указав все необходимое.

 

Поле таблицы - элемент специального справочника "полей" файла. Существует множество предопределенных (гтовых) "полей", которые следует использовать для специального поведения при импорте экспорте.

Так, например поле "ОКПО Клиента" указывает на значение реквизита КодПоЕДПОУ для реквизита Клиент объекта обмена.

"Поля" можно создавать самостоятельно. наименование поля - реквизит объекта, как он назван в конфигурации. Можно использовать точку для субреквизитов (только для экспорта).

Определенное значение поля - строгое значение.

Если предопределеное поле "Пропустить данное поле", которое указывает на то, что при импорте данное поле не будет учитываться.

 

Заголовок поля по-умолчанию - Имя поля.

 

Данное поле принадлежит к объекту-хозяину таблицы. Так, если загружается таблица, можно указать поле объекта-хозяина (документа ли справочника).

 

Коэффициент для числовых полей - позволяет умножить значение при импорте и раздеить при экспорт на определенный коэффициент.

Например, если загружется колонка количество, и в файле испольъется количество в тысячах штук, то коэффициент следует установить = 1000.

 

Тип для DBF устанавливается строго по законам оглавления полей файлов DBase, но рекомендуем указывать его в любом случае (для других форматов файла). Так программа сможет определить, что поле числовое или дата.

 

Назначение поля - комментарий описания данного поля.

 

Имя поля в файле должно быть заполнено обязательно. Оно может понадобится для строки заголока текстового документа, оно необходимо для обмена через файлы DBF и выгрузки результатов произвольного запроса.

При работе с DBF имя поля обрезается до 10 символов и переводится на латинские символы.

 

Признак "Не заполнено, тогда принимает предыдущее значение" позволяет организовать при импорте данных данного поля пропуск пробелов, распространяя предыдущее значение поля на все последующие строки, если поле незаполнено.

 

Данное поле ключевое - определяет первостепенные поля при поиске данных (при импорте). Справочники имеют предопределенные ключевые поля - код и наименование. Документы - номер и дату.

Например, если такой признак стоит для поля Артикул номенклатуры - при импорте наменлатура в первую очередь будет определяться по своему артикулу.

 

Параметр Использовать заголовки блоков данных - позволяет организавть работу с файлами, у которых в строках данных встречаются "заголвки" - блоки информации, имеющие собственную структуру.

О том, что данная строка файла - заголовок можно определить за счет двух условий.

В первом условии Поле - имя поля (из обычной последовательности полей) отвечает условию и значению.

Или условие заголовка блока данных - наличие подстроки.

Поля заголовка составляются в таблице в том же виде - перечнем из справочника "полей".

 

Закладка "Настройки шаблона" предназначена для указания настроек поведения при импорте и экспорте.

Для обновления существующих элементов - указывает на то, что при импорте система будет в первую очередь искать существующие объекты по значению ключевых полей и обновлять данных этих объектов.

 

Всегда создавать новый элемент или документ - указывает обратное действие при импорте. Независимо от наличия и значения ключевых полей система будет создавать новые элементы справочников и документов.

 

Использовать в качестве ключевых только поля из таблицы - указывает, что не будут использоваться при поиске предопределенные ключевые поля (для справочников - код и наименование, для документов - номер и дата).

 

Открывать форму каждого импортируемого - по результатам импорта все основные загруженные объекты один за другим будут открыты (их карточки).

 

При экспорте, перед записью, выводить окно с перечнем выгрузки для отбора (только "толстый клиент"). В этом режиме программа при экспорте сначала сфомирует таблицу для экспорта и предложит ее для редактирования пользователю, а затем произведет экспорт отредактированной таблицы.

 

Не пропускать объекты помеченные на удаление. По умолчанию система не учитывает такиеобъекты при импорте и экспорте. При установке данного признака - не пропускает, то есть учитывает.

 

Создавать элементы при отсутствии - указывает на то, что при импорт будлут создаваться новые объекты, если система не обнаружила их по значению ключевых полей.

 

Составлять наименование по выражению из имен полей. Так, например при наличии в таблице различных полей с именами Фамилия, Имя, Отчество можно составить наименование "Имя Отчество Фамилия".

 

Составлять наименование группы из имен полей - будет попытка поиска группы с сотсавным имененм и (нсли не найдено) создание новой группы.

 

Разделитель в составном наименовании. Если вернуться к примеру с ФИО, можно заметить, что при незаполненом поле "Отчество" в наименование подет сразу два пробела. Если указать в данном поле пробел, система сократит их.

 

Определенные значения для новой номенклатуры для шаблонов с видом "Товары":

Номенклатурная группа, Группа (родитель) для новой номенклатуры, Тип новой номенклатуры, Составлять характеристику номенклатуры (есл используются характеристики) из имен полей, Способ пополнения номенклатуры,

Счет учета Товаров, Счет учета затрат, Единица измерения (если не заполнена базовая), Направление деятельности, Способ списания.

 

Правила для документов:

Попытаться проводить документы (для импорта).

 

Открывать форму только у непроведенных документов, которые система пыталась провести при импорте, но это не удалось.

 

Не пропускать непроведенные документы (по умолчанию непроведенные документы пропускаются).

 

Выдавать запрос для интервала документов - он служит ограничением при импорте и отбором для экспорта.

 

Устанавливать дату документам при иморте (если не определена) определенную или Устанавливать текущую дату.

 

Блок Условия операций с документами денежных средств:

Обрабатывать платежные документы - пара типов днежных документов для обмена. Это расход со счета/поступление на счет, расход из кассы/поступление в кассу, план расхода/пла поступления или платежная ведомость (только расход).

Например, для загрузки информации из ПО Банк-клент нужно указать пару "Поступление и расход со счета".

 

Только приходные/только расходные.

 

Суммировать данные документов Расход ДС План при выгрузке для одного Клиента от одной организации в течении суток - механизм позволяющий аккамулировать при выгрузке в ПО банк-клиент платежки одного характера в единую платежку на общую сумму.

 

Суммировать, только если платежки с одним номером - суммироваться буду платежки по признаку одного Счета организации, одного счета Клиента, одной Даты платежа и: с одинаковыми номерами платежей.

 

Вставлять между текстами назначения следующий текст - позволяет собрать текст назначения такой аккумулирующейся платежки из текстов назначений документа План расхода ДС с разделителем. Например, - запятой.

 

Сообщать о недостаточном заполнении при экспорте документов для ПО Банка - при экспорте данных для банковского ПО программа будет сообщеть, если критические поля (например, номер счета или данные ОКПО) не заполнены.

 

Номер документа (в системе) устанавливать по Номеру входящего документа. Иначе - номер новым документам назначает программа автоматически, согласно нумератору.

 

Устанавливать признак аванса документам денежных средств.

 

Статьи ДДС Расхода и прихода по умолчанию.

 

Налогообложение НДС по умолчанию.

 

Валюта не национальная. При импорте данных банк клиента возможно понадобится использовать поле признак "Валютная операция", но без указание на наименование или код конкретной валюты.

В этом случае следет заполнить это поле, чтобы определить ненациональную валюту операции.

 

Отбор по организация/по счетам и кассам - такой отбор будет условием при экспорте и отбором при импорте данных. Он может состоять из любого числа оргаизаций и/или расчетных счетов и касс организаций.

 

Сложное условие дебета. При отсутсвии в файле поля признака дебета или кредита операции, можно организовать сложное условие дебета, Указав Имя поля, условие и значение.

 

Блок Данные Клиентов описывает поведение системы с Клиентами документов денежных средств.

Создавать Клиентов по ОКПО. Если есть значение поля ОКПО и Клиент по нему не найден, он будет создан.

 

Группа новых Клиентов - где он будет создан.

 

Создавать р/счета - расчетные счета Клиентов.

 

Создавать договора (если подходящий договор не обнаружен).

 

Устанавливать первый подходящий договор Клиента или Устанавливать договор Клиента, если подходит только один.

 

Поиск в назначении платежа номера счета и договора включает алгоритм при импорте документов, который пытается определить значение Договора с Клиентом и Документ основания типа Счет поставщику/покупателю по содержанию (тексту) назначения платежа.

 

Закладка Фильтры правила и программа.

Условия фильтра импорта-экспорта - таблица простых фильтров для отбора при экспорте и ограничения при импорте данных.

Условие фильтра базируется на номере поля из таблицы "последовательность полей", условии и значении.

Определяется поведением (пропустить/принять пустое значение). Не следует путать с данный механизм "правилами поведения".

 

Использовать правила поведения - позволяет организовать работу сложно-зависимых правил, чтобы наполнить объект импорта в зависимости от данных или выполнить какой либо программный код при условии.

 

Правила поведения указываются в таблице по порядку и хранятся в отдельном справочнике. Они могут быть использованы в любом количестве шаблонов файлов.

В карточке правила поведения нужно указать Имя реквизита (для импорта) или имя поля/колонки (для экспорта), условие поведение (импорт) и значение.

В табличной части - собственно перечислить поведение для импорта или условия для экспорта.

Приведем такой пример, - предположим, необходимо чтобы при импорте данных в случае если в значении реквизита "Комментарий" встречается слово "прочий" установить значение реквизита "Статья" в значение "Прочие расходы" из справочника Статьи движения денежных средств.

Тогда в шапке "поведения" Имя реквизита - "Комментарий", условие - "Содержит", значение - ""прочий"", а в табличной части: Это реквизит таблицы - "нет", Имя - "Статья", Условие - не заполнено, Значение - "Прочие расходы" из справочника Статьи движения денежных средств.

 

При экспорте условие выполняется в обратную сторону - если соблюдены все условия из табличной части, то поле (из шапки) принимает значение (из шапки).

 

Признак Выполнять программы позволяет вставить программный код, который будет исполнять во время импортных и экспортных операций для каждого обрабатываемого файла:

 

Перед началом, после окончания, в начале обработки строки, после окончания обраотки строки, в начале обработки каждого поля, в окончании обработки каждого поля.

Примеры объектов, которые можно использовать в программе: ОбъектОперации, СтрокаТаблицыИлиЭлемент, АдресФайла, ШаблонФайла.

 

Признак "Данный шаблон использовать только для объектов с определенными реквизитами" включает механизм, который отбирает для импорта и экспорта объекты с определенным значением реквизитов.

 

На закладке "Отбор по объектам" можно указать быстрый отбор по значению реквизитов Организация и/или Клиент, а в таблице - указать более сложные условия отбора.

 

Команда "Заполнить по реквизитам объекта" позволит не угадывать имена реквизитов объекта или его таблицы (какими они были заданы в конфигураторе).

 

Качество условия (И, ИЛИ, НЕ) позволяет собрать сложное условие (они проверяются последовательно). Так, за счет этого механизма, можно задать, чтобы например, были выгружены все документы с условием ПометкаУдаления = Истина.

Если для реквизита указано Имя таблицы, то условие использует значение в первой строке таблицы.

 

Дополнительное заполнение при импорте - механизм, позволяющий заполнить любые поля объекта импорта значениями по-умолчанию.

 

Команда "Заполнить по реквизитам объекта" позволит не угадывать имена реквизитов объекта или его таблицы (какими они были заданы в конфигураторе).

Если заполнено Имя таблицы - данное значение будет установлено для всех строк таблицы
