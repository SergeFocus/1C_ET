Довідник Постачальники

Призначений для картотеки Постачальників. Використовується, якщо включений облік по постачальниках.

Передбачений зумовлений Постачальник "Приватна особа" для реєстрації даних про неперсоніфікованих надбаннях. Тобто таких, для яких немає необхідності відстежувати заборгованість. Цього постачальника можна перейменувати.

Довідник ієрархічний. Тобто Постачальників можна групувати.

Реквізити:

Вид Цін (якщо включена посістема Складного Механізму обліку Цін) визначає ціну за замовчуванням для оприбуткування товару від постачальника. Вид Цін можна призначати так само для всієї групи постачальників.

Регіон (якщо включений облік регіонів) визначає розташування Постачальника. За цією ознакою можна групувати Постачальників у звітах.

Адреса,Телефон,ОКПО,Правовий статус,Електронна пошта, Реквізити безготівкового розрахунку - Додаткові дані про постачальників.

Склад постачальника (при включеній підсистемі) "Облік за складами" дозволяє вказати склад за замовчуванням для Постачальника. Тоді, при виборі Постачальника в документах, такий склад буде встановлено автоматично.

Найменування для друку - Текст, який буде виводитися в поле "Постачальник" при друку документів з програми. Цей текст може відрізнятися від звичайного найменування Постачальника. Наприклад, найменування постачальника - Парус, ТОВ, а найменування для друку - ТОВ "Парус", Одеса, вул.Мала Арнаутська, 9.

Основна Валюта (якщо включений Облік валют) встановлюється для того, щоб у документах при виборі цього постачальника автоматично встановлювалася певна валюта.

Власний співробітник (якщо включений облік заробітної плати) може виступати Постачальником. Такі Постачальники створюються автоматично при введенні інформації по співробітникам. Це необхідно для того, щоб врахувати взаєморозрахунки з таким постачальником при розрахунку заробітної плати співробітника.

Виробник - (якщо включений облік) Посилання на Виробника всієї номенклатури, купленої у даного постачальника. Так само потрібно у випадку, коли сам Постачальник це Виробник номенклатури. Потім у звіті "Товари. Залишки і обороти"Можна у варіанті звіту" по виробниках "вивчити руху його товару. Так само в звіті по Продажам передбачений варіант "продажу номенклатури по виробниках".

Врахуйте, що якщо в програмі вести облік по постачальникам, але не вести облік грошей, То при аналізі звіту "За розрахунками з постачальниками" картина виглядатиме однобічно. Неначе товар купувався, але не оплачувався, оскільки немає можливості зареєструвати в програмі оплату постачальнику при відключеному обліку грошей.

Коли потрібно зробити взаємозалік боргу між Постачальниками та / або Клієнтами з урахуванням надходження / вибуття грошей, слід скористатися спеціальною формою коригування "Взаємозалік боргу контрагентів".

Контрагенти - загальне найменування для клієнтів і постачальників.

За командою "Друк" можна отримати картку Постачальника з поточним боргом і його документами.
