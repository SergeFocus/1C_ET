Directory "SUPPLIERS"

Designed for catalogs Distributors. Used if included accounting providers.

Provides predefined Supplier "Private person" for the registration of non-personal data acquisitions. Ie, those for which there is no need to keep track of debt. You can rename this supplier.

Directory hierarchy. Ie Distributors can be grouped.

Details:

View Price (Unless Included posistema Complex mechanisms accounting prices) determines the price of default for posting goods from the supplier. View Price can assign the same for the whole group of suppliers.

Region (Unless included accounting regions) determines the location of the supplier. According to this feature, you can group Distributors reports.

Address,Phone,Of Enterprise,Legal status,E-mail, details of cashless payment - Additional information about the suppliers.

Warehouse Supplier (When the subsystem) "Accounting Warehouse" allows you to specify the default warehouse for the Supplier. Then, when choosing a supplier in documents such storage will be installed automatically.

Name Print - The text that will be displayed in the "Supplier" when printing documents from the program. This text may differ from the usual name of the supplier. For example, the name of the supplier - Sail Ltd, a name for the print - "Parus", Odessa, Malaya Arnautskaya 9.

Summary Currency (if enabled Accounting rates) is set to the documents if you choose this provider automatically sets certain currency.

Net employee (if enabled payroll accounting) can act Supplier. Such providers are created automatically when you enter information employees. It is necessary to take into account settlements with the supplier when payroll employee.

Manufacturer - (if enabled account) Reference to the Manufacturer of all range purchased from this supplier. The same should be the case when he is Manufacturer Supplier nomenclature. Then in the report "Goods. Balances and transactions"Version of the report can be" by producers "to explore the movement of its goods. Likewise in Sales Reports provided the option "sales nomenclature by producers."

Note that if the program to keep records by providers, but not lead keeping moneyThen in analyzing report "to suppliers" picture will look one-sided. As if the goods were bought but not paid for, so it is not possible to register in the program payment to the supplier when the power of money account.

When it is necessary to offset the debt between the supplier and / or Customers taking into account income / outflow of money, you should use a special form of corrections "Offsetting debt counterparties. "

Accounts - the common name for customers and suppliers.

At the command "Print" you can get a card with the Supplier and its current debt instruments.
