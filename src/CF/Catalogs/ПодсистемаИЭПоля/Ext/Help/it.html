Riferimento "file di modello di campo"

La directory è inteso per memorizzare un elenco di possibili campi per file modelliUtilizzato per le operazioni di import-export con i file.

Vi è un grande insieme di campi predefiniti per i quali programmate comportamento durante il carico e lo scarico. In particolare per caricare e scaricare la tabella "Prodotti" per lo scambio di "banca-cliente".

Nome - Nome di autorizzazione del database puntelli severi, che corrisponde a sostenere 1C configurazione. Se si utilizza un nome con un punto (Nomenklatura.Kod), il programma tenterà di scaricare per scaricare il contenuto subrekvizita (codice articolo). Se questo campo viene utilizzato nel modello, come una chiave, il programma tenta carico di dati per determinare il valore dell'oggetto (nomenclatura) di subrekvizitu (codice).

Un commento una descrizione di base dei campi del modello.

Tipo di campo (String S, numeri N, Data D) utilizzati in formato DBF per determinare la colonna del file di database di tipo. Lunghezza eAccuratezza - Determinare la lunghezza del campo di DBF. Precisione tra l'altro, suggeriscono fortemente che il campo - il numero.

Golf Una certa importanza specificare un valore fisso per lo scarico e carico, indipendentemente dai dati contenuti.

Campo di intestazione di default campo specifica il nome del file di database. In uno scambio con DBF verrà troncato a 10 caratteri e tradotto in latino.

La bandiera che Campo appartiene al proprietario della tabella dell'oggetto Esso consente di specificare i documenti campi di intestazione o guide della carta nel carico e scarico di tavolo parti.

Campi di Exchange prodotte nella directory Modelli di import-export. Quando scambiate i modelli, scaricato lo stesso campo utilizzato.
