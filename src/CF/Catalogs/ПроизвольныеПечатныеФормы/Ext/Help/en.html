Directory Custom Printed Forms

Guide for the registration and storage of their own arbitrary printed forms.

Custom printed forms - an alternative mechanism for printing elements of directories and documents. Through it, users can create their own print form, without making any changes to the configuration, share, modify and use on your own.

Custom printing plates are set for the type of printing plate.

If the actual type of an arbitrary shape for the printing plate, according to the conditions not set, printing will be done the usual built-in printed form.

Custom printed forms, you can also ask the user to register a strictly "No printed forms."

Layout of arbitrary shape can be printed in an external file or mxl be loaded into the database.

The structure of the layout should contain a print area with print options described on the "Description". They are different for different print objects.

You can specify complex programmable parameters and produce intermediate calculations in the formation of the printing plate.

Programmable parameters and common parameters intermediate calculations can also be arranged in the external layout of the printing plate.

The card printed form, you can set strict parameters of the print.

Using the printing plate can be limited to a period and a list of users.

The program conditions for automatic selection allows you to set a condition that the document of the same species in the same period of time for a single user to print different printing plates.

In such a program in the language of 1C can specify a condition to the props "SsylkaNaElement'Which indicates a document or directory entry to be arbitrary printing.

For example, the program:

TipZnch (SsylkaNaElement.KlientPostavschik) = Type ("SpravochnikSsylka.Klienty")
