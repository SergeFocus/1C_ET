L'elenco è destinato a memorizzare un elenco di possibili campi per i file modelliUtilizzato per le operazioni di import-export con i file.

Vi è un grande insieme di campi predefiniti per cui il comportamento programmato durante il carico e scarico. In particolare per caricare e scaricare la tabella "Prodotti" per lo scambio di "Banca-cliente".

Nome - Nome di autorizzato di database puntelli severi che corrisponde a sostenere 1C configurazione. Se si utilizza un nome con un punto (Nomenklatura.Kod), il programma tenterà di scaricare quando scarico subrekvizita contenuto (numero di articolo). Se questo campo verrà utilizzato nel modello come chiave, allora i dati vengono caricati il ​​programma tenta di determinare il valore dell'oggetto (nomenclatura) in subrekvizitu (codice).

Commento si tratta di una descrizione di base dei campi del modello.

Tipo di campo (String S, numero N, Data D) utilizzati in formato DBF per determinare la colonna del file di database di tipo. Lunghezza ePrecisione - Determinare la lunghezza del campo di DBF. Accuratezza, tra l'altro suggeriscono fortemente che il campo - numero.

Campo Un certo valore specificare un valore fisso per lo scarico e carico, indipendentemente dai dati di contenuto.

Campo di intestazione di default specifica il campo del nome del file di database. In uno scambio con DBF verrà troncato a 10 caratteri e tradotto in latino.

Flag che Campo appartiene al proprietario della tabella dell'oggetto consente di specificare i documenti campi di intestazione o scheda di riferimento per lo scarico e carico delle parti della tabella.

Campi di Exchange prodotte nella directory di modelli import-export. Quando scambiate i modelli, scaricato lo stesso campo utilizzato.
