Directory "Fields template files"

The directory is intended to store a list of possible fields for files templatesUsed for the import-export operations with files.

There is a large set of predefined fields for which the programmed behavior during loading and unloading. In particular To load and unload the table "Products" for the exchange of "Bank-client".

Name - Name of authorized strict props database that corresponds to prop configuration 1C. If you use a name with a dot (Nomenklatura.Kod), the program will try to unload when unloading content subrekvizita (item number). If this field will be used in the template as a key, then the data is loaded the program will attempt to determine the value of the object (nomenclature) in subrekvizitu (the code).

Comment it is a basic description of the template fields.

Field type (String S, number N, Date D) used in DBF format to determine the type column of the database file. Length andAccuracy - Determine the length of the field DBF. Accuracy inter alia strongly suggest that the field - number.

Field A certain value to specify a fixed value for unloading and loading, regardless of the content data.

Header field by default specifies the name field of the database file. In an exchange with DBF will be truncated to 10 characters and translated into Latin.

Flag that Field belongs to the owner of the object table allows you to specify header fields documents or reference card for the unloading and loading of the table parts.
