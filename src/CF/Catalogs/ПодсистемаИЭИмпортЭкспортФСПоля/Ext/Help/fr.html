Répertoire "Fields fichiers de modèle"

Le répertoire est destiné à stocker une liste de champs possibles pour les fichiers modèlesUtilisé pour les opérations d'import-export avec les fichiers.

Il ya un grand nombre de champs prédéfinis pour lesquels le comportement programmé pendant le chargement et le déchargement. En particulier pour charger et décharger les tables "Produits" pour l'échange de «Banque-client».

Nom - Nom de base de données autorisée des accessoires strictes qui correspond pour soutenir la configuration 1C. Si vous utilisez un nom avec un point (Nomenklatura.Kod), le programme va essayer de décharger lors du déchargement subrekvizita contenu (numéro d'article). Si ce champ sera utilisé dans le modèle comme une clé, puis les données sont chargées du programme tentera de déterminer la valeur de l'objet (nomenclature) dans subrekvizitu (le code).

Commentaire ce est une description de base des champs de modèle.

Type de champ (String S, nombre N, Date D) utilisé en format DBF pour déterminer la colonne de type de fichier de base de données. Longueur etPrécision - Déterminer la longueur de la DBF sur le terrain. Précision notamment suggèrent fortement que le champ - nombre.

Domaine Une certaine valeur spécifier une valeur fixée pour le déchargement et le chargement, quelles que soient les données de contenu.

champ d'en-tête par défaut spécifie le champ du fichier de base de données de nom. Dans un échange avec DBF sera tronqué à 10 caractères et traduit en latin.

Drapeau Champ appartient au propriétaire de la table d'objet vous permet de spécifier les documents des champs d'en-tête ou de la carte de référence pour le déchargement et le chargement des pièces de table.

champs de change produites dans le répertoire de modèles d'import-export. Lorsque modèles échangés, déchargé le même domaine utilisé.
