Довідник "Шаблони файлів"

Шаблони файлів імпортно-експортних операцій доступні також в розділі "Налаштування і адміністрування" в меню зліва. У папці шаблонів "Зумовлені" зберігаються шаблони, встановлені програмістом.

Зміни в ці шаблони можуть вноситися користувачами тільки з дозволу програміста системи, оскільки поведінка програми при імпорті та експорті для них нестандартне.

З вікна списку шаблонів доступні унікальні функції системи:

Список команд І-Е (Імпорт-експорт) після включення системи з'являється в списках і картках всіх стандартних довідників і документів системи. Даний перелік команд дозволяє запустити вивантаження або завантаження для списку елементів або одного елемента бази даних. Якщо відповідний шаблон тільки один, він буде обраний системою автоматично.

Але в списку довідника "Шаблони файлів імпорту-експорту" в списку команд І-Е доступні додаткові:

Зумовлені операції імпорту-експорту - для запуску обміну по унікальним алгоритмам, закладеним розробниками системи.

Робота з такими завданнями вимагає персонального навчання і не описана в рамках даної довідки.

Пакетний обмін для набору файлів - Коли необхідно завантажити або вивантажити об'єкти бази даних, використовуючи одночасно два і більше файлів.

Наприклад, один для шапки документа, а два інших - для табличних частин документа.

У даній обробці необхідно вказати Тип об'єкта обміну АБО власне об'єкт обміну, Вказати шаблон файлу імпорту-експорту для шапки об'єкта, адресу файлу для нього. Можна вказати період (якщо мова йде про документи).

Потім заповнити таблицю, вказавши ІмяТабліци об'єкта, яким воно задано в конфігураторі, шаблон файлу опису даної таблиці, відповідний адресу файлу.

Список задач обміну - Для запуску імпортно-експортних операцій списком в певній послідовності.

У даній обробці можна заповнити таблицю списку набором з шаблонів файлів і їх адрес, вказати напрямок "Імпорт - Так, Експорт - Ні".

Вказати Тип об'єкта обміну і його ІмяТабліци, якщо необхідно. А потім запустити обмін списком операцій. Для операцій над документами можна вказати період відбору.

Обробка Обмін документами грошових коштів (так само з розділу "Грошові кошти") надає можливість виконати список операцій з імпорту (Ctrl + I) Та експорту (Ctrl + E) Для переліку завдань. При відкритті обробка підбирає в список підходящі шаблони (для обміну документами грошових коштів).

Інші команди списку шаблонів файлів:

Журнал імпортно-експортних операцій. У цей журнал заноситься інформація про зроблених у системі імпортних та експортних операціях, якщо для них в шаблоні файлу вказано "Вести журнал".

Наприклад, для всіх імпортних операцій з прапором "Завантажувати всі файли з папки" ведеться журнал, щоб виключити повторне завантаження файлів.

Завантажити шаблони / Зберегти шаблони. Дані операції дозволяють обмінятися шаблонами файлів імпорту-експорту з іншими користувачами системи або розробниками системи.

Файли опису зберігаються в спеціальному форматі "ieUNFSH" і при імпорті синхронізуються по найменуванню шаблону. Не всі опції шаблону можна перенести за рахунок завантаження шаблонів. Ті параметри, які індивідуальні для бази даних, необхідно буде вказати вручну.

Картка шаблону файлу імпорту-експорту.

Це найскладніший об'єкт системи. Для зручності його слід заповнювати зверху-вниз справа наліво.

Наприклад, галочки в шапці "тільки для експорту" або "тільки для імпорту" допоможуть заховати параметри шаблону, які не будуть використані в даному випадку.

Вид шаблону визначає основну задачу даного шаблону, крім параметра "Будь-який варіант". Радимо вибрати вид шаблону, який максимально відповідає Вашої задачі.

У зв'язку з вибором того чи іншого виду шаблону стануть доступні ті або інші параметри шаблону файлу. Кожен параметр описаний нижче.

А види шаблону можуть бути наступними:

    • Для картки довідника або документа - Шаблон описує сам об'єкт або яку або таблицю для елемента бази даних довідника або документа.

Такий шаблон дозволить відповідно завантажити або вивантажити об'єкт або його таблицю. Одна строчка такого файлу імпорту-експорту - один реквізит шапки або один рядок таблиці.

    • Для списку елементів довідника або документів - Такий шаблон описує поля з розрахунку одна строчка файлу шаблону - один елемент довідника або один документ.

На закладці "Налаштування шаблону" з'явиться блок параметрів "Правила для роботи зі списками".

    • Для таблиці Товари - Для табличних частин "Товари" документів. При імпорті даних такого шаблону, програма спробує дозаповнити дані рядків для товарів.

На закладці "Налаштування шаблону" з'являться спеціальні настройки. Наприклад, якщо буде встановлено прапор "створювати нові об'єкти, то буде видно блок параметрів значень за замовчуванням для нової номенклатури" правила для роботи з запасами ".

    • Для обміну документами грошових коштів - Спеціальний вид шаблону для реалізації поведінки (заповнення при імпорті) для документів грошових коштів в т.ч. для обміну з системами Банк-Клієнт.

Блок "Умови операцій з документами грошових коштів" дозволяє описати умови обміну докладніше.

    • Результат довільного запиту - Вид шаблону, що дозволяє організувати експорт результатів довільного запиту на мові 1С 8.

Для даного виду запиту на закладці "Налаштування шаблону" можна вказати текст довільного запиту, написаного на мові 1С 8, а так само задати значення параметрів запиту в таблиці нижче.

При цьому, на закладці "Послідовність полів" в "Ім'я поля" слід вказувати імена полів - результатів запиту.

Кнопки "Завантажити параметри з тексту запиту", "Створити поля файлу за результатами запиту", "Перевірити запит" дозволяють наповнити таблиці полів і параметрів автоматично, перевірити працездатність запиту. Кнопка "Виконати вивантаження" проводить вивантаження даних прямо з картки шаблону.

Ім'я таблиці - Параметр для вказівки таблиці для довідника або документа (такий, як вона названа в конфігураторі), яка і буде об'єктом імпортно-експортної операції.

Далі описані параметри шаблону, які зустрічаються на закладках його картки залежно від його виду:

Закладка "Параметри файлу" служить для вказівки параметрів файлу (ів) імпорту або експорту.

Завантажувати всі файли з папки - Вказує на те, що це імпорт для масиву з файлів. Нижче тоді слід вказати каталог, а маска файлу визначить відбір підходящих файлів.

Вказуйте маску *. * Для завантаження всіх файлів. Якщо в журналі імпортно-експортних операцій вже є запис про таку файлі - він буде пропущений.

Шукати в підкаталогах - Для завантаження будуть обрані файли по масці в каталозі і його підкаталогах.

Строгий адресу файлу - Можна вказати файл для імпорту / експорту відразу. Інакше, перед операцією імпорту або експорту буде викликатися діалог вибору файлу.

Файл в архіві - Упаковувати даний файл після експорту / обробляти всі файли даного архіву при імпорті;

Маска файлу - Можна організувати завантаження або вивантаження в файл, ім'я якого сформовано по спеціальній масці з урахуванням дати.

Так, наприклад текст # рррр # в масці буде замінений на 4 символу поточного року, а ## - ## - #### перетвориться на 31-03-2011 (якщо Завтра перше квітня).

Вести журнал імпортно-експортних операцій - Чи буде записуватися (у разі успіху) інформація про кожну операцію з цим шаблоном до реєстру.

Видаляти файл після вдалого імпорту.

Формат файлу. На даний момент система підтримує обмін для форматів файлу: текстовий, файл MS Excel, DBase.

Для використання формату MS Excel в системі (де розташований сервер 1С, якщо робота ведеться через нього) повинен бути встановлений програмний пакет MS Excel.

Конкретна кодування для текстового файлу.

Кодування DOS (866). Інакше - 1251.

Роздільник полів - Основний роздільник у файлі. Наприклад, крапка з комою або "|" або будь-який інший в т.ч. НЕ Односимвольний роздільник.

Додатковий роздільник полів - Якщо використовуються "заголовки блоків даних", які так само описують необхідні дані, то в них може застосовуватися інший роздільник полів.

Роздільники зустрічаються між знаками. Бава, що прості роздільники (наприклад крапка з комою) зустрічаються власне в даних файлу.

Для таких випадків прийнято обмежувати поля містять разделітеь спеціальними символами. Наприклад символом (").

Кожне поле - новий рядок. Для форматів файлу, коли кожне поле описано власної рядком, а роздільником по суті є кінець рядка.

Знак між ім'ям поля і значенням. У разі формату файлу "Кожне поле - новий рядок" може бути прийнятий принцип заповнення рядка Мінлива = Значення.

У такому випадку потрібно встановити знак між ім'ям поля і значенням "=".

Порожні рядки між записами вказує на те, що якщо для роздільника полів застосовується нова рядок, то для роздільника власне записів - порожній рядок.

Формат полів дати - Форматна рядок, що дозволяє описати формат перетворення з даних файлу (імпорт) або для даних файлу (експорт) всіх полів типу дата.

Пропустити число символів з початку рядка. Це символи які не будуть вважатися не роздільниками і не даними.

Пропустити число символів в кінці рядка.

Номер поля, яке завжди заповнене - Колонка в файлі Excel, за якою буде визначатися заповнене Чи має значення - чи досягнутий кінець файлу.

Допустима кількість порожніх рядків при пошуку кінця файлу - При імпорті програма пропустить, але не зупиниться за наявності певного числа порожніх рядків (зроблених, наприклад, з метою розриву для друку).

Номер аркуша у книзі - У книзі excel є поняття "Аркуш" (WorkSheet). Дані для імпорту або при експорті можуть знаходиться на будь-якому з них (за номером).

Число рядків з початку файлу, які слід пропустити.

Роздільник дати. У файлі може бути прийнятий інший роздільник дати, ніж системний. Наприклад кома замість крапки або зовсім знак дефіс.

Роздільник дробової частини чисел. У файлі може бути прийнятий інший роздільник дробової частини чисел, чим системний.

Перший рядок заголовок сформована з найменувань полів.

Використовувати рядок-заголовок може містити будь-який текст для імпорту.

Не враховувати останній рядок файлу (одну) або Число рядків з кінця файлу.

Фільтрувати невідповідні символи. При даній установці всі символи з кодом поза (& gt; 31 і & lt; 127) і (& gt; тисячу тридцять дев'ять і & lt; 1 104) будуть відфільтровані.

Додати текст в шапку / підвал файлу - Довільний текст для експорту файлів з певною шапой і (або) підвалом.

На закладці "Послідовність полів" задається перелік полів файлу в строгій послідовності.

Спеціальні команди "Оцінити"Допоможуть заповнити таблицю переліку полів по конкретному файлу чи об'єкту конфігурації або його таблиці.

Після їх використання потрібно уважно відредагувати таблицю переліку полів, вказавши все необхідне.

Поле таблиці - Елемент спеціального довідника "полів" файлу. Існує безліч зумовлених (ГТОВ) "полів", які слід використовувати для спеціального поведінки при імпорті експорті.

Так, наприклад поле "ОКПО Клієнта" указує на значення реквізиту КодПоЕДПОУ для реквізиту Клієнт об'єкта обміну.

"Поля" можна створювати самостійно. найменування поля - реквізит об'єкта, як він названий в конфігурації. Можна використовувати точку для субреквізітов (тільки для експорту).

Певне значення поля - суворе значення.

Якщо приречення поле "Пропустити дане поле", яке вказує на те, що при імпорті дане поле не буде враховуватися.

Тема поля по-замовчуванню - Ім'я поля.

Дане поле належить до об'єкта-господареві таблиці. Так, якщо завантажується таблиця, можна вказати поле об'єкта-господаря (документа чи довідника).

Коефіцієнт для числових полів - Дозволяє помножити значення при імпорті та раздеіть при експорт на певний коефіцієнт.

Наприклад, якщо загружется колонка кількість, і у файлі ісполь'ется кількість в тисячах штук, то коефіцієнт слід встановити = 1000.

Тип для DBF встановлюється строго за законами змісту полів файлів DBase, але рекомендуємо вказувати його в жодному разі (для інших форматів файлу). Так програма зможе визначити, що поле числове або дата.

Призначення поля - Коментар опису даного поля.

Ім'я поля у файлі має бути заповнено обов'язково. Воно може знадобиться для рядка заголока текстового документа, воно необхідне для обміну через файли DBF і вивантаження результатів довільного запиту.

При роботі з DBF ім'я поля обрізається до 10 символів і перекладається на латинські символи.

Ознака "Не заповнене, тоді приймає попереднє значення"Дозволяє організувати при імпорті даних даного поля пропуск прогалин, поширюючи попереднє значення поля на всі наступні рядки, якщо поле незаполнена.

Дане поле ключове - Визначає першорядні поля при пошуку даних (при імпорті). Довідники мають зумовлені ключові поля - код і найменування. Документи - номер і дату.

Наприклад, якщо така ознака коштує для поля Артикул номенклатури - при імпорті наменлатура в першу чергу буде визначатися за своїм артикулу.

Параметр Використовувати заголовки блоків даних - Дозволяє організавть роботу з файлами, у яких в рядках даних зустрічаються "заголвкі" - блоки інформації, що мають власну структуру.

Про те, що даний рядок файлу - заголовок можна визначити за рахунок двох умов.

У першій умові Поле - Ім'я поля (зі звичайної послідовності полів) відповідає умові і значенню.

Або умова заголовка блоку даних - наявність підрядка.

Поля заголовка складаються в таблиці в тому ж вигляді - переліком з довідника "полів".

Закладка "Налаштування шаблону"Призначена для вказівки налаштувань поведінки при імпорті та експорті.

Для оновлення існуючих елементів - Вказує на те, що при імпорті система буде в першу чергу шукати існуючі об'єкти за значенням ключових полів і оновлювати даних цих об'єктів.

Завжди створювати новий елемент або документ - Вказує зворотну дію при імпорті. Незалежно від наявності та значення ключових полів система буде створювати нові елементи довідників і документів.

Використовувати в якості ключових тільки поля з таблиці - Вказує, що не використовуватимуться при пошуку зумовлені ключові поля (для довідників - код і найменування, для документів - номер і дата).

Відкривати форму кожного імпортованого - За результатами імпорту всі основні завантажені об'єкти один за іншим будуть відкриті (їх картки).

При експорті, перед записом, виводити вікно з переліком вивантаження для відбору (тільки "товстий клієнт"). У цьому режимі програма при експорті спочатку сфомірует таблицю для експорту та запропонує її для редагування користувачеві, а потім справить експорт відредагованою таблиці.

Не пропускати об'єкти позначені на видалення. За замовчуванням система не враховує такіеоб'екти при імпорті та експорті. При установці даної ознаки - не пропускає, тобто враховує.

Створювати елементи при відсутності - Вказує на те, що при імпорт будлут створюватися нові об'єкти, якщо система не виявила їх за значенням ключових полів.

Складати найменування за висловом з імен полів. Так, наприклад при наявності в таблиці різних полів з іменами Прізвище, Ім'я, По батькові можна скласти найменування "Ім'я По батькові Прізвище".

Складати найменування групи з імен полів - Буде спроба пошуку групи з сотсавним імененм і (нслі, не знайдено) створення нової групи.

Роздільник в складеному найменування. Якщо повернутися до прикладу з ПІБ, можна помітити, що при незаповнених поле "По батькові" в найменування подет відразу два пробілу. Якщо вказати в даному полі пробіл, система скоротить їх.

Рахунок обліку Товарів,Рахунок обліку витрат,Одиниця виміру (якщо не заповнена базова), Напрям діяльності,Спосіб списання.

Закладка Фільтри правила і програма.

Умови фільтра імпорту-експорту - Таблиця простих фільтрів для відбору при експорті та обмеження при імпорті даних.

Умова фільтра базується на номері поля з таблиці "послідовність полів", умови ізначенні.

Визначається поведінкою (пропустити / прийняти порожнє значення). Не слід плутати з даний механізм "правилами поведінки".

Використовувати правила поведінки - Дозволяє організувати роботу складно-залежних правил, щоб наповнити об'єкт імпорту в залежності від даних або виконати який або програмний код за умови.

Правила поведінки вказуються в таблиці по порядку і зберігаються в окремому довіднику. Вони можуть бути використані в будь-якій кількості шаблонів файлів.

У картці правила поведінки потрібно вказати Ім'я реквізиту (для імпорту) або ім'я поля / колонки (для експорту), умова поведінку (імпорт) і значення.

У табличній частині - власне перерахувати поведінка для імпорту або умови для експорту.

Наведемо такий приклад, - припустимо, необхідно щоб при імпорті даних у разі якщо у значенні реквізиту "Коментар" зустрічається слово "інший" встановити значення реквізиту "Стаття" в значення "Інші витрати" з довідника Статті руху грошових коштів.

Тоді в шапці "поведінки" Ім'я реквізиту - "Коментар", умова - "Містить", значення - "" Інший "", а в табличній частині: Це реквізит таблиці - "Ні", Ім'я - "Стаття", Умова - не заповнено, Значення - "Інші витрати" з довідника Статті руху грошових коштів.

При експорті умова виконується у зворотний бік - якщо дотримані всі умови з табличної частини, то поле (з шапкі) приймає значення (з шапки).

Ознака Виконувати програми дозволяє вставити програмний код, який виконуватиме під час імпортних та експортних операцій для кожного оброблюваного файлу:

Перед початком, після закінчення, на початку обробки рядка, після закінчення обраоткі рядки, на початку обробки кожного поля, в закінченні обробки кожного поля.

Приклади об'єктів, які можна використовувати в програмі: Об'ектОпераціі, СтрокаТабліциІліЕлемент, АдресФайла, ШаблонФайла.

Ознака "Даний шаблон використовувати тільки для об'єктів з певними реквізитами"Включає механізм, який відбирає для імпорту та експорту об'єкти з певним значенням реквізитів.

На закладці "Відбір по об'єктах"Можна вказати швидкий відбір за значенням реквізитів Організація та / або Клієнт, А в таблиці - вказати більш складні умови відбору.

Команда "Заповнити за реквізитами об'єкта"Дозволить не вгадувати імена реквізитів об'єкта або його таблиці (якими вони були задані в конфігураторі).

Якість умови (І, АБО, НЕ) дозволяє зібрати складну умову (вони перевіряються послідовно). Так, за рахунок цього механізму, можна задати, щоб наприклад, були вивантажені всі документи з умовою ПометкаУдаленія = Істина.
