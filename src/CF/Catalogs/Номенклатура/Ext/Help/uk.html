Довідник номенклатури (товари, послуги)

Під загальним поняттям "номенклатура" маються на увазі: товари, послуги (роботи), елементи "наборів" з товарів і послуг і власне самі "набори".Номенклатура може бути зібрана в групи.
Номенклатура може враховуватися в кількісному обліку чи ні (роботи або товари на суму).
Номенклатура повинна бути оцінена для реалізації і мати свою оцінку при придбанні в разі, якщо необхідно з'ясувати валовий прибуток від її реалізації.

Передбачені елементи "Товар на суму" і "Послуга на суму" для реєстрації даних про таких продажах, для яких немає необхідності відстежувати залишки товару.Рухи для такого товару ігнорують показник "кількість".Ціну такого товару можна встановлювати кожен раз вручну.Ви також можете перейменувати ці Елементи.

Група "Архів" потрібна для перенесення застарілих елементів.Група "Проблеми завантаження" для визначення такої номенклатури при обміні.ти групи не можна видалити, але можна перейменувати.

При виборі номенклатури програма орієнтується в основному на її Найменування.Ви можетет почати найменування з унікального коду, щоб швидко укзивать товар в Документах.Наприклад, "05 Кулькова ручка".При введенні "05" в поле номенклатура в документіВитрати товару воно буде вибрано автоматично, якщо немає іншої номенклатури, найменування якої починається з "05".

Відсоток Знижкиякщо включений облік знижок) призначений для акцій, уцінки конкретної номенклатури.Якщо він встановлений для товару і при реалізації уКлієнта (При веденні Обліку по клієнтам) встановлено свій персональний відсоток знижки, то ціна формується з урахуванням більшого з цих відсотків знижки.У налаштуваннях програми можна змінити цю поведінку.

НЕ Відстежувати залишок (послуга)слід призначати для тих товарів і робіт, де залишки не потрібні.Наприклад для робіт по комплектації (збірці) або упаковці, доставці.Інший приклад: для товару типу "упаковка", якщо облік їх числа не ведеться.Ця ознака не можна змінити для номенклатури, якщо вона вже бере участь в проведенихдокументах, Їх все слід скасувати.

Рекомендований мінімальний залишок. Вналаштуваннях програми можна вказати поведінку при витрачанні товару менша за цей показник мінімального залишку - ігнорування / попередження / відмова.Не використовується для "наборів".

Планка кількості для продажу за пільговою ціноюякщо включений цей механізм) Вказує на таку кількість товару при продажу якого (або більше) буде призначена пільгова ціна.Коефіцієнт такої ціни можна встановитив настройках програми або в картціВиду Цен при складному механізмі ціноутворення.

Кількість за замовчуванням - дозволяє оперувати упаковками номенклатури.Наприклад при продажу сигарет по-штучно, можна поставити "20" для реалізації пачками.Програма буде всюди підставляти цю кількість.І при необхідності його можна скорегувати.

Номенклатура може бути "набором"Зі складу конкретних товарів (якщо включений механізмУ цьому випадку програма не враховує залишки і обороти "набору".Облік залишків ведеться для всіх елементів його складу.Але при цьому облікпродажів ведеться по "наборам". Набором не може стати номенклатура, яка вже була вказана в будь-якому документі.
Реалізація набору означає, що його "зібрали" з зазначеного складу товару та продали цілком, зрозуміло витративши всі товари складу.Повернення від покупців необхідно фіксувати так само "наборами".
Автоматичне ціноутворення для "набору" використовує суму цін усіх елементів його складу.При використанніСкладного механізмуобліку цін, рекомендується встановлювати ціни для "наборів" спеціальної підпрограмою документаУстановка цін - Кнопкою "Заповнити таблицю наборами і розрахувати їм ціни".
Після вказівки "набору" в будь-якому документі зміна його складу неможливо.Слід створювати інші набори з іншим складом.Набори можуть бути вкладеними, як елементи іншого набору.Але це може бути складно для обліку та візуалізації.
Для того, щоб набори виділялися в обліку, рекомендується в їх найменуванні вказувати на те, що це складовий елемент.
Так як набори маю незалежну ціну, показники валового прибутку для його компонент будуть кілька спотворюватися.У сумі показники валового прибутку будуть розраховані вірно, але ось для окремих компонент набору можуть істотно спотворитися.
Для аналізу продажів "наборів" користуйтеся варіантомзвіту "Продажі" для наборів.

Відмінність набору від комплектації: При реалізації набору списуються вся номенклатура його складу.При комплектації списуються матеріали і формується на складі новий екземпляр продукту.Який зберігається і числиться окремо, окремо реалізується, але може бути демонтований.

Номенклатура може вестися поСерійЦе означає що кожну одиницю номенклатури буде супроводжувати один елемент її серії.Серію можна встановити обов'язковою для вибору.У такій випадку неможливо буде вказувати кількість товару.Він буде приходоваться, переміщатися і витрачатися по-серійно.

Передбачені "повний" і "простий" режими обліку серій товару.

У повному врахуванні серію слід вказувати під час всіх подій - під час вступу, переміщенні, реалізації номенклатури.

В "простому режимі" серію досить вказати тільки при придбанні номенклатури (потім використовувати пошук).Таким чином можна тільки виявити де і як здобувалася ця серія.

Серії формуються на етапі прибуткового документа.Кожен серійний номер може бути оформлений власним Штрих-кодом і кодом виробника.

Якщо номенклатура вже використовувалася, призначити її серійної не можна.Слід створювати нову номенклатуру.

Неможливо використовувати серії для номенклатури з параметром "Не відстежувати залишок".

Середня ціна придбань це вся сума вартості придбань поділена на всю кількість придбання.використовується як оцінка для розрахунку валового прибутку у відсутності партійного обліку.

Використовується середньозважений обліку поточного залишку.В цьому режимі валовий прибуток залишається поняттям управлінського характеру, а не вважається для регламентованої (встановленої державою звітності).

Так, звичайно товари куплені по різному курсу, дають "скачки" в показниках валового прибутку в національній валюті.Але з іншого боку, може слід валютою обліку взяти долар (або євро)? Тоді стрибків не буде.

Скачков "стираються" і при хороших оборотах і не дуже то і помітні. У програмі можна прямо вказати продаж при поверненні (і надходження при поверненні постачальнику), щоб уникнути помилок.
