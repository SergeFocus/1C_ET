Справочник номенклатуры (товары, услуги)

 Под общим понятием "номенклатура" подразумеваются: товары, услуги (работы), элементы "наборов" из товаров и услуг и собственно сами "наборы". Номенклатура может быть собрана в группы.
Номенклатура может учитываться в количественном учете или нет (работы или товары на сумму).
Номенклатура должна быть оценена для реализации и иметь свою оценку при приобретении в случае, если необходимо выяснить валовую прибыль от ее реализации.
 

Предусмотрены элементы "Товар на сумму" и "Услуга на сумму" для регистрации данных о таких продажах, для которых нет необходимости отслеживать остатки товара. Движения для такого товара игнорируют показатель "количество". Цену такого товара можно устанавливать каждый раз вручную. Вы также можете переименовать эти Элементы.

Группа "Архив" нужна для переноса устаревших элементов. Группа "Проблемы загрузки" для определения такой номенклатуры при обмене. ти группы нельзя удалить, но можно переименовать.

Справочник иерархический. Это значит, что номенклатуру можно группировать.

При выборе номенклатуры программа ориентируется в основном на ее Наименование. Вы можетет начать наименование с уникального кода, чтобы быстро укзывать товар в Документах. Например, "05 Шариковая ручка". При вводе "05" в поле номенклатура в документе Расхода товара оно будет выбрано автоматически, если нет другой номенклатуры, наименование которой начинается с "05".

Параметры номенклатуры:

Цена - текущая цена товара. Если включен Сложный Механизм учета Цен, то цены номенклатуры регистрируются на дату для каждого Вида Цен. Иначе, если учет Цен "простой", цена реализации вносится вручную прямо в карточку номенклатуры или формируется автоматически по формуле с учетом Цены приобретения из документов Поступление Товара и Корректировки и Ввод остатков, указанной в настройках программы.

Процент Скидки - (если включен учет скидок)предназначен для акций, уценки конкретной номенклатуры. Если он установлен для товара и при реализации у Клиента (при ведении Учета по клиентам) установлен свой персональный процент скидки, то цена формируется с учетом большего из этих процентов скидки. В настройках программы можно изменить это поведение.

Наименование для печати следует заполнять только в тех случаях, когда в документах и прайсе номенклатура должнеа прозвучать иначе. Например, номенклутара "хот-дог" в печати может именоваться "Булка с сосиской".

Номенклатурная Группа (если включен учет в разрезе номенклатурных групп) - информационный разрез, позволяющий иначе взглянуть на товары. Номенклатурная группа может устанавливать наименование номенклатуры и быть связана с Группой Номенклатуры. Есть много интересных механизмов, связанных с использованием этой аналитики.

Производитель (если ведется учет по производителям) номенклатуры. Может обладать собственной формулой формирования Штрих-кода (если их использование в свою очередь включено в настройках).

Основной штрихкод (при использовании штрих-кодов). Если номенклатура обладает только одним штрих-кодом, то его можно указать прямо в карточке. Или сформировать за счет формулы из настроек программы или формулы в карточке производителя. Если Штрих-кодов у номенклатуры несколько, их можно перечислить в специальном регистре Штрих-кодов. Штрих-коды товара уникальные и не могут повторяться.

НЕ Отслеживать остаток (услуга) следует назначать для тех товаров и работ, где остатки не нужны. Например для работ по комплектации (сборке) или упаковке, доставке. Другой пример: для товара типа "упаковка", если учет их числа не ведется. Этот признак нельзя изменить для номенклатуры, если она уже участвует в проведенных документах, их все следует отменить.

Рекомендуемый минимальный остаток. В настройках программы можно указать поведение при расходовании товара меньше этого показателя минимального остатка - игнорирование/предупреждение/отказ. Не используется для "наборов".

Планка количества для продажи по льготной цене (если включен этот механизм) указывает на такое количество товара при продаже которого (или больше) будет назначена льготная цена. Коэффициент такой цены можно установить в настройках программы или в карточке Вида Цен при сложном механизме ценообразования.

Количество по умолчанию - позволяет оперировать упаковками номенклатуры. Например при продаже сигарет по-штучно, можно поставить "20" для реализации пачками. Программа будет везде подставлять это количество. И при необходимости его можно скорректировать.

Номенклатура может быть "набором" из состава конкретных товаров (если включен механизм). В этом случае программа не учитывает остатки и обороты "набора". Учет остатков ведется для всех элементов его состава. Но при этом учет продаж ведется по "наборам". Набором не может стать номенклатура, которая уже была указана в каком либо документе.
Реализация набора означает, что его "собрали" из указанного состава товара и продали целиком, разумеется израсходовав все товары состава. Возвраты от покупателей необходимо фиксировать так же "наборами".
Автоматическое ценообразование для "набора" использует сумму цен всех элементов его состава. При использовании Сложного механизма учета цен, рекомендуется устанавливать цены для "наборов" специальной подпрограммой документа Установка цен - кнопкой "Заполнить таблицу наборами и рассчитать им цены".
После указания "набора" в каком либо документе изменение его состава невозможно. Следует создавать другие наборы с другим составом. Наборы могут быть вложенными, как элементы другого набора. Но это может быть сложно для учета и визуализации.
Для того, чтобы наборы выделялись в учете, рекомендуется в их наименовании указывать на то, что это составной элемент.
Так как наборы имею независимую цену, показатели валовой прибыли для его компонент будут несколько искажаться. В сумме показатели валовой прибыли будут рассчитаны верно, но вот для отдельных компонент набора могут существенно исказиться.
Для анализа продаж "наборов" пользуйтесь вариантом отчета "Продажи" для наборов.

Отличие набора от комплектации: При реализации набора списываются вся номенклатура его состава. При комплектации списываются материалы и формируется на складе новый экземпляр продукта. Который хранится и числится отдельно, отдельно реализуется, но может быть демонтирован.

Номенклатура может вестись по Сериям. Это значит что каждую единицу номенклатуры будет сопровождать один элемент ее серии. Серию можно установить обязательной для выбора. В такой случае невозможно будет указывать количество товара. Он будет приходоваться, перемещаться и расходоваться по-серийно.

Предусмотрены "полный" и "простой" режимы учета серий товара.

В полном учете серию следует указывать во время всех событий - при поступлении, перемещении, реализации номенклатуры.

В "простом режиме" серию достаточно указать только при приобретении номенклатуры (потом использовать поиск). Таким образом можно только обнаружить где и как приобреталась данная серия.

Серии формируются на этапе приходного документа. Каждый серийный номер может быть оформлен собственным Штрих-кодом и кодом производителя.

Если номенклатура уже использовалась, назначить ее серийной нельзя. Следует создавать новую номенклатуру.

Условия серийного учета могут быть также продиктованы группой номенклатуры или номенклатурной группой.

Невозможно использовать серии для номенклатуры с параметром "Не отслеживать остаток".

Средняя цена приобретений это вся сумма стоимости приобретений поделенная на все количество приобретения. используется как оценка для расчета валовой прибыли в отсутствии партионного учета.

Для группы номенклатуры может быть установлен Процент комиссионного вознаграждения для комиссионеров.

Партионный учет в конфигурации не предусмотрен. Реализован учет серий. Этого остаточно для управленческого учета персональной валовой прибыли отдельныхх серий (партий) товара.

Используется средневзвешенный учете текущего остатка. В этом режиме валовая прибыль остаётся понятием управленческого характера, а не считается для регламентированной (установленной государством отчетности).

Да, конечно товары купленные по разному курсу, дают "скачки" в показателях валовой прибыли в национальной валюте. Но с другой стороны, может следует валютой учёта взять доллар (или евро)? Тогда скачков не будет.

Скачков "стираются" и при хороших оборотах и не слишком то и заметны. В программе можно прямо указать продажу при возврате (и поступление при возврате поставщику), чтобы избежать ошибок.
