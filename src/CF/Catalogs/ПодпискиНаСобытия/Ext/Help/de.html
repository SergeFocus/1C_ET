Directory-Abonnements, um Ereignisse

Verzeichnis können Sie festlegen, komplexe Bedingungen zu abonnieren, um Ereignisse mit Datenbankobjekten:

  • Aufzeichnungselement-Verzeichnis oder ein Dokument;
  • Besitz eines Dokuments;
  • Markierung auf der Abtrennung des Mitglieds Handbuch oder Dokument;
  • Entdeckung (oder Versuch, eine neue Bekanntschaft mit einem vorhandenen Verzeichnis oder ein Dokumentelement zu erstellen);
  • Schließen.

Rechtliches Abonnement kann auf alle Objekte des gleichen Typs, oder bestimmte Datenbankobjekte gelten.

Im Programm können Sie Bedingungen setzen die solche Sprache 1C, die die Notwendigkeit, auf ein Ereignis reagieren zu klären wird. Sie können auf das Objekt durch beziehen. "SsylkaNaObekt".

Beispielsweise die Bedingung, SsylkaNaObekt.Proveden = TRUE

Folgende Methoden der Reaktion auf das Ereignis:

  • Benutzer melden
  • Nehmen Sie die Log-Datei
  • Senden von E-
  • Führen Sie das Programm

Es kann der beabsichtigten Empfänger von Nachrichten von Benutzer.

Eigenschaften:

Beim Schreiben eines Programms sollte berücksichtigt werden,

Für ein laufendes Programm Objektverweis Abonnement (für den Code) aufgerufen wird: Quelle (aber Typ-Link)

Wenn das Programm mit dem Ziel durchgeführt, sein Name: Quelle (Objekttyp)

Während der Verarbeitung Software-Abonnement (Reaktion) alle sonstigen Abonnement ab. Das heißt zum Beispiel, wenn die Reaktion des Abonnements - die Umsetzung des Codes ein neues Produkt zu schaffen, die Ereignisabonnements für ein neues Produkt wird nicht aufgerufen. Um diese Einschränkung zu umgehen, müssen Sie beginnen, diesen Code zu schreiben: ParametrySeansa.VypolnenieKodaZablokirovano = FALSE;
