Le sottoscrizioni Directory agli eventi

Directory consente di specificare le condizioni complesse iscriversi agli eventi con oggetti di database:

  • registrazione directory elemento o documento;
  • possesso di un documento;
  • segno sulla rimozione del manuale utente o di un documento;
  • scoperta (o tentativo di creare una nuova conoscenza con una directory esistente o un elemento del documento);
  • chiusura.

Modalità di sottoscrizione possono essere applicate a tutti gli oggetti dello stesso tipo, o oggetti di database specifici.

Nel programma, è possibile impostare le condizioni di tale 1C lingua condizionale, che chiarirà la necessità di reagire a un evento. È possibile fare riferimento all'oggetto attraverso. "SsylkaNaObekt".

Ad esempio la condizione, SsylkaNaObekt.Proveden = TRUE

I seguenti metodi della reazione all'evento:

  • Segnala utente
  • Registrare il file di registro
  • L'invio di e-mail
  • Eseguire il programma

Ci può essere il destinatario del messaggio per gli utenti.

Caratteristiche:

Quando si scrive un programma dovrebbe tener conto

Per un abbonamento di riferimento oggetto di programma in esecuzione (per il codice) si chiama: Fonte (ma tipo di collegamento)

Se il programma viene eseguito con l'oggetto, il suo nome: Fonte (tipo di oggetto)

Durante la lavorazione sottoscrizione software (reazione) ogni altra iscrizione via. Cioè, per esempio, se la reazione della sottoscrizione - l'attuazione del codice per creare un nuovo prodotto, non saranno chiamati abbonamenti di evento per un nuovo prodotto. Per ovviare a questa limitazione, è necessario iniziare a scrivere questo codice: ParametrySeansa.VypolnenieKodaZablokirovano = FALSE;
