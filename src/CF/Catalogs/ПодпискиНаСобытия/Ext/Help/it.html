Eventi di sottoscrizione Directory

Directory consente di definire condizioni complesse iscriversi agli eventi con oggetti di database:

  • directory di registrazione elemento o documento;
  • possesso di un documento;
  • contrassegno libro rimozione o elemento del documento;
  • scoperta (o tentativo di creare una nuova conoscenza con una directory esistente o elemento del documento);
  • chiusura.

Condizioni di Sottoscrizione possono applicarsi a tutti gli oggetti dello stesso tipo, o oggetti di database specifici.

In termini di programma, è possibile specificare una lingua 1C condizionale, che chiarirà la necessità di reagire agli eventi.È possibile accedere all'oggetto attraverso. "SsylkaNaObekt

Ad esempio la condizione,SsylkaNaObekt.Proveden = TRUE

I seguenti modi per rispondere all'evento:

  • Il messaggio all'utente
  • Scrivendo nel file Journal
  • L'invio di e-mail
  • Eseguire il programma

destinatari dei messaggi dal numero di utenti possono essere impostati.

Caratteristiche:

Quando si scrive un programma dovrebbe tener conto

Per collegare il programma in esecuzione su un oggetto di sottoscrizione (per il codice) è chiamato:Sorgente (ma il tipo di link)

Se il programma viene eseguito con l'oggetto, è il nome:Fonte (tipo Object)

Durante la sottoscrizione (reazione) tutti gli altri programmi di sottoscrizione siano spenti durante la lavorazione.Cioè, per esempio, se la risposta alla sottoscrizione - l'attuazione del codice per creare un nuovo prodotto, non verrà chiamato abbonamenti di evento per un nuovo prodotto.Per ovviare a questa limitazione, è necessario avviare un programma del genere, inserire il codice:ParametrySeansa.VypolnenieKodaZablokirovano = FALSE;
