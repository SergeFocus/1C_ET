이벤트 디렉토리 구독

디렉토리는 복잡한 조건이 데이터베이스 개체와 이벤트를 구독 지정할 수 있습니다 :

  • 요소 디렉토리 또는 문서를 기록하는 단계;
  • 문서를 들고;
  • 회원 수첩 또는 문서의 제거에 표시;
  • 발견 (또는 기존 디렉토리 또는 문서의 요소와 새로운 지인을 만들려고)
  • 폐쇄.

가입 조건은 동일한 유형, 또는 특정 데이터베이스 개체의 모든 개체에 적용 할 수 있습니다.

프로그램에서는 조건에게 이벤트에 반응 할 필요성을 명확히 것 같은 조건부 언어 (1C)를 설정할 수있다. 당신은을 통해 객체를 참조 할 수 있습니다. "SsylkaNaObekt".

예를 들어, 상태, SsylkaNaObekt.Proveden = TRUE

이벤트에 대한 반응의 다음과 같은 방법 :

  • 보고서 사용자
  • 로그 파일을 녹음
  • 이메일 보내기
  • 프로그램을 실행

사용자의 메시지의 수신자가있을 수 있습니다.

특징 :

고려해야하는 프로그램을 작성할 때

프로그램 오브젝트를 실행하면, 그 이름 : 소스 (개체 유형)

소프트웨어 등록 (반응) 떨어져 다른 모든 구독을 처리하는 동안. 가입의 경우 즉, 반응은, 예를 들면, - 부호의 구현이 새로운 제품을 만들기 위해, 새로운 제품에 대한 이벤트 구독 호출되지 않을 것이다. 이 문제를 해결하려면이 코드를 작성하기 시작해야합니다 ParametrySeansa.VypolnenieKodaZablokirovano = FALSE;
