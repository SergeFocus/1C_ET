Directory abonēšana notikumiem

Directory ļauj norādīt sarežģīti apstākļi abonēt notikumiem ar datu bāzes objektus:

  • ierakstīšanas elements direktoriju vai dokumentu;
  • turot dokumentu;
  • atzīme par izņemšanas loceklis rokasgrāmatā vai dokumentu;
  • atklāšana (vai mēģinājums radīt jaunu pazīšanos ar esošu direktorijā vai dokumenta elementu);
  • slēgšana.

Noteikumi parakstīšanās var piemērot visiem objektiem tā paša veida, vai īpašas datu bāzes objektus.

Programmā, jūs varat iestatīt nosacījumus, piemēram nosacītu valodas 1C, kas precizētu nepieciešamību reaģēt uz notikumu. Jūs varat atsaukties uz objektu, izmantojot ".SsylkaNaObekt".

Piemēram nosacījums, SsylkaNaObekt.Proveden = TRUE

Šādas metodes reakcija uz notikuma:

  • Ziņojums lietotājs
  • Reģistrēt log failu
  • Sūtot e-pastu
  • Palaist programmu

Tur var būt iecerētais saņēmējs ziņojumus no lietotājiem.

Features:

Rakstot programmu, būtu jāņem vērā

Par darbības programmu objekts atsauces abonementu (par kods) sauc: Source (bet tips Link)

Ja programma ir izpildīta ar objektu, tā nosaukums: Source (Object tips)

Laikā apstrādes programmatūras abonementu (reakcija) visus citus abonēšanas off. Tas ir, piemēram, ja reakcija abonēšanas - īstenošana kodu, lai izveidotu jaunu produktu, notikums abonēšana par jaunu produktu, netiek pieprasīta. Lai apietu šo ierobežojumu, jums ir nepieciešams, lai sāktu rakstīt šo kodu: ParametrySeansa.VypolnenieKodaZablokirovano = FALSE;
