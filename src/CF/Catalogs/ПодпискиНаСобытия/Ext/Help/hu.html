Directory Előfizetői események

Könyvtár lehetővé teszi, hogy meghatározza az összetett feltételek feliratkozás események adatbázis-objektumok:

  • felvétel eleme könyvtár vagy dokumentum;
  • aki olyan dokumentumot;
  • jel megszűnik a tag kézikönyv vagy dokumentumot;
  • felfedezése (vagy kísérel meg létrehozni egy új ismeretség egy létező könyvtár, vagy egy dokumentum elem);
  • zárás.

Feltételek előfizetés lehet alkalmazni, hogy minden tárgy azonos típusú, vagy konkrét adatbázis-objektumokat.

A programban beállíthatja körülmények között, például a feltételes nyelven 1C, amely tisztázza, hogy szükség van, hogy reagáljon egy eseményt. Akkor nézze meg az objektumot át. "SsylkaNaObekt".

Például a feltétellel, SsylkaNaObekt.Proveden = TRUE

A következő módszerek a reakció esetén:

  • Felhasználó bejelentése
  • Jegyezze fel a log fájl
  • Küldés e-mailben
  • A program futtatásához

Lehet, hogy a címzett az üzenetet a felhasználók.

Jellemzők:

Amikor egy olyan program írásával figyelembe kell venni

Egy futó program objektum referencia előfizetés (a kód) nevezzük: Forrás (de típus Link)

Ha a program végrehajtása azzal a céllal, neve: Forrás (légvonalban)

A feldolgozás során szoftverelőfizetési (reakció) az összes többi előfizetéses ki. Azaz, ha például a reakció az előfizetés - a végrehajtás a kódot egy új termék, a rendezvény előfizetések egy új termék nem nevezhető. Kerülő korlátozás, meg kell kezdeni, hogy ezt a kódot írod: ParametrySeansa.VypolnenieKodaZablokirovano = false;
