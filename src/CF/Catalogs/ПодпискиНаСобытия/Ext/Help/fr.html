Abonnements Directory à des événements

Répertoire vous permet de spécifier des conditions complexes abonner à des événements avec des objets de base de données:

  • enregistrement répertoire de l'élément ou document;
  • possession d'un document;
  • marque sur le retrait du manuel de membre ou d'un document;
  • découverte (ou tentative de créer une nouvelle connaissance avec un répertoire existant ou un élément de document);
  • fermeture.

Modalités de souscription peuvent appliquer à tous les objets du même type, ou des objets de base de données spécifiques.

Dans le programme, vous pouvez définir des conditions telles 1C de langage conditionnel, qui permettra de clarifier la nécessité de réagir à un événement. Vous pouvez vous référer à l'objet à travers. "SsylkaNaObekt».

Par exemple, l'état, SsylkaNaObekt.Proveden = TRUE

Les procédés de la réaction à l'événement suivant:

  • Signaler utilisateur
  • Enregistrez le fichier journal
  • Envoi email
  • Exécutez le programme

Il peut être le destinataire de messages d'utilisateurs.

Caractéristiques:

Lors de l'écriture d'un programme devrait prendre en compte

Pour un abonnement de référence d'objet de programme en cours (pour le code) est appelé: Source (mais de type Lien)

Si le programme est exécuté avec l'objet, son nom: Source (type d'objet)

Pendant le traitement, abonnement logiciel (de réaction) tous les autres hors de souscription. Cela est, par exemple, si la réaction de l'abonnement - la mise en œuvre du code pour créer un nouveau produit, les abonnements aux événements pour un nouveau produit ne sera pas appelé. Pour contourner cette limitation, vous devez commencer à écrire ce code: ParametrySeansa.VypolnenieKodaZablokirovano = FALSE;
