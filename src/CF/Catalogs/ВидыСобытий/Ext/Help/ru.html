Справочник видов событий

Должен быть включен учет событий (CRM)  в настройках.

Справоник позволяет создавать шаблоны для документов регистрации событий. Такой шаблон подробно описывает событие и задает ему первоначальные значения.

Справочник позволяет задать свои виды событий с уникальным набором условий.

Для примера вид события "телефонный звонок клиенту" указывает на то, что:

  • Задана тема (тема звонка)
  • Используется телефон (телефонный номер клиента)
  • Участвует клиент (ссылка на элемент справочника клиентов)
  • Участвует сотрудник (звонящий сотрудник нашего предприятия)
  • Указывается дата проведения (время звонка)
  • Длительность (длительность звонка)
  • Подводится результат (например, записывается ответ клиента на заданную тему)

И т.д. и тому подобное.

На закладках "Шаблон нового события" и в "Таблицы шаблона" можно сразу внести готовые значения для событий этого вида.

При регистрации новых событий с этим видом будут задействованные (видимы пользователю) блоки и заполняет предопределенными данными.

Так, например все события вида "телефонный звонок клиенту от Иванова" с установленным сотрудником Иванов, будут сразу по умолчанию зарегистрированы на него.
