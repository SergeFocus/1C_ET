Арыенцір Кліенты

Прызначаны для падачы кліентаў. Выкарыстоўвайце, калі у тым ліку інтэграцыю кліентаў,

Забяспечвае наканаваных кліента "Прыватны прадпрымальнік" для рэгістрацыі некамерцыйных-асабістых дадзеных аб продажах. Гэта значыць, тыя, для якіх няма неабходнасці адсочваць запазычанасці. Гэта могуць быць перайменаваны.

Каталог іерархія. Гэта значыць, вы можаце групаваць кліентаў.

Рэквізіты:

Тып Кошт (калі ўключана posistema Складаны механізм, улічваючы кошт) вызначае тып цэн па змаўчанні ў рэалізацыя Стан кліента. Ўзровень цэн можа быць прызначаны той жа для ўсіх груп кліентаў.

Плошчу (калі у тым ліку бухгалтарскі ўлік Рэгіёны) вызначае месцазнаходжанне кліента. На гэтай аснове, вы можаце групаваць кліентаў у справаздачах.

Працэнтныя скідкі - (калі яны ўключаныя Бухгалтарскія зніжкі) устанаўлівае персанальную зніжку для кліента. Калі ж працэнт, то продаж намэнклятураУ карты, якія таксама паказвае стаўку дыскантавання, праграма выбірае самыя высокія працэнтныя зніжкі. Працэнт скідкі могуць быць прызначаныя групам кліентаў.

Адрас,Тэлефон,ОКПО,Прававы статус,Электронная пошта, дэталі безнаяўнай аплаты - Дадатковая інфармацыя пра Заказчыка.

Фондавы кліент Калі падсістэма "Улік запасаў", дазваляе паказаць сховішча па змаўчанні для кліента. У гэтым выпадку, выбар дакументаў Кліента, склад будзе ўсталяваны аўтаматычна.

Формула тэкставыя пазнакі (калі выкарыстоўваць формулу) Вызначае унікальную формулу для фарміравання тэксту на этыкетцы пры друку этыкетак для кліента. Такія формулы захоўваюцца ў Арыенцір формулы, Напрыклад, калі дакумент "паступленне тавараў" Усталюйце кліент з яго формулай тэкставых пазнак, калі вы называеце Падпраграма друку этыкетак прынтэр этыкетка з тэкстам, спароджанай гэтай формулы. Калі формула тэкст этыкеткі кліент не запоўнена, але б ўліку склады і формула для поўнага Склад Рэалізацыя, ён будзе выкарыстоўвацца. Апошняе, што з'яўляецца выкарыстанне асноўную формулу для фарміравання тэксту этыкеткі Налады праграмы, Калі гэта не запоўнена, а затым тэкст пазнакі будзе роўная тэксце Штрых намэнклятура,

Прыклад формула тэкставыя пазнакі:

SokrLP (Int (кошт) + 1) + "" + Лео (Nomenklatura.NomenklaturnayaGruppa.Kod, 2) + Лео (Nomenklatura.Proizvoditel.Kod, 2) + "х" + Фармат (ПДВ, "CC = 9; CHDTS = 0 ")

Формулы павінны быць напісаны на мове праграмавання 1С.

Імя для друку - Тэкст, які будзе адлюстроўвацца ў "Кліент" пры друку дакументаў з праграмы. Гэты тэкст можа адрознівацца ад звычайнага імя кліента. Напрыклад, назва Кліента ў каталогу - Ветразі, ТАА, і яго імя для друку - ТАА «Ветразь», Адэсе, на Малой Арнаутском 9.

Дадому Валюта (калі яны ўключаныя Ўлік валюты) усталёўваецца для дакументаў ў выбары кліента аўтаматычна ўсталёўвае пэўны валюту.

Асобная персанал (калі яны ўключаныя разлік заработнай платы) можа быць кліентам. Такія кліенты генеруюцца аўтаматычна пры ўводзе інфармацыі супрацоўнікі, Гэта неабходна для таго, каб даць супрацоўнікам зніжкі на куплю тавараў і ўлічваць разлікі з заказчыкам на заработнай платы супрацоўнік.

Для групы кліентаў, вы можаце ўсталяваць працэнт камісійных збораў. Ён будзе працаваць, калі вы не выкарыстоўваеце працэнт Пагадненне з Кліентам.

Звярніце ўвагу, што калі праграма вядзе ўлік кліентаў, але не прыводзіць рахунак грошы, Пры аналізе даклад «Дэбіторская запазычанасць" карціна будзе выглядаць аднабока. Як, калі тавар быў рэалізаваны, але не аплачваюцца кліентам, так як няма магчымасці зарэгістравацца ў праграме Грашовыя паступленні ад заказчыкаў пры адключаным зарэгістравана грошай.

Пры неабходнасці, каб кампенсаваць доўг паміж кліентам і / або Пастаўшчыкі прымаючы пад увагу квітанцыя / адток грошай, вы павінны выкарыстоўваць спецыяльную форму карэкціроўкі "Ўзаемазалік запазычанасці контрагентаў".

Падрадчыкі - агульная назва для кліентаў і пастаўшчыкоў.
