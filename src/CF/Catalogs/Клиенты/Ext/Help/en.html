Directory "CLIENTS"

Designed for client files. Used if included accounting clients.

Provides predefined client "Private person" for the recording of data on non-personal sales. Ie, those for which there is no need to keep track of debt. You can rename it.

Directory hierarchy. That is, clients can be grouped.

Details:

View Price (Unless Included posistema Complex mechanisms accounting prices) determines the kind of price, by default implementation goods to the customer. View Price can assign the same for the whole group of customers.

Region (Unless included accounting regions) determines the location of the Customer. According to this feature, you can group clients in the reports.

Discount - (if enabled registration discounts) establishes personal discount percentage for the Client. If this percentage is set, then the sale NomenclatureIn cards which also shows the percentage discount, the program chooses the largest percentage discount. Discount percentage can be assigned to a group of clients.

Address,Phone,Of Enterprise,Legal status,E-mail, details of cashless payment - Additional information about the Customer.

Warehouse Client When the subsystem "Accounting Warehouse" allows you to specify the default warehouse for the Client. In this case, the choice of the Customer's documents, this warehouse will be installed automatically.

Formula text labels (Unless use the formula) Defines a unique formula of formation of the text on the label when printing labels for the client. Such formulas are stored in handbook formulas. For example, if document "flow of goods" Set with his client by text labels, calling subroutine label printing the printer will label with text generated by this formula. If Formula text labels on the client is not filled, but used accounting Warehouse and a formula for the full Warehouse implementation, it will be used. The last step is to use the basic formula of the formation of the text labels program settings. If it has not been filled and then the text labels will be equal to the text Barcode Nomenclature.

Example formula label text:

SokrLP (Int (Price) + 1) + "" + Leo (Номенклатура.НоменклатурнаяГруппа.Код, 2) + left (Номенклатура.Производитель.Код, 2) + "x" + Format (price, "CC = 9; CHDTS = 0 ")

Formulas should be written in language programming 1C.

Name Print - The text that will be displayed in the "Customer" when printing documents from the program. This text may differ from the usual name of the client. For example, Client name in the directory - Sail, LLC, and his name for print - "Parus", Odessa, Malaya Arnautskaya 9.

Summary Currency (if enabled Accounting rates) is set to in the documents when choosing this Client automatically sets certain currency.

Net employee (if enabled payroll accounting) can serve the client. Such customers are created automatically when you enter information employees. This is necessary in order to give employees a discount on the purchase of goods and take into account settlements with the customer at payroll employee.

For a group of clients, you can specify percentage commission. He will work unless you use a percentage of Treaty with the Client.
