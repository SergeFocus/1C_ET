Справочник "КЛИЕНТЫ"

Предназначен для картотеки клиентов. Используется, если включен учет по клиентам.

Предусмотрен предопределенный Клиент "Частное лицо" для регистрации данных о неперсонифицированных продажах. То есть таких, для которых нет необходимости отслеживать задолженность. Его можно переименовать.

Справочник иерархический. То есть Клиентов можно группировать.

Реквизиты:

Вид Цен (если включена посистема Сложного Механизма учета Цен) определяет вид цен по-умолчанию при реализации товара клиенту. Вид Цен можно назначать так же для всей группы клиентов.

Регион (если включен учет регионов) определяет расположение Клиента. По этому признаку можно группировать Клиентов в отчетах.

Процент скидки - (если включен учет скидок) устанавливает персональный процент скидки для данного Клиента. Если установлен такой процент, то при продаже номенклатуры, в карточке которой также указан процент скидки, программа выбирает самый большой процент скидки. Процент скидки можно назначить для группы Клиентов.

Адрес, Телефон, ОКПО, Правовой статус, Электронная почта, Реквизиты безналичного расчета - дополнительные данные о Клиенте.

Склад клиента при включенной подсистеме  "Учета по Складам" позволяет указать склад по-умолчанию для Клиента. В этом случае, при выборе Клиента в документах, данный склад будет установлен автоматически.

Формула текста этикетки (если используются формулы) определяет уникальную формулу формирования текста на этикетке при печати этикетки для данного клиента. Такие формулы хранятся в справочнике формул. Например, если в документе "Расход товара" указан Клиент со своей формулой текста этикетки, то при вызове подпрограммы печати этикеток на принтер пойдет этикетка с текстом, сформированным по этой формуле. Если Формула текста этикетки у клиента не заполнена, но используется учет по складам и такая формула заполнена для Склада реализации, то будет использована она. В последнюю очередь будет использована Основная формула формирования текста этикеток из настроек программы. Если не заполнена и она, то текст этикетки будет равен тексту Штрих-кода номенклатуры.

Пример формулы текста этикетки:

СокрЛП(Цел(Цена) + 1) + " " + Лев(Номенклатура.НоменклатурнаяГруппа.Код, 2) + Лев(Номенклатура.Производитель.Код, 2) + "x" + Формат(Цена, "ЧЦ=9; ЧДЦ=0")

Формулы должны быть написаны языком программирования 1С.

Наименование для печати - текст, который будет выводиться в поле "Клиент" при печати документов из программы. Этот текст может отличаться от обыкновенного наименования Клиента. Например, наименование Клиента в справочнике - Парус, ООО , а его наименование для печати - ООО "Парус", Одесса, ул.Малая Арнаутская, 9.

Основная Валюта (если включен Учет валют) устанавливается для того, чтобы в документах при выборе этого Клиента автоматически устанавливалась определенная валюта.

Собственный сотрудник (если включен учет заработной платы) может выступать Клиентом. Такие Клиенты создаются автоматически при вводе информации по сотрудникам. Это необходимо для того, чтобы дать сотрудникам скидки при приобретении товаров и учесть взаиморасчеты с таким клиентом при расчете заработной платы сотрудника.

Для группы Клиентов можно задать Процент Комиссионного Вознаграждения. Он будет действовать, если не используется процент из Договора с Клиентом.

 Учтите, что если в программе вести учет по клиентам, но не вести учет денег, то при анализе отчета "Расчетам с клиентами" картина будет выглядеть односторонне. Как будто товар реализовывался, но не оплачивался клиентами, так как нет возможности зарегистрировать в программе поступление денег от покупателя при отключенном учете денег.

Когда нужно произвести взаимозачет долга между Клиентами и/или Поставщиками с учетом поступления/выбытия денег, следует воспользоваться специальной формой корректировки "Взаимозачет долга контрагентов".

Контрагенты - общее наименование для Клиентов и Поставщиков.
