Довідник Клієнти

Призначений для картотеки клієнтів. Використовується, якщо включений облік по клієнтах.

Передбачений зумовлений Клієнт "Приватна особа" для реєстрації даних про неперсоніфікованих продажах. Тобто таких, для яких немає необхідності відстежувати заборгованість. Його можна перейменувати.

Довідник ієрархічний. Тобто Клієнтів можна групувати.

Реквізити:

Вид Цін (якщо включена посістема Складного Механізму обліку Цін) визначає вид цін по-замовчуванням при реалізації товару клієнту. Вид Цін можна призначати так само для всієї групи клієнтів.

Регіон (якщо включений облік регіонів) визначає розташування Клієнта. За цією ознакою можна групувати Клієнтів у звітах.

Відсоток знижки - (якщо включений облік знижок) встановлює персональний відсоток знижки для даного Клієнта. Якщо встановлено такий відсоток, то при продажу номенклатури, В картці якої також вказаний відсоток знижки, програма вибирає найбільший відсоток знижки. Відсоток знижки можна призначити для групи Клієнтів.

Адреса,Телефон,ОКПО,Правовий статус,Електронна пошта, Реквізити безготівкового розрахунку - Додаткові дані про Клієнта.

Склад клієнта при включеній підсистемі "Облік за складами" дозволяє вказати склад по-замовчуванню для Клієнта. У цьому випадку, при виборі Клієнта в документах, даний склад буде встановлено автоматично.

Формула тексту етикетки (якщо використовуються формули) Визначає унікальну формулу формування тексту на етикетці при друку етикетки для даного клієнта. Такі формули зберігаються в довіднику формул. Наприклад, якщо в документі "Витрата товару" вказано Клієнт зі своєю формулою тексту етикетки, то при виклику підпрограми друку етикеток на принтер піде етикетка з текстом, сформованим за цією формулою. Якщо Формула тексту етикетки у клієнта не заповнена, але використовується облік за складами і така формула заповнена для Складу реалізації, то буде використана вона. В останню чергу буде використана Основна формула формування тексту етикеток з налаштувань програми. Якщо не заповнена і вона, то текст етикетки буде дорівнює тексту Штрих-коду номенклатури.

Приклад формули тексту етикетки:

СокрЛП (Цел (Ціна) + 1) + "" + Лев (Номенклатура.НоменклатурнаяГруппа.Код, 2) + Лев (Номенклатура.Проізводітель.Код, 2) + "x" + Формат (Ціна, "ЧЦ = 9; ЧДЦ = 0 ")

Формули повинні бути написані мовою програмування 1С.

Найменування для друку - Текст, який буде виводитися в полі "Клієнт" при друку документів з програми. Цей текст може відрізнятися від звичайного найменування Клієнта. Наприклад, найменування Клієнта в довіднику - Парус, ТОВ, а його найменування для друку - ТОВ "Парус", Одеса, вул.Мала Арнаутська, 9.

Основна Валюта (якщо включений Облік валют) встановлюється для того, щоб у документах при виборі цього Клієнта автоматично встановлювалася певна валюта.

Власний співробітник (якщо включений облік заробітної плати) може виступати Клієнтом. Такі Клієнти створюються автоматично при введенні інформації по співробітникам. Це необхідно для того, щоб дати співробітникам знижки при придбанні товарів і врахувати взаєморозрахунки з таким клієнтом при розрахунку заробітної плати співробітника.

Для групи Клієнтів можна задати Відсоток комісійної винагороди. Він буде діяти, якщо не використовується відсоток з Договору з Клієнтом.

Врахуйте, що якщо в програмі вести облік по клієнтам, але не вести облік грошей, То при аналізі звіту "Розрахунки з клієнтами" картина виглядатиме однобічно. Неначе товар реалізовувався, але не оплачувався клієнтами, так як немає можливості зареєструвати в програмі надходження грошей від покупця при відключеному обліку грошей.

Коли потрібно зробити взаємозалік боргу між Клієнтами та / або Постачальниками з урахуванням надходження / вибуття грошей, слід скористатися спеціальною формою коригування "Взаємозалік боргу контрагентів".
