Reference მომხმარებელთა

განკუთვნილია შეტანის კლიენტებს. გამოიყენეთ თუ მათ შორის ინტეგრაციის მომხმარებელს.

უზრუნველყოფს განსაზღვრული კლიენტი "კერძო პირი" რეგისტრაციის არასამთავრობო პირადი მონაცემების გაყიდვების. რომ არის, იმ რისთვისაც არ არის საჭირო იმისათვის, რომ შევინარჩუნოთ სიმღერა ვალი. ეს შეიძლება დაარქვა.

დირექტორია იერარქიაში. ეს არის ის, შეგიძლიათ ჯგუფის კლიენტებს.

რეკვიზიტები:

ტიპი ფასი (თუ შედის posistema კომპლექსური მექანიზმი გათვალისწინებით ფასი) განსაზღვრავს ტიპის ძირითადად ფასები განხორციელება საქონელი კლიენტს. ფასი ტიპის შეიძლება დაეკისროს ამავე ყველა მომხმარებელს ჯგუფები.

ფართი (თუ მათ შორის საბუღალტრო რეგიონები) განსაზღვრავს ადგილმდებარეობის კლიენტს. ამ დოკუმენტის საფუძველზე, შეგიძლიათ ჯგუფის კლიენტებს ანგარიშები.

პროცენტული ფასდაკლება - (თუ შედის საბუღალტრო ფასდაკლება) ადგენს პირადი ფასდაკლება კლიენტს. თუ იგივე პროცენტული, მაშინ იყიდება ნომენკლატურაIn ბარათი, რომელიც ასევე მიუთითებს ფასდაკლება კურსი, პროგრამა ირჩევს უმაღლესი პროცენტული ფასდაკლება. პროცენტული ფასდაკლება შეიძლება დაეკისროს ჯგუფები კლიენტებს.

მისამართი.ტელეფონი.Okpo.ლეგალური სტატუსი.E-mail დეტალები უნაღდო - დამატებითი ინფორმაცია დამკვეთი.

საფონდო კლიენტს როდესაც ქვესისტემის "საბუღალტრო საფონდო," გაძლევთ მიუთითოთ სტანდარტული შენახვის კლიენტს. ამ შემთხვევაში, არჩევანი კლიენტის დოკუმენტები, საწყობი იქნება დაყენებული ავტომატურად.

ფორმულა ტექსტი ეტიკეტები (თუ გამოვიყენოთ ფორმულა) განსაზღვრავს უნიკალური ფორმულა ფორმირების ტექსტში ეტიკეტზე, როდესაც თქვენ ბეჭდვა ეტიკეტები ამისთვის მომხმარებელს. ასეთი ფორმულები ინახება Reference ფორმულები. მაგალითად, თუ დოკუმენტი "ნაკადის საქონლის" უცნობია კლიენტს მისი ფორმულა ტექსტი ეტიკეტები, როცა რეკავთ subroutine ეტიკეტების საბეჭდი პრინტერი წარწერა ტექსტით მიერ გამომუშავებული ეს ფორმულა. იმ შემთხვევაში, თუ ფორმულა ტექსტი ეტიკეტები კლიენტს არ არის შევსებული, მაგრამ მეორადი საბუღალტრო საწყობები და ფორმულა სრული საწყობი შესრულების, იქნება გამოყენებული. ბოლო რაც არის გამოიყენოს ძირითადი ფორმულა ფორმირების ტექსტი ეტიკეტების პროგრამის პარამეტრების. თუ იგი არ არის შევსებული და შემდეგ text label ტოლფასი იქნება ტექსტი შტრიხ ნომენკლატურა.

მაგალითი ფორმულა ტექსტი ეტიკეტები:

SokrLP (Int (ფასი შეთანხმებით) + 1) + "" + ლომი (Nomenklatura.NomenklaturnayaGruppa.Kod, 2) + ლომი (Nomenklatura.Proizvoditel.Kod, 2) + "x" + ფორმატი (დღგ, "CC = 9; CHDTS = 0 ")

ფორმულები უნდა იყოს დაწერილი პროგრამირების ენა 1C.

სახელი Print - ტექსტი უნდა აისახოს "კლიენტი" ბეჭდვა დოკუმენტები პროგრამა. ეს ტექსტი შეიძლება განსხვავდება ჩვეულებრივი სახელი კლიენტს. მაგალითად, სახელი კლიენტის მომსახურება - Sail, შპს და მისი სახელი ბეჭდვა - შპს "Sail", ოდესის, მალაია Arnautskaya 9.

მთავარი ვალუტა (თუ შედის საბუღალტრო ვალუტაში) არის დოკუმენტების შერჩევა კლიენტს ავტომატურად ადგენს გარკვეული ვალუტაში.

თანამშრომლები (თუ შედის სახელფასო საბუღალტრო) შეიძლება კლიენტს. ასეთი კლიენტები გენერირდება ავტომატურად, როდესაც შედიხარ ინფორმაცია თანამშრომლები. ეს აუცილებელია, რათა თანამშრომლებს ფასდაკლებით შეძენის საქონელი და გაითვალისწინოს დასახლებულ მომხმარებელს სახელფასო თანამშრომელი.

ჯგუფის კლიენტებს, როდესაც თქვენ დააჭერთ პროცენტული ტარიფებში. იგი იმუშავებს თუ თქვენ იყენებთ პროცენტული შეთანხმება კლიენტს.

გაითვალისწინეთ, რომ თუ პროგრამა შენარჩუნება ჩანაწერი მომხმარებლებთან, მაგრამ არ გამოიწვევს ანგარიშის ფული, იმ ანალიზი ანგარიშში, "დებიტორული" სურათი გამოიყურება ცალმხრივი. თითქოს საქონლის განხორციელდა, მაგრამ არ იხდის კლიენტი, რადგან არ არსებობს შესაძლებლობა, რომ დარეგისტრირდეთ პროგრამა თანხის ქვითრები მომხმარებელს ერთად გათიშული რეგისტრირებული ფული.

როდესაც ეს აუცილებელია იმისთვის, რომ მოხდეს ვალის კლიენტს შორის ან / და პროვაიდერები იმის გათვალისწინებით, რომ მიღების / გადინების ფული, თქვენ უნდა გამოიყენოთ სპეციალური ფორმა მომართვის "გადახურვა ვალის კონტრაგენტების".
