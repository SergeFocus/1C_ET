Для додаткових обробок або звітів, Призначених для роботи з конкретними об'єктами, можна визначити склад об'єктів, в яких будуть доступні команди обробки або звіту (команда Призначення обробки).

Команди обробок або звітів можуть бути доступні зі списку обраних об'єктів, якщо встановлено прапор Використовувати для форм списку і з форми самого об'єкта, якщо встановлено прапор Використовувати для форм об'єкта.

Для обробок і звітів, які викликаються з командного інтерфейсу (Не прикріплюються до об'єктів конфігурації):

  • необхідно вказати розділи інтерфейсу, у яких команди обробок будуть доступні користувачеві через команди Додаткові звіти і Додаткові обробки .
  • для кожної команди можна вказати користувачів, для яких команда буде доступна за замовчуванням (команда Швидкий доступ до команди), А також розклад, за яким буде запускатися регламентне завдання для виконання цієї команди (команда Вказати розклад команди).

За допомогою реквізиту Публікація можна включити або відключити доступність обробки або звіту для користувача. Реквізит може приймати наступні значення:

  • використовується - Обробка доступна для використання;
  • не використовується - Обробка не доступна для використання;
  • режим налагодження - Обробка доступна для використання тільки користувачеві з роллю "Додавання і зміна додаткових обробок" або роллю "Повні права".

За допомогою команд завантажити файл і Вивантажити файл можна оновити обробку в інформаційній базі, а також вивантажити її з бази а у файл.

Як правило, більшість обробок повинні виконуватися в безпечному режимі. Це гарантрірует, що вони не містять «ненадійний» програмний код, який виконується на сервері. Однак деякі обробки можуть вимагати виконання в небезпечному режимі. Такі обробки слід завантажувати тільки з надійних джерел (постачальників) або перед завантаженням самостійно виконувати їх аудит.
