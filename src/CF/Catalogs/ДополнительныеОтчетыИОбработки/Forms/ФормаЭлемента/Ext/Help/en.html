To Additional treatments or reportsDesigned to work with concrete objects, you can determine the composition of objects that will be available processing commands or report (the team Appointment processing).

Teams treatments or reports can be accessed from the list of selected objects, if the flag Use the form to list and shape of the object, if the flag Used to form the object.

For treatments and reports that are called from the command line interface (not attached to objects configuration):

  • you must specify the interface section, in which teams of treatments are available to the user via the command Additional reports and Additional processing .
  • for each command you can specify the users for whom the team will be available by default (the team Quick access to the team), And the schedule on which to run scheduled jobs to run this command (command The flight schedules of the team).

Using props Published You can enable or disable access to treatment or report to the user. Props can take the following values:

  • used - Treatment is available for use;
  • not used - Treatment is not available for use;
  • debugging mode - Treatment is available for use only by the user with the role "Adding or changing additional processing" or the role of "full rights".

Using commands Upload file and Upload file can update processing in the information database, and to unload it from the database as a file.

As a rule, the majority of the treatments must be carried out in safe mode. It garantriruet that they do not contain any "untrusted" program code that is executed on the server. However, some treatments may require implementation in an unsafe mode. Such treatments should only download from trusted sources (suppliers) or before downloading to independently carry out their audit.

For more information about safe mode, see. Documentation on 1C: Enterprise.
