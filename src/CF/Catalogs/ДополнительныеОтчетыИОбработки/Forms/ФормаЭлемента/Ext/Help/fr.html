À Des traitements ou des rapports supplémentairesConçu pour fonctionner avec des objets concrets, vous pouvez déterminer la composition des objets qui seront traitement des commandes ou du rapport (l'équipe Traitement de Nomination).

Équipes traitements ou rapports peuvent être consultés à partir de la liste des objets sélectionnés, si le drapeau Utilisez le formulaire à la liste et la forme de l'objet, si le drapeau Utilisé pour former l'objet.

Pour les traitements et les rapports qui sont appelés à partir de l'interface de ligne de commande (non rattaché à la configuration des objets):

  • vous devez spécifier la section d'interface, dans laquelle les équipes de traitements sont disponibles pour l'utilisateur via la commande Des rapports supplémentaires et Un traitement supplémentaire .
  • pour chaque commande, vous pouvez spécifier les utilisateurs pour lesquels l'équipe sera disponible par défaut (l'équipe Accès rapide à l'équipe), Et le calendrier sur lequel exécuter les tâches planifiées pour exécuter cette commande (commande Les horaires de vol de l'équipe).

En utilisant des accessoires Publié Vous pouvez activer ou désactiver l'accès à un traitement ou d'un rapport à l'utilisateur. Props peuvent prendre les valeurs suivantes:

  • utilisé - Le traitement est disponible pour une utilisation;
  • non utilisé - Traitement ne sont pas disponibles pour l'utilisation;
  • mode de débogage - Le traitement est disponible pour être utilisé uniquement par l'utilisateur avec le rôle "Ajout ou modification de traitement supplémentaire» ou le rôle des «pleins droits».

Utilisation des commandes Uploader le fichier et Uploader le fichier peut mettre à jour la base de données de traitement de l'information, et à la décharger de la base de données sous forme de fichier.

En règle générale, la majorité des traitements doit être effectuée en mode sans échec. Il garantriruet qu'ils ne contiennent pas de code de programme "non fiables" qui est exécuté sur le serveur. Cependant, certains traitements peuvent nécessiter la mise en œuvre dans un mode non sécurisé. Ces traitements ne doivent télécharger à partir de sources de confiance (fournisseurs) ou avant de télécharger pour effectuer indépendamment de leur vérification.

Pour plus d'informations sur le mode sans échec, voir. Documentation sur 1C: Enterprise.
