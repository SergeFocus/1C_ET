საწყობები დირექტორია

განკუთვნილია საბუღალტრო ბალანსის და მოძრაობები საქონლის მასშტაბით მრავალი საწყობი.

არსებობს pedopredelennye "მთავარი შენახვის" და "ტრანზიტის". საწყობი ტრანზიტის არ უნდა იქნას გამოყენებული იმ საბუთების მიღების და განხორციელების.

იმისათვის, რომ გამოიყენოთ კატალოგი პროგრამის პარამეტრების ეს უნდა იყოს გამშვები ნიშნის შემდეგ "საბუღალტრო საწყობი. " "ვორჰაუზ" გაძლევთ საშუალებას გამოყოფა საქონლის გადაადგილებისა და ნაშთები და მათი ფიზიკური თუ ვირტუალური ადგილმდებარეობა.

დირექტორია "საწყობების" ნებაყოფლობითია (ეს შეიძლება იყოს ნაპოვნი "დამატებითი Reference ფანჯრის "პარამეტრები ბუღალტერია, proscale").

ფორმულა ტექსტი ეტიკეტები - (თუ იგი მოიცავდა გამოყენება ფორმულები) განსაზღვრავს უნიკალური ფორმულა ფორმირების ტექსტში ეტიკეტზე როდესაც თქვენ ბეჭდვა ეტიკეტები მოცემული საფონდო. ასეთი ფორმულები ინახება Reference ფორმულები.

მაგალითად, თუ დოკუმენტი "მოძრავი საქონელი" როგორც საწყობი მიმღები უცნობია საწყობი მისი ფორმულა tesksta ეტიკეტები, როცა რეკავთ subroutine ეტიკეტების საბეჭდი პრინტერი წარწერა ტექსტით მიერ გამომუშავებული ეს ფორმულა.

იმ შემთხვევაში, თუ ფორმულა ტექსტი ეტიკეტები საათზე დეპოს არ არის შევსებული, იგი გამოიყენებს ძირითადი ფორმულა ფორმირების ტექსტი ეტიკეტების პროგრამის პარამეტრების. თუ იგი არ არის შევსებული და შემდეგ text label ტოლფასი იქნება ტექსტში შტრიხ ნომენკლატურა.

ფორმულა ტექსტი ეტიკეტები მომხმარებელს განყოფილებიდან მას აქვს უპირატესობა ფორმულა საწყობი. ასე რომ, თუ თქვენ ბეჭდვის ეტიკეტები მაგიდა დოკუმენტი მოხმარების საქონელი განხორციელების კლიენტს, პირველ რიგში, უნდა იყოს ჩართული ფორმულა მითითებული დამკვეთის ბარათი.

მაგალითი ფორმულა ტექსტი ეტიკეტები:

SokrLP (Int (Nomenklatura.Tsena) +1) + "" + ლომი (Nomenklatura.NomenklaturnayaGruppa.Kod, 2) + ლომი (Nomenklatura.Proizvoditel.Kod, 2) + "x" + ფორმატი (დღგ, "CC = 9; CHDTS = 0 ")

ფორმულები უნდა იყოს დაწერილი პროგრამირების ენა 1C.

ნახეთ მიხვდა ფასები - (თუ ჩართულია კომპლექსური მექანიზმი საბუღალტრო ფასი) გაძლევთ საშუალებას აკონტროლოთ ხედი განხორციელების ფასი მაღაზია (ავტომატურად შერჩეული, როდესაც ვირჩევთ dokmuente საწყობი).

პასუხისმგებელი ოფიცერი - (თუ შედის სახელფასო საბუღალტრო) A თანამშრომელი, ზე ხელფასის რომელიც შეიძლება uchetst შედეგები ინვენტარი ამ საფონდო. ანგარიში რჩება საქონლის სახის ფასი მობილური ანგარიშში არ არის გათვალისწინებული "პასუხისმგებელი თანამშრომლები."

საწყობის შევსება - თუ მათ შორის საბუღალტრო საწყობი. როდესაც ხორციელდება მაღაზიაში შეიძლება იყოს სიტუაცია, სადაც პროდუქტი არ მაღაზია (გაყიდვაში, "მაღაზიაში"), მაგრამ ის უკან ოთახში. იმისათვის, რომ არ წარმოქმნის პროგრამა სამგზავრო დოკუმენტი ამ პროდუქტის დამხმარე შენახვის საწყობში მაღაზიაში, სანამ მისი განხორციელება, თქვენ შეგიძლიათ მიუთითოთ საწყობი "სათავსო", როგორც საწყობის შევსება ბარათის მაღაზია "მაღაზია". ამის შემდეგ პროგრამის ჩამოწერის საკითხი ამ მაღაზიაში შევსება იმ შემთხვევაში, განხორციელების საფონდო მან უკვე დასრულდა. ჯაჭვის რამდენიმე საწყობი შევსება არ მუშაობს. ეს მექანიზმი მოქმედებს მხოლოდ ქაღალდის მოხმარება საქონელი. ეს მექანიზმი არ ვრცელდება შემადგენელი ელემენტები დადგენილი დასახული. ასეთ შემთხვევაში, მომხმარებელს უნდა წინასწარი რეგისტრაცია საქონლის.

წონა, სიგრძე, სიგანე, სიმაღლე - აღწერა საწყობი.

თუ დადგინდა, რომ ამ საფონდო არ გაიყიდება, ხარჯების განხორციელების დოკუმენტის კლიენტი ამ მაღაზიაში არ იქნება რეგისტრირებული. აუცილებელია, მაგალითად, შენახვა ან შუალედური შენახვის ქორწინება.
