Справочник склады

 Предназначен для ведения учета остатков товара и их движений по нескольким складам.

Предусмотрены педопределенные "основной склад" и "товары в пути". Склад для товара в пути не следует использовать для документов поступления или реализации.

Для использования справочника в настройках программы должна быть установлена галочка напротив пункта "Учет по складам". "Склад" позволяет отдельно учитывать движения и остатки товара по их физическому или виртуальному местонахождению.

Справочник "склады" является дополнительным (его можно найти в разделе "дополнительные справочники окна настройки учета, программы).

Формула текста этикетки - (если включено использование формул) определяет уникальную формулу формирования текста на этикетке при печати этикетки для данного склада. Такие формулы хранятся в справочнике формул.

Например, если в документе "Перемещение товара" в качестве Склада-получателя указан Склад со своей формулой текста этикетки, то при вызове подпрограммы печати этикеток на принтер пойдет этикетка с текстом, сформированным по этой формуле.

Если Формула текста этикетки у Склада не заполнена, то будет использована Основная формула формирования текста этикеток из настроек программы. Если не заполнена и она, то текст этикетки будет равен тексту Штрих-кода номенклатуры.

Формула текста для этикетки карточки Клиента имеет преимущество над формулой склада. Так, если идет печать этикетки для таблицы документа Расход товара при реализации клиенту, то в первую очередь будет задействована формула, указанная в карточке Клиента.

Пример формулы текста этикетки:

СокрЛП(Цел(Номенклатура.Цена) + 1) + " " + Лев(Номенклатура.НоменклатурнаяГруппа.Код, 2) + Лев(Номенклатура.Производитель.Код, 2) + "x" + Формат(Цена, "ЧЦ=9; ЧДЦ=0")

Формулы должны быть написаны языком программирования 1С.

Вид цен Реализации - (если включен сложный механизм учета цен) позволяет управлять видом цен при реализации с это склада (автоматически выбирать в Документе при выборе склада).

Ответственный сотрудник - (если включен учет заработной платы) такой сотрудник, в зарплате которого можно учетсть результаты инвентаризации на этом складе. В отчете по Остаткам Товара в Видах Цен предусмотрен вариант отчета "по ответственным сотрудникам".

Склад Пополнения - если включен учет по складам. При реализации в магазине может возникнуть ситуация, когда товара нет в магазине (на складе "магазин"), но он есть в подсобном помещении. Чтобы не формировать в программе документ перемещения этого товара из подсобного склада в склад магазина перед его реализацией, можно указать склад "подсобного помещения" в качестве Склада Пополнения в карточке склада "магазин". Тогда программа будет списывать товар с этого склада пополнения в том случае, если на складе реализации он уже закончился. Цепочка из нескольких складов пополнения НЕ работает. Этот механизм действует только в документе расхода товара. Этот механизм не действует для элементов состава набора пр реализации набора. В таких случаях пользователю нужно заранее зарегистрировать перемещение товара.

Вес, Длина, Ширина, Высота - описание склада.

Если установлено, что с данного склада продавать нельзя, то расходный документ реализации на Клиента с данного склада зарегистрировать не получится. Это нужно, например для склада брака или промежуточных хранилищ.
