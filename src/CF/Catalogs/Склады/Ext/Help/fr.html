Répertoire entrepôts

Conçu pour les soldes et les mouvements de marchandises à travers de multiples entrepôts comptables.

Il pedopredelennye "la mémoire centrale" et "les marchandises en transit". L'entrepôt pour les marchandises en transit ne devrait pas être utilisé pour la réception de documents ou de mise en œuvre.

Pour utiliser le répertoire les paramètres du programme Il devrait être une coche à côté de «entrepôts comptables. " «Entrepôt» vous permet de séparer la circulation des marchandises et les restes de leur emplacement physique ou virtuel.

Répertoire "entrepôts" est facultative (il peut être trouvé dans la "Référence supplémentaire fenêtre «Paramètres de la comptabilité, ProScale").

Étiquettes de texte de formule - (Si Il comprenait l'utilisation de formules) Définit une formule unique pour la formation du texte sur l'étiquette lorsque vous imprimez des étiquettes pour un stock donné. De telles formules sont stockés dans Formules de référence.

Par exemple, si le document "Déplacement des marchandises" en tant que bénéficiaire d'entrepôt Entrepôt Set avec ses étiquettes formule de tesksta, lorsque vous appelez impression d'étiquettes sous-programme l'imprimante étiquette avec le texte généré par cette formule.

Si les étiquettes de texte de la formule du dépôt ne sont pas remplies, il utilisera la formule de base pour la formation du texte de l'étiquette les paramètres du programme. Si elle est pas rempli, puis l'étiquette de texte sera égal à la nomenclature texte de code à barres.

Étiquettes de texte de formule carte de client Il a un avantage sur la formule entrepôt. Donc, si vous imprimez des étiquettes pour le document de table Biens de consommation la mise en oeuvre du client, la première chose à être associé à une formule spécifiée dans la carte du client.

Exemple Formule étiquettes de texte:

SokrLP (Int (Nomenklatura.Tsena) 1) + "" + Leo (Nomenklatura.NomenklaturnayaGruppa.Kod, 2) + Leo (Nomenklatura.Proizvoditel.Kod, 2) + "x" + Format (TVA, "CC = 9; CHDTS = 0 ")

Formules doivent être rédigés dans un 1C du langage de programmation.

Voir les prix réalisés - (Si activé mécanisme complexe prix comptable) Vous permet de contrôler la vue de la mise en œuvre du prix est de stocker (sélectionné automatiquement lors du choix d'entrepôt de dokmuente).

Agent responsable - (Si comptabilité de la paie inclus) UN employé, à la paie qui peut avoir des résultats uchetst Inventaire sur ce stock. Le rapport sur la Reste de la marchandise dans le Prix types version du rapport est fourni "pour les employés responsables."

Entrepôt reconstitution - Si y compris la comptabilité Entrepôt. Lorsque mis en œuvre dans un magasin peut être une situation où produit pas dans la boutique (en stock, "store"), mais il est dans l'arrière-salle. Afin de ne pas générer le programme document de Voyage ce produit à partir de la mémoire auxiliaire dans un magasin-entrepôt avant sa mise en œuvre, vous pouvez spécifier l'entrepôt «cellier» comme Entrepôt reconstitution dans la carte magasin "boutique". Ensuite, le programme va écrire sur les articles de ce magasin reconstitution dans le cas où la mise en œuvre de stock-il déjà fini. La chaîne de plusieurs reconstitution de l'entrepôt ne fonctionne pas. Ce mécanisme ne fonctionne que dans biens de consommation de papier. Ce mécanisme ne concerne pas les éléments constitutifs énoncés la mise en œuvre de l'ensemble. Dans de tels cas, l'utilisateur doit pré-enregistrer le mouvement des marchandises.

Poids, longueur, largeur, hauteur - Une description de l'entrepôt.

Si l'on détermine que à partir de ce stock ne peut être vendu, Les dépenses sur la mise en œuvre du document client de ce magasin ne sera pas enregistrée. Il est nécessaire, par exemple pour le stockage ou le stockage intermédiaire du mariage.
