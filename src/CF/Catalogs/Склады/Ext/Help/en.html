Warehouses Directory

Designed for accounting balances and movements of goods across multiple warehouses.

There pedopredelennye "main storage" and "goods in transit". The warehouse for goods in transit should not be used for the receipt of documents or implementation.

To use the directory in program settings It should be a check mark next to 'Accounting warehouses. " "Warehouse" allows you to separate the movement of goods and the remains of their physical or virtual location.

Directory "warehouses" is optional (it can be found in the "Additional Reference window "Settings Accounting, proscale").

Formula text labels - (if It included the use of formulas) Defines a unique formula for the formation of the text on the label when you print labels for a given stock. Such formulas are stored in Reference formulas.

For example, if the document "Moving goods" as a warehouse recipient Set Warehouse with its formula tesksta labels, when you call subroutine label printing the printer will label with the text generated by this formula.

If the formula text labels at the Depot is not filled, it will use the basic formula for the formation of the text of the label program settings. If it is not filled and then the text label will be equal to the text Barcode nomenclature.

Formula text labels customer card It has an advantage over formula warehouse. So, if you are printing labels for the table document Consumption goods the implementation of the client, the first thing to be involved on a formula specified in the customer's card.

Example formula text labels:

SokrLP (Int (Nomenklatura.Tsena) +1) + "" + Leo (Nomenklatura.NomenklaturnayaGruppa.Kod, 2) + Leo (Nomenklatura.Proizvoditel.Kod, 2) + "x" + Format (VAT, "CC = 9; CHDTS = 0 ")

Formulas should be written in a programming language 1C.

View realized prices - (If enabled complex mechanism accounting price) Allows you to control the view of the implementation of the price is to store (automatically selected when choosing dokmuente warehouse).

Responsible officer - (if included payroll accounting) A employee, at wages which can be uchetst results Inventory on this stock. The report on the Remains of the goods in the types Price version of the report is provided "for the responsible employees."

Warehouse Replenishment - if including accounting Warehouse. When implemented in a store may be a situation where product not in the shop (in stock, "store"), but he is in the back room. In order not to generate the program travel document this product from the auxiliary storage in a warehouse store before its implementation, you can specify the warehouse "store room" as Warehouse Replenishment in the card store "shop". Then the program will write off items from this store replenishment in the event that the implementation of stock he already ended. The chain of several warehouse replenishment is not working. This mechanism operates only in paper consumption goods. This mechanism does not apply to the constituent elements set forth the implementation of the set. In such cases, the user must pre-register the movement of goods.

Weight, Length, Width, Height - A description of the warehouse.
