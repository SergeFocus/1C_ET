Directory WAREHOUSES

Designed for accounting balances of goods and their movements across multiple warehouses.

Provided pedopredelennye "main storage" and "goods in transit". Warehouse for goods in transit should not be used for receipts or documents implementation.

To use it in program settings should be a check mark next to 'Account Warehouse. " "Warehouse" allows you to separately consider the movement of goods and the remains of their physical or virtual location.

Reference is optional (it can be found in the section "Additional Reference window "Settings Accounting, progammy").

Formula text labels - (If included the use of formulas) Defines a unique formula of formation of the text on the label when printing labels for this warehouse. Such formulas are stored in handbook formulas.

For example, if the document "Moving goods" as a recipient Set Warehouse Warehouse with his formula tesksta labels, calling subroutine label printing the printer will label with text generated by this formula.

If Formula text boxes around Depot is not filled, it will use the basic formula of the formation of the text labels program settings. If it has not been filled and then the text will be equal to the text labels Barcode nomenclature code.

Formula for text labels Client cards has an advantage over formula warehouse. For example, if printing labels for a table document Consumption goods the implementation of the client, the first thing will be employed on a formula specified in the customer's card.

View realized price - (If enabled complex mechanism accounting prices) Allows you to control the appearance of prices with the implementation of this stock (in dokmuente automatically select when choosing a warehouse).

Warehouse Replenishment - If included accounting Warehouse. When implemented in a store can be a situation when Item Do not store warehouse ("shop"), but it is in the utility room. In order not to generate the program travel document this product from the utility store warehouse in warehouse before they are implemented, you can specify the warehouse "back room" as Warehouse Replenishment cards in the warehouse "store." Then the program will deduct items from this store replenishment if the stock implementation, it has ended. Chain of several stores replenishment does not work. This mechanism operates only in document the flow of goods. This mechanism does not apply to the elements of the set of direct implementation of a set. In such cases, the user needs to pre-register the movement of goods.
