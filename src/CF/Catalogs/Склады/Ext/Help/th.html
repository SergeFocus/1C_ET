ไดเรกทอรีของคลังสินค้า

ออกแบบมาสำหรับเศษสินค้าบัญชีและการเคลื่อนไหวของพวกเขาข้ามคลังสินค้าหลาย

มี "การจัดเก็บหลัก" และ "สินค้าในการขนส่ง" pedopredelennyeคลังสินค้าสำหรับสินค้าในระหว่างการขนส่งไม่ควรนำมาใช้ในการรับเอกสารหรือการดำเนินการ

หากต้องการใช้ไดเรกทอรีการตั้งค่าโปรแกรม มันควรจะเป็นเครื่องหมายถัดจาก "บัญชีโดยพยางค์.""โกดัง" ช่วยให้คุณถ่ายแยกเข้าบัญชีการเคลื่อนไหวของสินค้าและซากของตำแหน่งทางกายภาพหรือเสมือนของพวกเขา

ไดเรกทอรี "โกดัง" เป็นตัวเลือก (มันสามารถพบได้ในส่วน "คู่มือเพิ่มเติมหน้าต่างการตั้งค่าโปรแกรมบัญชี

ป้ายข้อความสูตร - (ถ้ามันรวมถึงการใช้สูตร) กำหนดสูตรที่ไม่ซ้ำกันสำหรับการก่อตัวของข้อความบนฉลากเมื่อคุณพิมพ์ป้ายชื่อสำหรับหุ้นที่กำหนดสูตรดังกล่าวจะถูกเก็บไว้ในสูตรอ้างอิง

ตัวอย่างเช่นถ้าเอกสาร"สินค้าย้าย"เป็นผู้รับคลังสินค้าตั้งคลังสินค้าด้วยสูตรของป้ายข้อความเมื่อคุณเรียกขั้นตอนการพิมพ์ฉลากเครื่องพิมพ์จะติดป้ายที่มีข้อความที่สร้างโดยสูตรนี้

ถ้าสูตรของป้ายข้อความในโกดังที่ไม่ได้เต็มไปก็จะใช้สูตรพื้นฐานสำหรับการก่อตัวของป้ายข้อความการตั้งค่าโปรแกรมถ้ายังไม่ได้เต็มไปแล้วข้อความของฉลากจะเท่ากับข้อความบาร์โค้ดศัพท์

ป้ายข้อความสูตรการ์ดไคลเอนต์ มันมีประโยชน์มากกว่าคลังสินค้าสูตรดังนั้นถ้าคุณกำลังพิมพ์ฉลากสำหรับตารางเอกสารเครื่องอุปโภคบริโภค การดำเนินงานของลูกค้าเป็นหลักสูตรที่ระบุไว้ในบัตรของลูกค้าที่จะนำมาใช้

ป้ายข้อความสูตรตัวอย่าง:

SokrLP (int (Nomenklatura.Tsena) + 1) + "" + สิงห์ (Nomenklatura.NomenklaturnayaGruppa.Kod 2) + สิงห์ (Nomenklatura.Proizvoditel.Kod 2) + 'X' + รูปแบบ (VAT "CC = 9; CHDTS = 0 ")

สูตรควรจะเขียนในภาษาของการเขียนโปรแกรม 1C

การใช้งานราคาชนิด - (หากเปิดใช้งานกลไกที่ซับซ้อนของการบัญชีราคา) ช่วยให้คุณควบคุมการปรากฏตัวของราคากับการดำเนินงานของคลังสินค้านี้ (เลือกโดยอัตโนมัติเมื่อคุณเลือกคลังสินค้าเอกสาร)

เจ้าหน้าที่ผู้รับผิดชอบ - (ถ้าเงินเดือนรวม) นี้ลูกจ้างที่ค่าจ้าง ซึ่งสามารถ uchetst ผลสินค้าคงคลัง ในสต็อกนี้รายงานบนซากของสินค้าในประเภทราคา รุ่นของรายงานที่มีให้ "สำหรับพนักงานผู้รับผิดชอบ."

เติมเต็มโกดัง - ถ้ารวมทั้งบัญชี คลังสินค้า.เมื่อนำมาใช้ในการจัดเก็บอาจจะเป็นสถานการณ์ที่สินค้า ไม่มีการจัดเก็บคลังสินค้า ( "ร้าน") แต่มันอยู่ในห้องด้านหลังเพื่อไม่ให้สร้างโปรแกรมเอกสารการเดินทาง ผลิตภัณฑ์นี้จากการจัดเก็บช่วยในการจัดเก็บคลังสินค้าเพื่อการดำเนินงานของสามารถระบุ Warehouse "ห้องเก็บ" เป็น เติมเต็มคลังสินค้า ในสต็อกการ์ด "เก็บ"จากนั้นโปรแกรมจะเขียนออกการเติมเต็มสินค้าของหุ้นในกรณีที่การดำเนินการของหุ้นจะได้สิ้นสุดลงแล้วห่วงโซ่ของการเติมเต็มคลังสินค้าหลายที่ไม่ได้ทำงานกลไกนี้จะมีผลเฉพาะในการไหลเวียนของสินค้ากระดาษกลไกนี้ใช้ไม่ได้กับชุดขององค์ประกอบขององค์ประกอบอื่น ๆ การดำเนินงานของตลาดหลักทรัพย์แห่งประเทศไทยในกรณีเช่นนี้ผู้ใช้จะต้องลงทะเบียนล่วงหน้าการเคลื่อนไหวของสินค้า

น้ำหนัก, ความยาวความกว้างความสูง - รายละเอียดของคลังสินค้า

หากมีการพิจารณาแล้วว่ามีสต็อกนี้ไม่สามารถขาย, ค่าใช้จ่ายในการดำเนินงานของเอกสารกับลูกค้าของหุ้นจะไม่ได้รับการจดทะเบียนมันเป็นสิ่งจำเป็นเช่นการจัดเก็บหรือจัดเก็บสื่อกลางของการแต่งงาน
