Довідник склади

Призначений для ведення обліку залишків товару і їх рухів за кількома складах.

Передбачені педопределенние "основний склад" і "товари в дорозі".Склад для товару в дорозі не слід використовувати для документів надходження або реалізації.

Для використання довідника вналаштуваннях програми повинна бути встановлена ​​галочка навпроти пункту "Облік по складах"."Склад" дозволяє окремо враховувати руху і залишки товару по їх фізичному або віртуальному місцезнаходженням.

Формула тексту етикетки - (Якщовключено використання формул) Визначає унікальну формулу формування тексту на етикетці при друку етикетки для даного складу.Такі формули зберігаються вдовіднику формул

Наприклад, якщо в документі"Переміщення товару"як Складу-отримувача вказано Склад зі своєю формулою тексту етикетки, то при викликупідпрограми друку етикетокна принтер піде етикетка з текстом, сформованим за цією формулою.

Якщо Формула тексту етикетки у Складу не заповнена, то буде використана Основна формула формування тексту етикеток зналаштувань програмиЯкщо не заповнена і вона, то текст етикетки буде дорівнює тексту Штрих-коду номенклатури.

Формула тексту для етикеткикартки Клієнта має перевагу над формулою складу.Так, якщо йде друк етикетки для таблиці документаВитрата товару при реалізації клієнту, то в першу чергу буде задіяна формула, зазначена в картці Клієнта.

Приклад формули тексту етикетки:

СокрЛП (Цілий (Номенклатура.Цена) + 1) + "" + Лев (Номенклатура.НоменклатурнаяГруппа.Код, 2) + Лев (Номенклатура.Проізводітель.Код, 2) + "x" + Формат (Ціна, "ЧЦ = 9; ЧДЦ = 0 ")

Формули повинні бути написані мовою програмування 1С.

Відповідальний співробітник - (Якщовключений облік заробітної плати) такоїспівробітник, взарплати якого можна учетсть результатиінвентаризації на цьому складі.У звіті поЗалишкам Товару в видах Цен передбачений варіант звіту "по відповідальним співробітникам".

Склад Поповнення - якщовключений облік по складах.При реалізації в магазині може виникнути ситуація, колитовару немає в магазині (на складі "магазин"), але він є в підсобному приміщенні.Щоб не формувати в програмідокумент переміщення цього товару з підсобного складу в склад магазину перед його реалізацією, можна вказати склад "підсобного приміщення" як складу Поповнення в картці складу "магазин".Тоді програма буде списувати товар з цього складу поповнення в тому випадку, якщо на складі реалізації він вже закінчився.Ланцюжок з кількох складів поповнення НЕ працює.Цей механізм діє тільки вдокументі витрати товаруЦей механізм не діє для елементів складу набору ін реалізації набору.У таких випадках користувачеві потрібно заздалегідь зареєструвати переміщення товару.
