Документ коригування, реєстрації залишків, взаємозаліку боргу контрагентів

Під час перенесення даних обліку в програму їх найлегше зареєструвати в документі "введення залишків".Там передбачені механізми для швидкої реєстрації нових даних.

  • Потім, під час роботи, може бути знадобиться внести коригування в дані програми.Мається на увазі така Коригування, яку вони передбачають інші документи програми.Наприклад, це може бути переоцінка боргу контрагента без супутнього руху товару і грошей.
  • Третій режим документа - реєстрація взаємозаліків контрагентів.Особливо може стати в нагоді, якщо є постачальники, які так само виступають клієнтами і навпаки.Таким документом можна списати борг одного за рахунок боргу перед іншим.

    • Для коригування боргів контрагентів (клієнтів і постачальників), пов'язаної з проведенням взаємозаліку служить спеціальна форма цього документа "взаємозалік БОРГУ КОНТРАГЕНТІВ" (описана нижче).

    Для блоку Товари, Ціни

    Призначити Склад для всіх рядків - якщо використовується облік поскладах, При введенні залишків або їх корекції для товару необхідно вказувати склад.

    ВстановлюватиВид цін - Закупівельна. для того, щоб для всіх товарів при введенні залишків зафікісровать ціну в якості закупівельної ціни, і, можливо, розрахувати з неї залежні види цін.Основним засобом реєстрації цін служить документ "Установка цінПри введенні залишків можна так само заповнити ціни в закладі "Установка цін".При включеному прапорі "не встановлювати залежні ціни" будуть встановлені тільки безпосередньо зазначені види цін.

    Встановлювати ціну з таблиці в якості ціни для реалізації товару (Рівні ціни).Інакше буде застосована формула націнки.Якщо складний облік цін Чи не використовується, ціни з таблиці товару можна зареєструвати відразу в якості ціни для реалізації (наприклад коли облік цін передбачає ціну надходження рівній ціні продажу).

    Одиницю виміру (При обліку одиниць виміру) слід вказувати тільки в тому випадку, якщо в цей раз вона відмінна від основної одиниці виміру номенклатури.Це правило для всіх документів конфігурації.

    Команди:

    • Створити Товар і додати"- Відкриється форма нового об'єкта номенклатури і він додається в таблицю.
    • Заповнити залишкамитовару "- таблиця товару буде заповнена показниками залишків товарів на дату / час документа.
    • Для введених товарів прочитати залишки"- Для всіх рядків таблиці значення" кількість "і" ціна "буде заповнена показниками залишків товарів на дату / час документа.
    • "Заповнити Всіма товарами" - Вся номенклатура потрапляє в таблицю товару.Зручно при введенні залишків, якщо номенклатура вже була завантажена вдовідник
    • "Заповнити Вид цін" - Заповнити всю колонку таблиці Ціни одним видом цін.
    • Встановити Ціни продажів"- Для всіх рядків таблиці значення" ціна "буде встановлено такою ціною, яка зареєстрована для продажу товару.За замовчуванням ціна в даній таблиці повинна бути ціною придбання (закупівельної).

    При регістріраціі коригування залишків товару типу "набір" програма застосує коригування для всіх елементів складу набору окремо.Настійно рекомендуємо все ж проводити коректування для кожного елемента набору окремо, щоб чітко встановити закупівельну ціну для кожного компонента "набору", його вартість і фактичні показники залишків.Якщо Вас не цікавить в програмі точний розрахунок показників прибутку, то цією рекомендацією можна знехтувати.

    Коригування цим документом не впливає на валовий прибуток.Списання слід виконувати документомВитрата товару без заповнення контрагента.

    За кнопці F9 доступний діалог швидкого введення товару з кількістю.При його використанні достатньо натиснути F9, потім ввести сканером або вручну штрих кодвін використовується) Або вибрати номенклатуру, вказати Кількість і F9 - завершити введення даних.

    За кнопці F11 введення поШтрих-коду вручну (якщовін використовується), Наприклад для випадків коли сканер коду відключений або не читає код.

    Для Блоку Розрахунків з Клієнтами

    Команди:

    • Створити Клієнта і додати"- Відкриється форма новогоКлієнта і він буде доданий в таблицю.
    • Заповнити залишками розрахунків з клієнтами"- Таблиця буде заповнена усіма даними про поточний стан заборгованості клієнтів.

    Взаєморозрахунки слід вказувати восновній валютіпрограми.Переплата Клієнта відбивається негативною цифрою (з мінусом).

    Для Блоку Розрахунків з Постачальниками

    Команди:

    • Створити Постачальника і додати"- Відкриється форма новогоПостачальника і він буде доданий в таблицю.
    • Заповнити залишками розрахунків з постачальниками"- Таблиця буде заповнена усіма даними про поточний стан заборгованості перед постачальниками.

    Взаєморозрахунки слід вказувати восновній валютіпрограми.Переплата пере Постачальником відбивається негативною цифрою (з мінусом).

    Взаємозалік БОРГУ КОНТРАГЕНТІВ

    Це спеціальний тип документа для реєстрації взаємозаліку боргу Клієнтів і / або Постачальників.У цей режим дає змогу списати борг одного Клієнта в рахунок переплати іншого Клієнта.Або коли, наприклад, клієнт і постачальник одну особу, провести для нього взаємозалік боргу.

    Так само можна відобразити тут надходження або видаток грошей (якщо ведеться облік грошей).

    Дані "Про рядку" -заметкі, інформація користувача.На друк не виводяться, але зберігаються в програмі.
