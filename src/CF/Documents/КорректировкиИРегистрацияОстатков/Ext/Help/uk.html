• РЕЄСТРАЦІЯ ЗАЛИШКІВ, КОРЕКТУВАННЯ

    Для початку роботи з програмою в документі слід відобразити всю інформацію про поточний стан по блоках обліку, які включені в настройках на дату / час документа.

    Це блоки:

    • "Залишки товарів"В закупівельних цінах
    • Початкові "Ціни" товарів в видах цін
    • Поточні показники розрахунків з Клієнтами
    • Поточні показники розрахунків з Постачальниками
    • Поточні залишки грошей в Валютах
    • Поточна заборгованість по заробітній платі співробітників.

    Далі документ може застосовуватися для коригування показників. Врахуйте, що:

    • Для коригування показників товарів рекомендується використовувати документ Інвентаризація.
    • При складному механізмі урахування цін, Для їх коригування та установки цін слід використовувати документ "Установка Цен".
    • Для коригування боргів контрагентів (клієнтів і постачальників), пов'язаної із взаємозаліком служить спеціальна форма цього документа "взаємозалік БОРГУ КОНТРАГЕНТІВ" (описана нижче).

    Для блоку Товари, Ціни

    Призначити Склад для всіх рядків - Якщо використовується облік по складам, При введенні залишків або їх корекції для товарів необхідно вказувати склад.

    Встановлювати Вид цін - Закупівельна. - для того, щоб для всіх товарів при введенні залишків зафікісровать ціну як закупівельної ціни, і, можливо, розрахувати з неї залежні види цін. Основним засобом реєстрації цін служить документ "Установка цін". При введенні залишків можна так само заповнити ціни в закладі" Установка цін ".

    Встановлювати ціну з таблиці в якості ціни для реалізації товару (Рівні ціни). Інакше буде застосована формула націнки. Якщо складний облік цін не використовується, ціни з таблиці товарів можна зареєструвати відразу в якості ціни для реалізації (наприклад коли облік цін передбачає ціну постапленія рівній ціні продажу).

    Команди:

    • "Створити Товар і додати"- Відкриється форма нового об'єкта номенклатури і він додається в таблицю.
    • "Заповнити залишками товарів"- Таблиця товарів буде заповнена показниками залишків товарів на дату / час документа.
    • "Для введених товарів прочитати залишки"- Для всіх рядків таблиці значення" кількість "і" ціна "буде заповнена показниками залишків товарів на дату / час документа.
    • "Заповнити Всіма товарами" - Вся номенклатура потрапляє в таблицю товарів. Зручно при введенні залишків, якщо номенклатура вже була завантажена в довідник.
    • "Заповнити Вид Цін" - Заповнити всю колонку таблиці Ціни одним видом цін.
    • "Встановити Ціни продажів"- Для всіх рядків таблиці значення" ціна "буде встановлено такою ціною, яка зареєстрована для продажу товару. За замовчуванням ціна в даній таблиці повинна бути ціною придбання (закупівельної).

    При регістріраціі коригування залишків товарів типу "набір" програма застосує коригування для всіх елементів складу набору окремо. Настійно рекомендуємо все ж проводити коректування для кожного елемента набору окремо, щоб чітко встановити закупівельну ціну для кожного компонента "набору", його вартість і фактичні показники залишків. Якщо Вас не цікавить у програмі точний розрахунок показників прибутку, то даною рекомендацією можна знехтувати.

    Коригування цим документом не впливає на валовий прибуток. Списання слід виконувати документом Витрата товару без заповнення контрагента.

    По кнопці F10 доступний діалог швидкого введення товару з кількістю. При його використанні досить натиснути F10, потім ввести сканером або вручну штрих код (Якщо він використовується) Або вибрати номенклатуру, вказати Кількість і F10 - завершити введення даних.

    По кнопці F11 введення по Штрих-коду вручну (якщо він використовується), Наприклад для випадків коли сканер коду відключений або не читає код.

    Для Блоку Розрахунків з Клієнтами

    Команди:

    • "Створити Клієнта і додати"- Відкриється форма нового Клієнта і він буде додано в таблицю.
    • "Заповнити залишками розрахунків з клієнтами"- Таблиця буде заповнена усіма даними про поточний стан заборгованості клієнтів.

    Взаєморозрахунки слід вказувати в основній валюті програми. Переплата Клієнта відбивається негативною цифрою (з мінусом).

    Для Блоку Розрахунків з Постачальниками

    Команди:

    • "Створити Постачальника і додати"- Відкриється форма нового Постачальника і він буде додано в таблицю.
    • "Заповнити залишками розрахунків з постачальниками"- Таблиця буде заповнена усіма даними про поточний стан заборгованості перед постачальниками.

    Взаєморозрахунки слід вказувати в основній валюті програми. Переплата пере Постачальником відбивається негативною цифрою (з мінусом).

    При вказівці коригування / залишків грошей

    Команда:

    • "Заповнити поточними залишками грошей"- Таблиця залишків буде заповнена за даними програми на дату / час документа.

    Взаємозалік БОРГУ КОНТРАГЕНТІВ

    Це специалит Вид документа для реєстрації взаємозаліку боргу Клієнтів та / або Постачальників. У цей режимі можна списати борг одного Клієнта в рахунок переплати іншого Клієнта. Або коли, наприклад, клієнт і постачальник одна особа, призвести для нього взаємозалік боргу.

    Так само можна відобразити тут надходження або витрату грошей (якщо ведеться облік грошей).
