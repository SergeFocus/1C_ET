Документ корректировки, регистрации остатков, взаимозачета долга контрагентов

Во время переноса данных учета в программу их легче всего зарегистрировать в документе "ввод остатков". Там предусмотрены механизмы для быстрой регистрации новых данных.

  • Затем, во время работы, может быть понадобится внести корректировку в данные программы. Подразумевается такая Корректировка, которую не предполагают другие документы программы. Например, это может быть переоценка долга контрагента без сопутствующего движения товара и денег.
  • Третий режим документа - регистрация взаимозачетов контрагентов. Особенно может пригодиться, если есть поставщики, которые так же выступают клиентами и наоборот. Таким документом можно списать долг одного за счет долга перед другим.

     

    Регистрация остатков, Корректировка

    Для начала работы с программой в документе следует отразить всю информацию о текущем состоянии по блокам учета, которые включены в настройках на дату/время документа.

    Это блоки:

    • "Остатки товара" в закупочных ценах
    • Первоначальные "Цены" товара в видах цен
    • Текущие показатели расчетов с Клиентами
    • Текущие показатели расчетов с Поставщиками
    • Текущие остатки денег в Валютах
    • Текущая задолженность по заработной плате сотрудников.

     

    Далее документ может применяться для корректировки показателей. Учтите, что:

    • Для корректировки показателей товара рекомендуется использовать документ Инвентаризация.
    • При сложном механизме учета цен, для их корректировки и установки цен следует использовать документ "Установка Цен".
    • Для корректировки долгов контрагентов (клиентов и поставщиков), связанной с взаимозачетом служит специальная форма этого документа "ВЗАИМОЗАЧЕТ ДОЛГА КОНТРАГЕНТОВ" (описана ниже).

     

    Для блока Товары, Цены

    Назначить Склад для всех строк - если используется учет по складам, при вводе остатков или их коррекции для товара необходимо указывать склад.

    Устанавливать Вид цен - Закупочная. - для того, чтобы для всех товаров при вводе остатков зафикисровать цену в качестве закупочной цены, и , возможно, рассчитать из нее зависимые виды цен. Основным средством регистрации цен служит документ "Установка цен". При вводе остатков можно так же заполнить цены в закладе "Установка цен". При включенном флаге "не устанавливать зависимые цены" будут установлены только непосредственно указанные виды цен.

    Устанавливать цену из таблицы в качестве цены для реализации товара (равные цены). Иначе будет применена формула наценки. Если сложный учет цен Не используется, цены из таблицы товара можно зарегистрировать сразу в качестве цены для реализации (например когда учет цен предполагает цену поступления равной цене продажи).

    Единицу измерения (при учете единиц измерения) следует указывать только в том случае, если в этот раз она отличная от основной единицы измерения номенклатуры. Это правило для всех документов конфигурации.

    Команды:

    • "Создать Товар и добавить" - откроется форма нового объекта номенклатуры и он добавляется в таблицу.
    • "Заполнить остатками товара" - таблица товара будет заполнена показателями остатков товаров на дату/время документа.
    • "Для введенных товаров прочитать остатки" - для всех строк таблицы значения "количество" и "цена" будет заполнена показателями остатков товаров на дату/время документа.
    • "Заполнить Всеми товарами" - Вся номенклатура попадает в таблицу товара. Удобно при вводе остатков, если номенклатура уже была загружена в справочник. 
    • "Заполнить Вид цен" - Заполнить всю колонку таблицы Цены одним видом цен. 
    • "Установить Цены продаж" - для всех строк таблицы значение "цена" будет установлено такой ценой, которая зарегистрирована для продажи товара. По умолчанию цена в данной таблице должна быть ценой приобретения (закупочной).

    При регистрирации корректировки остатков товара типа "набор" программа применит корректировку для всех элементов состава набора отдельно. Настоятельно рекомендуем все же производить корректировку для каждого элемента набора отдельно, чтобы четко установить закупочную цену для каждого компонента "набора", его стоимость и фактические показатели остатков. Если Вас не интересует в программе точный расчет показателей прибыли, то данной рекомендацией можно пренебречь.

    Корректировка этим документом не влияет на валовую прибыль. Списание следует выполнять документом Расход товара без заполнения контрагента.

    По кнопке F9 доступен диалог быстрого ввода товара с количеством. При его использовании достаточно нажать F9, затем ввести сканером или вручную штрих код (если он используется) или выбрать номенклатуру, указать Количество и F9 - завершить ввод данных.

    По кнопке F11 ввод по Штрих-коду вручную (если он используется), например для случаев когда сканер кода отключен или  не читает код.

     

    Для Блока Расчетов с Клиентами

    Команды:

    • "Создать Клиента и добавить" - откроется форма нового Клиента и он будет добавлен в таблицу.
    • "Заполнить остатками расчетов с клиентами" - таблица будет заполнена всеми данными о текущем состоянии задолженности клиентов.

    Взаиморасчеты следует указывать в основной валюте программы. Переплата Клиента отражается отрицательной цифрой (с минусом).

     

    Для Блока Расчетов с Поставщиками

    Команды:

    • "Создать Поставщика и добавить" - откроется форма нового Поставщика и он будет добавлен в таблицу.
    • "Заполнить остатками расчетов с поставщиками" - таблица будет заполнена всеми данными о текущем состоянии задолженности перед поставщиками.

    Взаиморасчеты следует указывать в основной валюте программы. Переплата пере Поставщиком отражается отрицательной цифрой (с минусом).

     

    При указании корректировки/остатков  денег

    Команда:

    • "Заполнить текущими остатками денег" - таблица остатков будет заполнена по данным программы на дату/время документа.

     

    Если используются ваучеры, остатки их наличия регистрируются через обработку "Операции с ваучерами" документом поступления.

     

    ВЗАИМОЗАЧЕТ ДОЛГА КОНТРАГЕНТОВ

    Это специальный тип документа для регистрации взаимозачета долга Клиентов и/или Поставщиков. В это режиме можно списать долг одного Клиента в счет переплаты другого Клиента. Или когда, например, клиент и поставщик одно лицо, произвести для него взаимозачет долга.

    Так же можно отразить тут поступление или расход денег (если ведется учет денег).

    Данные "О строке" -заметки, информация пользователя. На печать не выводятся, но хранятся в программе.
