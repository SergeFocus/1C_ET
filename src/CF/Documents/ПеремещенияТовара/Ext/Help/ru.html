Документ перемещения товара

При использовании нескольких складов, товары с количественным учетом в программе также следует перемещать со склада на склад.

Перемещение с использованием склада "товар в пути" необходимо для случаев, когда нужно учесть, что товар временно недоступен в связи с отсутствием на таких складах, с которых его можно реализовать.

При перемещении товара типа "Набор", на самом деле производится перемещение всех элементов его состава. Но для четкого учета фактических остатков рекомендуется все же перемещать каждый элемент такого "набора" отдельно. При перемещении "набора" сумма учета для его компонент рассчитывается автоматически для каждого элемента. Указанная в таблице документа игнорируется.

Когда используется Сложный Механизм Учета цен и в документе указан Вид цен, то учет списания со склада источника и оприходования на склад получатель производится по установленному виду цен в т.ч. с учетом Валюты Цены (если используется учет по Валютам).

Колонки цена/сумма, реквизит вид цен используются только для отображения в документе

Программа использует внутренние виды цен.

Единицу измерения (при учете единиц измерения) следует указывать только в том случае, если в этот раз она отличная от основной единицы измерения номенклатуры. Это правило для всех документов конфигурации.

Данные "О строке" -заметки, информация пользователя. На печать не выводятся, но хранятся в программе.

По кнопке F9 доступен диалог быстрого ввода товара с количеством. При его использовании достаточно нажать F9, затем ввести сканером или вручную штрих код (если он используется) или выбрать номенклатуру, указать Количество и F9 - завершить ввод данных.

По кнопке F11 ввод по Штрих-коду вручную (если он используется), например для случаев когда сканер кода отключен или  не читает код.
