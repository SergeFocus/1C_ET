Документ "ПЕРЕМЕЩЕНИЕ ТОВАРА"

При использовании нескольких складов, товары с количественным учетом в программе также следует перемещать со склада на склад.

Перемещение с использованием склада "товар в пути" необходимо для случаев, когда нужно учесть, что товар временно недоступен в связи с отсутствием на таких складах, с которых его можно реализовать.

При перемещении товара типа "Набор", на самом деле производится перемещение всех элементов его состава. Но для четкого учета фактических остатков рекомендуется все же перемещать каждый элемент такого "набора" отдельно. При перемещении "набора" сумма учета для его компонент рассчитывается автоматически для каждого элемента. Указанная в таблице докмента игнорируется.

Когда используется Сложный Механизм Учета цен и в документе указан Вид Цен, то учет списания со склада источника и опримходования на склад получатель производится по установленному виду цен в т.ч. с учетом Валюты Цены (если используется учет по Валютам).

Колонки цена/сумма, реквизит Уид Цен используются только для отображения в документе

Программа использует внутренние виды цен.

Данные "О строке" -заметки, информация пользоателя. На печать не выводятся, но хранятся в программе.

По кнопке F10 доступен диалог быстрого ввода товара с количеством. При его использовании достаточно нажать F10, затем ввести сканером или вручную штрих код (если он используется) или выбрать номенклатуру, указать Количество и F10 - завершить ввод данных.

По кнопке F11 ввод по Штрих-коду вручную (если он используется), например для случаев когда сканер кода отключен или  не читает код.
