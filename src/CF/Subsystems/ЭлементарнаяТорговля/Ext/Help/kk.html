ОПИСАНИЕ ПОНЯТИЙ (ТЕРМИНОВ)

В документации и справке по конфигурации существуют понятия (термины), суть которых, возможно, нужно уточнить перед тем как знакомиться с программой глубже:

Справочник - это каталог или картотека для чего либо. Например "Справочник Номенклатуры (Товаров)" - картотека товаров. В среде 1С принято товары обозначать словом "номенклатура". Отсюда двойное наименование справочника.

Справочник в конфигурации обычно выглядит как список с элементами, который можно пролистать. Двойной щелчек откроет или выберет элемент справочника.

Если справочник "иерархический", значит его элементы можно разместить по группам. Элементы могут принадлежать какой либо группе или находится вне всех групп.

Элемент справочника - собственно элемент картотеки. Так для справочника Номенклатуры это описание собственно самого товар. Карточка элемента справочника содержит его реквизиты (параметры) - Наименование, Код и другие. В зависимости от того что это за справочник. Так у справочника "Клиенты" есть реквизиты "адрес", "телефон" и т.п. Реквизитами элемента справочника могут выступать ссылки на элементы другого справочника. Так реквизит "Регион" в карточке "Клиента" это ссылка на элемент в справочнике Регионов.

Среди реквизитов могут быть обязательные и необязательные. Так, например не следует оставлять незаполненными Наименования элементов. А вот "адрес" Клиента нужно указывать только, если он известен. Обычно обязательные реквизиты подчеркнуты красным.

Элементы могут располагаться в группах (желтые папки). Могут быть помечены на удаление ( с красным крестиком).

 Документ (регистратор) - это средство изменения состояния данных. Так, документ "перемещение товаров" регистрирует в системе выбытие товаров на одном складе и приходование их на другом. Изменятся показатели "остатков" товара. Для это следует открыть перечень документов "Перемещения", создать новый документ (клавиша INS или кнопка "Создать"). В открывшемся документе ввести все необходимые реквизиты документа: указать Дату и время перемещения, указать склады откуда и куда переместился товар, а затем заполнить таблицу документа  - ввести перечень тех товаров, которые были перемещены. Для товаров следует указать "Количество" - сколько товаров было перемещено.

Документ может быть записан, проведен или помечен на удаление. Записанный документ в отличии от проведенного, не производит изменений в базе данных.

 Отчет - специальный механизм конфигурации, в котором можно запустить анализ какой то части информации и получить результат - таблицу данных. Так отчет по Остаткам товара покажет таблицу для товаров с показателями - как изменялись состояния товара - сколько было их на начало периода, сколько поступило, сколько выбыло и сколько осталось. Для этого, перед формированием отчета, следует указать период анализа данных и, возможно, отбор по конкретному товару. Во многих отчетах предусмотрены готовые Варианты настроек, доступные по кнопке "Выбрать вариант". При этом пользователь может создавать собственные варианты на их основе. Самым интересным является блок Отбор в окне редакции варианта. Там можно отобрать данные для отчета гораздо точнее, чем с основной формы (шапки) окна отчета. См. справку по отчетам 1С.

 Обработка данных - специальный элемент конфигурации, который может выступать как средством анализа, так и изменения данных. Обработка Выгрузка и загрузка данных позволяет формировать файл с данными и обмениваться ими с другой копией базы данных.

Окно Настроек Учета, Программы - специальное место в конфигурации, где можно задать все условия работы программы, а затем менять их при необходимости.

 Роль пользователя определяет его права на изменение и даже просмотр определенных данных в базе данных. В конфигурации предусмотрены роли "Полные права", "Реализатор", "Реализация и прием товара, установка цен и курсов", "Только просмотр". Пользователь с ролью "Реализатор" не может вести взаиморасчеты с Поставщиками. А пользователи с ролью "Только просмотр" вообще ничего не могут изменить в программе - только просматривать данные.

Для каждого окна в конфигурации предусмотрена справка  (F1), в которой можно ознакомиться и уточнить вопросы относительно этого окна. Так, в окне карточки справочника "Номенклатура" откроется описание и инструкция по заполнению карточки товара.

Если работать с какой либо формой неудобно, ее можно изменить "под себя" по кнопке  настройки формы.
