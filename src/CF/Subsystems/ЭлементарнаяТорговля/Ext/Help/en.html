DESCRIPTION concepts (terms)

The documentation and help for configuration there are concepts (terms), the essence of which may need to be clarified before deeply acquainted with the program:

Directory - Is a directory or file cabinet for something. For example, "Directory Nomenclature (Products)"- File cabinet products. Among 1C made goods denoted by the word "nomenclature". Hence the double name of the directory.

Directory configuration usually looks like a list with elements that can be scrolled. Double-clicking selects an item or open the directory.

If the directory "hierarchical", It means that the elements can be placed on groups. Items may belong to any group, or is outside all the groups.

Directory entry - The actual item catalogs. So for the directory itself is a description of the Nomenclature of the goods. Card member directory contains its details (parameters) - Name, Code and others. Depending on what kind of book. So the directory "Clients"There are details of the" address "," phone ", etc. DETAILS element of the directory may make reference to other elements of the directory. So props" Region "in the cards" Client "is a reference to an element in the directory Regions.

Additional details may be required and optional. For example should not be left blank Names of elements. But the "address" of the Client only need to specify if it is known. Usually the required details are underlined in red.

The elements may be arranged in groups (Yellow folder). Can be marked for deletion (With the red cross).

Document (the receptionist) - It is a means of changing the data state. Thus, the document "movement of goods"Registers in the system of disposal of goods in one stock prihodovanie and blocked at the other. Change indicators "remnants" of the goods. To do this, open the list of documents "movements", create a new document (INS key or button "Create"). In the document, enter all the required document details: Specify the date and time of travel, warehouses specify how and where goods moved, and then fill document table - Enter a list of those items that have been moved. For goods should indicate "No." - how many products have been displaced.

The document can be written,held or marked for deletion. Recorded document unlike conducted, does not produce changes in the database.

Report - Special configuration mechanism in which you can run an analysis which is part of the information and get a result - the data table. So Report on balances Item will show a table for products with indicators - how the state of the goods - as they were at the beginning of the period, how many were received, how many dropped out and how much is left. For this purpose, before the formation of the report should indicate during data analysis and optionally selection for a particular product. Many reports are provided ready Dining settings available on the "Choose". The user can create their own versions based on them. The most interesting is a block selection in the amended version. There you can select the data for the report is much more accurate than the basic form (cap) of the report window. See Help for reports 1C.

Data processing - Special configuration element that can act as a means of analysis and data changes. Processing Unloading and loading of data allows you to create a data file and share them with another copy of the database.

Settings window Accounting Programme - a special place in the configuration where you can set all the conditions of the program and then change them if necessary.

User Role determines its right to modify or even browsing the specific data in a database. In the configuration provided as a "full rights", "Implementer", "Implementation and acceptance of goods, setting prices and exchange", "View Only". A user with the role of "implementer" can not lead to mutual Suppliers. A user with a "View Only" nothing can change in the program - only to view data.

For each window in the configuration provided reference (F1), in which you can explore and clarify issues regarding this window. Thus, in the card directory "Nomenclature" open description and instruction for completing the item card.

If you work with any form of uncomfortable, it can be changed "by itself" button setting a form.
