Concepts de description (termes)

La documentation et l'aide pour la configuration, il ya des concepts (termes), l'essence de ce qui peut avoir besoin d'être clarifiée avant que le programme plus profondément familiariser avec:

Répertoire - Un répertoire ou fichier CAB pour quelque chose. Par example "Nomenclature Directory (Produits)"-. Un catalogue de fichiers de la carte L'environnement 1C produits fabriqués désignés par le mot «gamme». D'où le double nom du répertoire.

Configuration manuelle apparaît généralement comme une liste des articles que vous pouvez faire défiler. Un double-clic ouvre l'élément ou choisissez le répertoire.

Si la référence "hiérarchique», Cela signifie que les éléments peuvent être placés sur la groupes. Articles peuvent appartenir à un groupe, ou est en dehors de tous les groupes.

Répertoire d'éléments - L'élément de dépôt effectif. Donc, pour ce répertoire Description de la nomenclature du produit lui-même. La carte contient son répertoire élément détails (paramètres) - Nom, ID, et d'autres. Selon le type de référence. Ainsi, le répertoire "Les clients"Il ya des détails de l '" adresse "," téléphone ", etc. REQUIS élément de répertoire peut agir sur les éléments de l'autre annuaire de liens. Donc accessoires" Région "dans les cartes" Client "est une référence à un élément dans le répertoire Les régions.

Des détails supplémentaires peuvent être requis et facultatifs. Par exemple ne devraient pas être laissés vacants noms d'éléments. Mais l '"adresse" du client doit être spécifié que si elle est connue. Typiquement, les informations requises sont soulignés en rouge.

Les éléments peuvent être disposés en groupes (dossier jaune). Peut être Pour la suppression (Avec une croix rouge).

Document (registraire) - Un moyen de changer la base de données d'état. Ainsi, le document "circulation des marchandises"Registres à la disposition des marchandises dans une Disponibilité et prihodovanie dans un autre. Changer indice des «restes» des marchandises. Pour ce faire, ouvrez la liste des documents «mouvements», créer un nouveau document (ou sur le bouton "Nouveau" touche INS). Dans le document, saisir toutes les requise détails du document: Spécifiez la date et l'heure de Voyage, de préciser comment et où les stocks déplacés produits, puis remplissez feuille de calcul - Entrez la liste des produits qui ont été déplacés. Pour les marchandises doivent indiquer "numéro" - combien de produits ont été déplacés.

Le document peut être enregistré par.en attente ou Pour la suppression. Le document enregistré contrairement menée ne produira pas des changements dans la base de données.

rapport - Une configuration particulière de mécanisme dans lequel vous pouvez exécuter une analyse que certains des renseignements et obtenir un résultat - la table de données. Ainsi Rapport sur les soldes Point présentera pour les produits avec des indicateurs - Comment changer le statut des marchandises - comme ils l'étaient au début de la période, tel que reçu, combien ont abandonné et combien est laissé. A cet effet, avant la formation du rapport devrait indiquer la période d'analyse de données et, éventuellement, sélection pour un produit spécifique. De nombreux rapports sont fournis prêts Options paramètres disponibles sur la «Choisir». L'utilisateur peut créer leurs propres versions basées sur eux. Le plus intéressant est la sélection du bloc dans la version modifiée. Là, vous pouvez sélectionner des données pour le rapport est beaucoup plus précise que la principale forme (caps) fenêtre de rapport. Cm. Aide pour les rapports 1C.

Traitement de l'information - Élément de configuration spéciale qui peut agir comme un moyen d'analyse et de modification des données. Traitement chargement et les données de téléchargement vous permet de créer des fichiers de données et de les partager avec une autre copie de la base de données.

Fenêtre Options Programme Comptabilité - une place particulière dans la configuration où vous pouvez définir toutes les conditions du programme, puis les changer si nécessaire.

Rôle de l'utilisateur détermine le droit de modifier ou même parcourant les données spécifiques dans la base de données. Dans la configuration prévue comme une "de plein droit", "mise en œuvre", "La mise en œuvre et l'acceptation des produits, la fixation des prix et des échanges", "View Only". L'utilisateur avec le rôle de la "mise en œuvre" ne peut pas conduire à mutuelle Fournisseurs. Un utilisateur avec le rôle "View Only" rien ne peut changer dans le programme - juste pour afficher les données.

Pour chaque fenêtre dans la configuration prévue référence (F1), dans lequel vous pouvez explorer et de clarifier des questions concernant cet écran. Ainsi, dans la carte manuel "nomenclature" Description ouverte et les instructions pour remplir la carte de l'article.

Bouton Si vous travaillez avec toute forme de mal à l'aise, il peut être modifié "par lui-même" Forme Paramètres.
